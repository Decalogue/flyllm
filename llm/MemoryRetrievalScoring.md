# 记忆检索得分函数设计

## 1. 核心定性
本质上，记忆检索得分函数是一个**多目标优化问题**，通过加权融合语义相关性、时间新鲜度和业务重要性三个正交维度，实现对海量记忆的高效排序与截断。

---

## 2. 具体流程

1. **向量化索引**：将历史记忆通过 Embedding 模型编码为稠密向量，构建 ANN 索引（如 HNSW、FAISS）实现亚线性近邻检索
2. **多因子打分**：对召回的 Top-K 候选记忆，分别计算向量相似度、时间衰减因子和重要性权重，代入融合公式得到最终得分
3. **重排序截断**：按融合得分降序排列，取 Top-N 作为最终检索结果注入 Prompt 上下文

---

## 3. 数学基础

### 3.1 融合得分公式（加性融合 + Sigmoid 归一化）

$$S_{final} = \alpha \cdot \sigma(S_{sim}) + \beta \cdot S_{time} + \gamma \cdot S_{imp}$$

**约束条件**：$\alpha + \beta + \gamma = 1$，且 $\alpha, \beta, \gamma \in [0, 1]$

其中：
- $S_{sim}$：向量余弦相似度，$S_{sim} = \frac{\vec{q} \cdot \vec{m}}{||\vec{q}|| \cdot ||\vec{m}||}$，$\vec{q}$ 为查询向量，$\vec{m}$ 为记忆向量
- $\sigma(\cdot)$：Sigmoid 函数，将相似度压缩到 $(0,1)$ 区间，增强区分度
- $S_{time}$：时间衰减因子，$S_{time} = e^{-\lambda \cdot \Delta t}$，$\Delta t = t_{now} - t_{mem}$（时间差，单位：小时）
- $\lambda$：衰减系数，通常取 $0.01 \sim 0.1$，控制"记忆半衰期"
- $S_{imp}$：重要性权重，可通过显式标注（用户标记）或隐式学习（访问频次、反馈信号）获得

### 3.2 乘性融合变体（对异常值更敏感）

$$S_{final} = S_{sim}^{w_1} \cdot S_{time}^{w_2} \cdot S_{imp}^{w_3}$$

**适用场景**：当任一维度出现极端低值时应直接淘汰该记忆（如完全过期的信息即使语义相关也不应召回）

## 4. 工程考量

| 维度 | Trade-off | 致命弱点 |
|------|-----------|----------|
| **向量相似度** | 牺牲精确匹配换取语义泛化能力 | 对专有名词、ID、代码片段的精确召回差；Embedding 模型领域适配成本高 |
| **时间衰减** | 简单指数衰减无法处理周期性事件（如"每周例会"） | 突发热点被快速遗忘，长期重要但低频访问的记忆被"雪藏" |
| **重要性权重** | 显式标注准确但用户体验差；隐式学习冷启动困难 | 反馈信号稀疏时，$S_{imp}$ 退化为均匀分布，失去区分能力 |
| **融合权重** | 静态权重 $\alpha/\beta/\gamma$ 无法自适应不同查询类型 | 事实类查询应重 $S_{sim}$，会话类查询应重 $S_{time}$，固定权重导致检索质量波动 |

**关键优化点**：
- 引入**动态权重网络**：根据 Query 意图分类（事实检索 vs 会话续接）动态调整 $\alpha/\beta/\gamma$
- 时间衰减升级：采用**双指数衰减**或引入"刷新机制"（访问即重置 $t_{mem}$）

## 5. 工业映射

在工业界，该机制被直接应用于以下系统：

| 系统 | 具体模块 | 技术细节 |
|------|----------|----------|
| **LangChain** | `VectorStoreRetriever` | 支持 `search_kwargs` 注入时间过滤器和重要性阈值，底层默认使用 MMR（最大边际相关性）做重排序 |
| **MemGPT** | 记忆分层架构 | 核心记忆（Core Memory）用高固定 $S_{imp}$ 保活，归档记忆（Archival Memory）依赖向量检索 + 时间衰减 |
| **OpenAI Assistant API** | File Retrieval | 采用 RRF（Reciprocal Rank Fusion）融合向量搜索与 BM25 关键词匹配，时间因子通过元数据过滤实现 |
| **Claude 3** | Long Context + RAG | 官方未披露细节，但实测显示其记忆检索对时间衰减极敏感，48h 前的对话引用优先级显著下降 |

---

## 总结

记忆检索得分函数的设计没有银弹：
- **加性融合**适合各维度独立贡献的场景，调参友好
- **乘性融合**适合"一票否决"的严格过滤场景
- **动态权重**是生产环境的必选项，静态权重只存在于 Demo 中

真正的工业级挑战不在于公式本身，而在于**重要性权重的在线学习**和**时间衰减的个性化建模**——这是区分玩具级记忆系统与生产级记忆系统的核心分水岭。
