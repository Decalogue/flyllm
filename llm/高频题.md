## 怎样用自己的话结构化表达（深入浅出）

### 表达结构：三层递进

| 层次 | 做什么 | 目的 |
|------|--------|------|
| **一句话** | 生活化比喻或一句话定义 | 让面试官立刻听懂「这是啥」 |
| **三句话** | 原理/流程/关键点，按 1-2-3 说清 | 结构清晰、不散、好记 |
| **可选的深** | 公式/变体/与其他技术对比 | 体现深度，问到再展开 |

原则：**先给锚点（一句话），再搭骨架（三句话），最后按需补细节。**

### 四个技巧

- **从「用」到「理」**：先说是解决什么问题的，再说数学/实现。
- **用类比**：如 LoRA ≈ 只学「小补丁」、KV Cache ≈ 算过的存起来下次直接用。
- **用对比**：和另一种方案比一句（如 Pre-LN vs Post-LN），说清「为啥选这个」。
- **控制深度**：首轮用「一句话 + 三句话」；追问再上公式、数据、论文。

### 自检清单（每题背完问自己）

- [ ] 能用**一句**非术语说清「是什么」？
- [ ] 能用**三个小点/三步**把主流程讲完？
- [ ] 有一个**类比或对比**？
- [ ] 被问「为什么」时能用「因为…所以…」接一句？

### 示例 1：Self-Attention 为什么要除以 √d_k

**一句话**：防止向量变长时 Q、K 点积过大，导致 softmax 太尖、梯度变小。

**三句话**：  
1）Q、K 点积的数值会随维度 d_k 增大而变大。  
2）变大后 softmax 会非常尖，几乎只有一个位置接近 1，梯度很小。  
3）除以 √d_k 把点积 scale 回合理量级，梯度更稳定。

**可选的深**：因此叫 "scaled" dot-product attention；可对比不加 scale 时梯度随 d_k 的变化。

### 示例 2：LoRA 是什么

**一句话**：不直接改大权重矩阵，只学一个低秩的「小补丁」加上去，省显存、好切换任务。

**三句话**：  
1）假设增量更新是低秩的，用 \( W' = W + BA \)（B、A 小矩阵）近似。  
2）只训练 B、A，原权重重用，显存和计算都省很多。  
3）换任务时只换 adapter（B、A），基座共用，便于多任务部署。

**可选的深**：秩 r 的选取（常用 8/16/32）；和 Adapter、Prefix 的对比。详见 [LoRA.md](./LoRA.md)。

## 一、Transformer 与架构

| 优先级 | 考点 | 答题要点 | 细节 |
|--------|------|----------|----------|
| ⭐ | **Self-Attention 公式** | 公式 \( \text{Attention}(Q,K,V) = \text{softmax}(QK^T/\sqrt{d_k})V \)；除以 √d_k 防梯度变小（见上示例）。 | 为什么除以 √d_k？为什么梯度变小？ |
| ⭐ | **多头注意力** | 作用、如何并行、和单头的区别。 |
| ⭐ | **位置编码** | 绝对 vs 相对；**RoPE** 原理和优势；ALiBi 等。 |
| ⭐ | **LayerNorm vs BatchNorm** | 为什么 Transformer 用 LayerNorm；**Pre-LN vs Post-LN**（如 LLaMA 用 Pre-LN）。 |
| ☆ | **Encoder / Decoder / Encoder-Decoder** | 代表模型（BERT / GPT / T5）及适用场景。 |
| ☆ | **Prefix LM vs Causal LM** | 区别和典型模型。 |
| ⭐ | GQA / MLA(RoPE) | ... | ... |

## 二、预训练与训练流程

| 优先级 | 考点 | 答题要点 |
|--------|------|----------|
| ⭐ | **预训练目标** | 自回归（GPT）、自编码（BERT）、Span 预测等。 |
| ⭐ | **从预训练到上线** | 预训练 → **SFT** → **奖励模型** → **RLHF/DPO** 各步在做什么。 |
| ⭐ | **RLHF 三阶段** | SFT → Reward Model → PPO（或 DPO 等替代）。 |
| ☆ | **SFT 数据** | 指令数据格式、质量要求、规模量级。 |
| ☆ | **DPO 与 RLHF** | DPO 如何避免显式 reward 模型、优缺点。 |

## 三、微调与高效训练

| 优先级 | 考点 | 答题要点 |
|--------|------|----------|
| ⭐ | **全量微调 vs PEFT** | 显存、效果、适用场景。 |
| ⭐ | **LoRA** | 低秩分解、\(W' = W + BA\)、秩 r、适用层；见上示例。→ [LoRA.md](./LoRA.md) |
| ☆ | **QLoRA** | 量化 + LoRA，显存与精度权衡。 |
| ☆ | **Adapter、Prefix Tuning** | 思路对比，和 LoRA 的异同。 |
| ○ | **混合精度** | FP16/BF16、梯度缩放、常见坑。 |

## 四、推理与部署

| 优先级 | 考点 | 答题要点 |
|--------|------|----------|
| ⭐ | **KV Cache** | 原理、为何能加速、显存与序列长度关系。 |

## 复习节奏与准备建议

**第一阶段（必背）**  
Transformer、Attention、位置编码、预训练/SFT/RLHF、LoRA、KV Cache、解码策略、CoT、幻觉。

**第二阶段（重要）**  
RAG、Agent、量化、Flash Attention、DPO、多模态、MoE、评估与安全。

**第三阶段（按岗位补）**  
长上下文、混合精度、更多评估与安全细节。
