# 位置编码有哪些方式？绝对位置编码和相对位置编码的区别是什么？

### 1. 核心定性
本质上，这是打破注意力排列不变性，注入时序或相对距离信息的特征增强机制。

### 2. 具体流程
1. **绝对位置编码**在网络输入层将Token的全局唯一索引映射为特征向量，与词嵌入（Embedding）直接相加。
2. **相对位置编码**放弃全局绝对坐标，在模型深层计算 Attention Score 阶段动态注入两个 Token 间的相对距离。
3. **核心区别**：绝对编码作用于**输入特征**（$X + PE$），相对编码直接干预**注意力路由矩阵**（$QK^T$ 的运算过程）。

### 3. 数学基础
**绝对位置编码 (Sinusoidal)**:
$$PE_{(pos, 2i)} = \sin\Big(\frac{pos}{10000^{2i/d_{model}}}\Big)$$
$$PE_{(pos, 2i+1)} = \cos\Big(\frac{pos}{10000^{2i/d_{model}}}\Big)$$
- $pos$: Token 的绝对位置索引
- $i$: 词特征维度的索引
- $d_{model}$: 模型的隐藏层总维度

**相对位置编码 (以旋转位置编码 RoPE 为例)**:
$$q_m^T k_n = (R_{m,d} q)^T (R_{n,d} k) = q^T R_{n-m,d} k$$
- $q, k$: 无位置信息的 Query 和 Key 向量
- $m, n$: 序列中两个 Token 的真实位置
- $R_{m,d}$: 带有位置 $m$ 信息的 $d$ 维稀疏正交旋转矩阵
- $n-m$: 内积结果天然只与相对距离（$n-m$）有关，完成从绝对坐标到相对距离的数学降维。

### 4. 工程考量
- **Trade-off**: 绝对编码计算零延迟（可预计算）且架构简单，但彻底丧失了长度外推性（Length Extrapolation）；相对编码（如RoPE、ALiBi）赋予了模型超越训练长度的外推潜力，但代价是推断计算量上升，且引入 KV Cache 管理复杂度。
- **致命弱点**: 绝对编码在推理时一旦遭遇长于预训练窗口的序列直接面临 OOD（分布外）崩溃；极端的相对编码（如 ALiBi 线性衰减偏置）在处理强依赖长程绝对坐标逻辑的代码推理任务时，局部注意力坍缩会导致性能显著退化。

### 5. 工业映射
在工业界，绝对位置编码已被早期 BERT 和 GPT-2 淘汰；基于相对距离映射的 **RoPE (旋转位置编码)** 被直接应用于 Llama 3、Qwen 2、Mistral 等顶级开源大模型中，用于应对 128k+ Tokens 甚至更高的高可用极限长上下文场景。
