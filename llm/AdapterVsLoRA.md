# Adapter 和 LoRA 的区别是什么？各自的适用场景？

## 1. 核心定性
本质上，**Adapter** 通过在 Transformer 层间插入新增模块（上投影+下投影+激活）实现参数微调，而 **LoRA** 通过低秩矩阵分解在现有权重上注入更新。Adapter 增加模型深度和延迟，LoRA 保持模型结构不变，两者分别代表**架构扩展**和**权重分解**两种参数高效微调范式。

## 2. 具体流程与结构

**Adapter 结构**:
```
   x 
   ↓
   LayerNorm
   ↓
   W (冻结) → 残差连接
   ↓
   W_down → Activation → W_up → Scale
   ↓
   + (残差)
```
- 在每个 Transformer 层后插入 Adapter 模块
- $W_\text{down} \in \mathbb{R}^{d \times r}$，$W_\text{up} \in \mathbb{R}^{r \times d}$，$r=16-64$
- 引入层归一化后的残差连接和缩放因子
- 训练时只更新 $W_\text{down}$ 和 $W_\text{up}$，冻结原权重 $W$

**LoRA 结构**:
```
   x
   ↓
   W_0 + BA
   ↓
   输出
```
- $B \in \mathbb{R}^{d \times r}$，$A \in \mathbb{R}^{r \times d}$
- 权重矩阵旁路注入低秩更新
- $BA$ 直接加到 $W_0$ 上，无额外组件

**实现对比**:
| 特性 | Adapter | LoRA |
|------|---------|------|
| **插入位置** | 层间新增模块 | 权重矩阵内部 |
| **参数形式** | 两个全连接层 | 低秩矩阵对 |
| **计算流** | 串联 | 并联 |
| **推理影响** | 增加延迟 | 零延迟（可合并） |

## 3. 数学基础

**Adapter 参数更新**:
$$h = W_0 x + s \cdot W_\text{up}(\sigma(W_\text{down} x))$$

其中 $s$ 是缩放因子（通常 0.1-0.3），$\sigma$ 是非线性激活（如 ReLU）。

**LoRA 参数更新**:
$$h = (W_0 + BA)x = W_0 x + B(Ax)$$

**参数数量对比** (以 d=768 为例):
| Method | r=16 | d=768 | 参数数 | 占比 |
|--------|------|-------|--------|------|
| Adapter | 16 | 768 | 2 × 768 × 16 = 24,576 | 4.2% |
| LoRA | 16 | 768 | 2 × 768 × 16 = 24,576 | 4.2% |

**看起来参数相同，但实际**:
- Adapter 增加 24K + LayerNorm + 激活 + 残差处理
- LoRA 纯矩阵乘法，实现更简单
- LoRA 可秩分解优化，实际计算可能更少

**表达能力分析**:
- **Adapter**：由于非线性激活，理论上可表达更复杂的函数
- **LoRA**：线性组合，表达能力较弱，但足够捕获任务适配

**适配位置**:
- **Adapter**：常应用于 FFN 输出，有时也会插入 Attention 层
- **LoRA**：通常只应用于 $W_q$ 和 $W_v$，跳过 $W_k$ 和 $W_o$

**梯度传播**:
- **Adapter**：梯度需通过非线性激活，反向传播更复杂
- **LoRA**：纯线性，梯度计算简单稳定

## 4. 工程考量

**性能对比**:

| 模型 | 方法 | GLUE 平均 | 推理延迟 | 训练时间 |
|------|------|-----------|----------|----------|
| BERT-base | 全量微调 | 82.1 | 1x | 1x |
| BERT-base | Adapter | 81.8 (-0.3) | 1.15x | 0.7x |
| BERT-base | LoRA | 81.5 (-0.6) | 1.0x | 0.8x |
| LLaMA 7B | LoRA | 6.8 | 1.0x | - |
| LLaMA 7B | Adapter | - | 1.2x | - |

**适用场景对比**:

**Adapter 适用**:
1. **多任务适配**：需要为同一模型训练多个适配器时，Adapter 的串联结构互不干扰
2. **小模型微调**：在 BERT-base 等小模型上，Adapter 的非线性有助于提升表达能力
3. **模块化解耦**：需要明确分离不同功能模块时，Adapter 更清晰
4. **需要新架构**：在权重冻结的场景外，需要添加新组件（如门控机制）

**LoRA 适用**:
1. **大模型微调**：LLaMA 65B+，LoRA 可合并权重，推理零开销至关重要
2. **多租户服务**：需要动态加载不同用户适配器，LoRA 的轻量化优势明显
3. **GPU 训练**：LoRA 反向传播更简单，训练速度比 Adapter 快 15-20%
4. **部署受限**：移动端、边缘设备，必须合并权重不能增加延迟

**Trade-off**:

- **表达 vs 简单**: Adapter 有非线性，理论上更强，但实际中 LoRA 足够且更简单
- **内存 vs 速度**: Adapter 推理需要额外内存，LoRA 可合并权重零开销
- **模块化 vs 统一**: Adapter 更模块化，但在现代框架中 LoRA 更统一
- **实现复杂度**: LoRA 仅需两行代码实现，Adapter 需要定义新模块

**致命弱点**:

**Adapter**:
- **延迟不可接受**: 在 65B 模型上，每层加 Adapter 推理延迟增加 20-30%，总延迟 2-3x，无法部署
- **梯度消失**: 深层模型中（48 层），梯度经过多层 Adapter 非线性，可能消失，需特殊初始化
- **层间干扰**: 不同层的 Adapter 可能学重复功能，导致冗余和冲突

**LoRA**:
- **秩瓶颈**: 对于需要全秩更新的任务，低秩假设失效，效果比 Adapter 差 1-2 点
- **稳定性**: 在 $r$ 较大时（>64），优化不稳定，需更小的学习率
- **因果性**: 在自回归生成中，LoRA 合并权重可能破坏因果掩码，需特殊处理

## 5. 工业映射

**HuggingFace PEFT 库**:
- `PEFT.Adaption` 支持 LoRA 和 Adapter，但 95% 用户选择 LoRA
- LoRA 在 GPT-3 微调中占主导地位，Adapter 主要保留用于 BERT 时代任务

**Microsoft LoRA Hub**:
- 在 65B 模型上，LoRA 适配器仅 100MB，可动态加载
- 在推理服务中，每个适配器加载时间 <1s，内存增加 <5%
- Adapter 因延迟问题已基本弃用

**Llama.cpp**:
- 支持 LoRA 权重合并，可在 16GB 内存运行 LLaMA + LoRA
- 不支持 Adapter（结构复杂，无法合并）

**FastChat**:
- 支持多 LoRA 服务，每个请求动态选择适配器
- 因 Adapter 的串联结构无法快速切换，只支持 LoRA

**选择建议**:
- **大模型（>7B）**: 必选 LoRA
- **小模型（<1B）**: 可尝试 Adapter
- **部署重要**: LoRA
- **研究探索**: 都可尝试，但 LoRA 更简单

**结论**: Adapter 是早期方案，LoRA 凭借其简洁性、零推理开销和强大效果，已成为工业标准
