# ALiBi 是什么？它如何实现位置编码？和 RoPE 比有什么优势？

### 1. 核心定性
本质上，ALiBi 是通过在注意力分数加入距离惩罚，实现免微调长文本外推的位置编码。

### 2. 具体流程
1. 取消 Embedding 阶段的任何位置编码注入。
2. 计算 Query 和 Key 的标准点积，得到原始注意力矩阵。
3. 在 Softmax 操作前，将 Token 间的相对距离乘以固定斜率作为负偏置直接加到得分中。

### 3. 数学基础
$Attention(Q, K, V) = \text{softmax}(QK^T + mD)V$

对于给定的 query $i$ 和 key $j$ ($i \ge j$)，其得分偏置公式为：
$S_{i,j} = q_i \cdot k_j - m \cdot (i - j)$

其中：
- $q_i \cdot k_j$: 原始的点积注意力得分
- $m$: 针对不同 Attention Head 预设的固定斜率常数（如 $2^{-1}, 2^{-2} \dots$）
- $i - j$: 当前 Token 与目标 Token 之间的单向相对距离

### 4. 工程考量
与 **RoPE (旋转位置编码)** 相比：
- **核心优势 (Zero-shot 外推)**：在短文本上训练，可直接在长文本上推理。无需像 RoPE 那样依赖位置插值（PI/NTK）微调，且没有三角函数的计算开销，推理速度更快。
- **Trade-off**：牺牲了模型在常规上下文长度下的拟合精度（同等参数下困惑度 PPL 表现略逊于 RoPE），换取了绝对的外推扩展性。
- **致命弱点**：线性增长的距离偏置经过 Softmax 后，会导致远距离 Token 的注意力权重被**指数级衰减压制**。在极长上下文的“大海捞针”（Needle In A Haystack）任务中，极易丢失早期的关键事实。

### 5. 工业映射
在工业界，该机制被直接应用于 **BLOOM** 和 **MPT** 等大规模语言模型中，用于在无需高昂微调成本的前提下，应对长文本扩展的高级推理场景。
