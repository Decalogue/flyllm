# LoRA 原理

### 1. 核心定性
本质上，**LoRA** 是为了解决大模型微调显存爆炸的问题，通过**低秩矩阵分解**旁路实现的参数高效微调（PEFT）算法。

### 2. 具体流程
1. 冻结预训练大模型的全部原有权重矩阵，完全切断其梯度计算链路。
2. 在原有线性层旁路，并联注入降维矩阵 $A$ 和升维矩阵 $B$ 构成的低秩分解模块。
3. 训练时仅更新旁路矩阵，推理时将 $BA$ 预先合并回原权重，实现零额外延迟。

### 3. 数学基础
线性层的正向传播状态转移方程为：
$h = W_0 x + \Delta W x = W_0 x + \frac{\alpha}{r} B A x$

其中：
- $W_0 \in \mathbb{R}^{d \times k}$: 冻结的预训练权重矩阵。
- $B \in \mathbb{R}^{d \times r}$: 升维矩阵，初始化为**全零矩阵**，确保初始状态 $\Delta W = 0$。
- $A \in \mathbb{R}^{r \times k}$: 降维矩阵，采用**高斯分布随机初始化**。
- $r$: 设定的极小低秩大小（$r \ll \min(d, k)$）。
- $\alpha$: 缩放因子（Scaling factor），用于控制微调更新量对原输出的权重影响。

### 4. 工程考量
- **Trade-off**: 牺牲了全参数空间的高维特征重构自由度，换取了训练显存开销的指数级下降及微调产物（Adapter）体积的极小化（通常几十MB）。
- **致命弱点**: 面对跨度极大的领域认知注入（如中英双语基座强行微调为纯代码/数学推理模型）时，低秩更新的本征表达能力会撞墙，导致严重欠拟合。

### 5. 工业映射
在工业界，该机制被直接应用于 **vLLM** / **TensorRT-LLM** 的 Multi-LoRA 动态路由服务中，用于应对云端“单一基座模型 + 海量租户个性化 Adapter”的高并发并发加载与低成本隔离场景。
