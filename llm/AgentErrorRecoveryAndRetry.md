# Agent 错误恢复机制与重试策略设计

## 1. 核心定性

本质上，Agent 错误恢复是一个基于**状态机 + 指数退避**的容错闭环，通过**可观测性分层**区分瞬时故障与致命错误，用**幂等执行**保障最终一致性。

---

## 2. 具体流程

1. **故障分层识别**：将异常分类为网络抖动（瞬时可恢复）、依赖服务降级（需熔断）、逻辑错误（不可重试），通过错误码前缀（如 `NET_`、`SRV_`、`LOGIC_`）快速路由到不同处理分支。

2. **自适应重试决策**：对可恢复错误启动指数退避重试（Exponential Backoff + Jitter），设置**最大重试次数 N** 和**总超时窗口 T**，任一条件触发即进入降级或人工介入流程。

3. **状态快照与恢复**：每次重试前持久化执行上下文（Checkpoint），失败时从最近一致状态回滚，确保幂等性以避免副作用累积。

---

## 3. 数学基础

### 指数退避间隔计算

$$
T_{backoff}(n) = \min(T_{max}, \, T_{base} \cdot 2^n + \text{Jitter})
$$

其中：
- $n$: 当前重试次数（从 0 开始）
- $T_{base}$: 基础退避间隔（通常 100ms ~ 1s）
- $T_{max}$: 最大退避上限（通常 30s ~ 60s）
- $\text{Jitter} \in [0, \, T_{base} \cdot 2^n \cdot 0.1)$: 随机扰动，打散重试风暴

### 重试成功概率模型（假设每次独立）

$$
P_{success}(N) = 1 - (1 - p)^N
$$

其中：
- $p$: 单次请求成功概率
- $N$: 最大重试次数

### 状态机核心结构

```
State: IDLE → RUNNING → [SUCCESS | FAILED → RETRYING → (BACKOFF) → RUNNING]
                    ↓
              CIRCUIT_OPEN → (HALF_OPEN_PROBE) → RUNNING
```

## 4. 工程考量

| 维度 | Trade-off | 致命弱点 |
|------|-----------|----------|
| **重试次数** | 过多→级联放大故障；过少→浪费可用窗口 | 下游已过载时，重试即 DDoS |
| **退避策略** | 固定间隔→易形成" thundering herd"；指数+Jitter→增加延迟 | 长尾任务堆积，耗尽线程池 |
| **幂等设计** | 全局幂等键→存储开销；业务层去重→侵入性强 | 幂等键失效或重复消费导致数据不一致 |
| **熔断机制** | 快速失败→牺牲部分可用性；无脑重试→拖垮依赖 | 半开状态探测流量击穿脆弱节点 |

**核心取舍公式**：
$$
\text{系统可用性} = \frac{\text{成功请求数}}{\text{总请求数} + \text{重试放大系数} \cdot \text{失败重试数}}
$$

## 5. 工业映射

| 组件 | 实现机制 | 应用场景 |
|------|----------|----------|
| **AWS Lambda** | 异步调用自带 2 次指数退避重试 + Dead Letter Queue | 事件驱动函数故障转移 |
| **Kubernetes Job** | `restartPolicy: OnFailure` + `backoffLimit` 控制重试 | 批处理任务容错 |
| **Temporal / Cadence** | 状态机持久化 + 可编程重试策略（最大尝试、超时、非重试错误类型） | 长流程工作流编排 |
| **Resilience4j** | CircuitBreaker（熔断）+ Retry（退避）+ RateLimiter（限流）组合 | 微服务间调用防护 |
| **gRPC** | 内置 `RetryPolicy` 配置 + 状态码白名单（`UNAVAILABLE`, `DEADLINE_EXCEEDED`） | 服务间 RPC 容错 |

> 在 LangChain / LlamaIndex 等 Agent 框架中，错误恢复通常封装在 `run_with_retry()` 或 `Retrying` 装饰器中，支持自定义**可重试异常类型列表**和**回调钩子**（如重试时切换模型、降级到缓存结果）。

---

**一句话总结**：Agent 错误恢复 = **分层熔断**（何时放弃）+ **指数退避**（何时再试）+ **幂等快照**（如何安全重试），三者缺一不可。
