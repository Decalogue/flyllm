# Agent 面试实战

## 一、Agent 基础理论（必考）

1. **AI Agent 和传统 LLM 的区别是什么？有哪些类型？**
   - 考察点：Agent的本质特征、自主性、工具使用能力
   - 答案要点：Agent = LLM + 工具 + 记忆 + 规划 + 执行

2. **什么是 ReAct 框架？它的核心思想是什么？**
   - 考察点：Reasoning + Acting 的循环机制
   - 答案要点：推理-行动-观察的循环，如何设计提示

3. **Agent 的自主性如何体现？如何控制 Agent 的自主性？**
   - 考察点：自主决策、边界控制、安全机制

4. **为什么需要 Agent？LLM 的局限性在哪里？**
   - 考察点：实时信息获取、工具调用、长期记忆、多步推理

---

## 二、ReAct 框架（高频）

5. **ReAct 框架如何实现？提示怎么设计？**
   - 考察点：推理-行动循环的实现、提示模板设计

6. **ReAct 中的推理（Reasoning）和行动（Acting）如何平衡？**
   - 考察点：何时推理、何时行动、如何避免过度推理

7. **如何设计 ReAct 的停止条件？**
    - 考察点：任务完成判断、最大步数限制、错误重试机制

---

## 三、Tool Calling / Function Calling（极高概率）

8. **Tool Calling 如何让模型学会使用工具？工具描述怎么设计？**
    - 考察点：工具描述格式、JSON Schema 设计、训练方法

9. **Function Calling 的格式是什么？JSON Schema 如何定义？**
    - 考察点：OpenAI Function Calling、工具描述规范

10. **Function Call 的多轮对话如何处理？为什么这是最难的部分？**
    - 考察点：状态机设计、流程调度逻辑、三种对话结构（信息补全型、工具链式、混合型）、何时追问何时调用工具、数据生成方法、模型稳定性保证

11. **Function Call 的参数提取错误如何处理？错误处理机制怎么设计？**
    - 考察点：参数验证、类型转换、默认值处理、错误恢复

12. **Function Call 的链式调用如何实现？依赖关系如何处理？**
    - 考察点：工具依赖图、执行顺序、并行执行

13. **Function Call 的容错机制如何设计？调用失败如何处理？重试策略怎么设计？**
    - 考察点：超时处理、重试策略、降级方案

## 四、Memory 机制（极高概率）

## 十、系统设计题（必考）

### 教育类 Agent 系统
**题目**：设计一个教育类 Agent 系统，支持个性化教学、知识问答、作业批改等功能。

**考察点**：
- Multi-Agent 架构设计（教学Agent、问答Agent、批改Agent）
- Memory 机制设计（学生知识图谱、学习历史）
- RAG 系统设计（知识库检索、教材检索）
- 工具调用设计（计算器、绘图工具、代码执行）
- 评估反馈机制

**回答要点**：
1. 系统架构：分层设计、Agent 分工、通信机制
2. 核心功能：个性化推荐、知识检索、智能问答、作业批改
3. 技术选型：LLM、向量数据库、图数据库、工具集成
4. 性能优化：缓存策略、并行处理、延迟优化
5. 质量保障：评估体系、错误处理、安全机制

### Multi-Agent 协作系统
**题目**：设计一个 Multi-Agent 系统，多个 Agent 协作完成复杂任务。

**考察点**：
- Agent 角色定义和分工
- 通信协议和消息传递
- 任务分配和负载均衡
- 冲突解决和协调机制
- 系统监控和调试

**回答要点**：
1. 架构设计：集中式 vs 分布式、Agent 编排
2. 通信机制：消息队列、事件驱动、同步/异步
3. 协调机制：任务分配、资源管理、冲突解决
4. 容错设计：错误恢复、降级方案、监控告警

### Agent 开发工具套件
**题目**：设计一个易用高效的 Agent 开发工具套件，降低 Agent 技术的使用门槛。

**考察点**：
- 框架设计：抽象层次、API 设计、开发体验
- 核心组件：Agent 定义、工具注册、执行引擎
- 质量保障：测试框架、调试工具、监控系统

**回答要点**：
1. 框架架构：核心组件、扩展机制、插件系统
2. 开发体验：配置化、可视化、模板化
3. 质量保障：单元测试、集成测试、性能测试
