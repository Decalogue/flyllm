```markdown
# 项目模板

## 项目X: LLaMA-70B推理服务优化

**Situation (背景)**: 业务需要部署70B聊天模型，要求P99<200ms, QPS>10，但A100(80GB)单卡无法支撑

**Task (任务)**: 作为核心工程师，负责推理性能优化

**Action (行动)**:
1. 诊断: 用Nsight分析，发现KV Cache占显存85%
2. 方案设计: 采用组合策略：GQA+INT8+PagedAttention
3. 实现: 修改modeling_llama.py，集成vLLM的PagedAttention
4. 调优: batch_size=8, 窗口大小=4K, 预热热点prompt

**Result (结果)**:
- 显存: 64GB → 12GB（节省80%）
- QPS: 3 → 18（提升6x）
- P99延迟: 850ms → 150ms（提升5.7x）
- 成本: 8卡A100 → 1卡A100（节省87.5%）

**Technical Highlights**:
- 组合优化的trade-off分析: 精度损失<3%可接受
- 踩坑：INT8量化后精度异常，发现是scale选择问题，改用per-token量化解决
- 创新点：预加载热点System Prompt的KV Cache，首token延迟降低90%
```

**包装技巧**:
1. **量化优先**: 所有效果用数字说话
2. **难点突出**: 强调技术挑战（OOM、精度loss、延迟）
3. **细节深入**: 准备1-2个技术细节，等面试官追问
