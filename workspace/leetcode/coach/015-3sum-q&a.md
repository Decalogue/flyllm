## 📚 题目信息

**题号**: 15
**难度**: Medium
**分类**: 数组与字符串 - 双指针+去重
**预计用时**: 40分钟

**题目描述**:
给定一个整数数组 `nums`，找出所有满足 `i < j < k` 且 `nums[i] + nums[j] + nums[k] == 0` 的三元组，返回所有不重复的三元组。

**示例**:
```python
输入: nums = [-1,0,1,2,-1,-4]
输出: [[-1,-1,2],[-1,0,1]]
```

**关键信息**:
- 需要找出**所有**满足条件的三元组
- **去重**：答案中不能有重复的三元组
- 数组长度最多 3000，需要 O(n²) 的算法

```
排序 + 双指针 + 去重

这是数组与字符串分类中"双指针"技术的经典应用：
- **排序**：O(nlogn) 预处理，为双指针创造条件
```

**问题4**: 题目说"答案中不可以包含重复的三元组"，这是什么意思？用 `nums = [-1,0,1,2,-1,-4]` 举例说明。

```
你的回答：
-----------------------------------------------------
排序后：[-4, -1, -1, 0, 1, 2]

可能的三元组：
- 使用第一个-1（index=1）：(-1, 0, 1) ✓ 有效
- 使用第二个-1（index=2）：(-1, 0, 1) ✗ 重复！

关键点：**值相同但索引不同的组合视为重复**

题目要求的是唯一的数值组合，而不是不同的索引组合。

所以最终答案只能是：[[-1,-1,2], [-1,0,1]]
不能出现重复的[-1,0,1]两次（即使用了不同的-1）

这就是为什么需要去重！
-----------------------------------------------------
```

**问题5**: 为什么要先排序？排序带来了什么好处？如果数组是无序的 `[-1,0,1,2,-1,-4]`，能找到 (-1,0,1) 这个组合吗？

```
你的回答：
-----------------------------------------------------
排序后可以：

1. **使用双指针技巧**：在有序数组上，可以通过左右指针向中间移动来寻找目标值
2. **高效去重**：相同的数会相邻，便于跳过重复元素
3. **剪枝优化**：
   - 如果当前最小的三个数之和都大于0，可以跳出循环
   - 如果当前数和最大的两个数之和都小于0，可以跳过

如果数组是无序的[-1,0,1,2,-1,-4]：
- 虽然能找到(-1,0,1)这个组合
- 但无法高效定位、去重和剪枝
- 暴力解法的时间复杂度O(n³)无法接受

排序的代价是O(nlogn)，但整体优化到O(n²)非常值得！
-----------------------------------------------------
```

**问题6**: 我们固定第一个数 `nums[i]`，然后需要在 `nums[i+1:]` 中找两个数使其和为 `-nums[i]`。在**有序数组**中，如何快速找到这样的两个数？可以用什么技巧？

```
你的回答：
-----------------------------------------------------
在有序数组中使用**双指针技巧**：

1. 初始化左指针left = i+1，右指针right = n-1
2. 计算当前和：sum = nums[left] + nums[right]
3. 如果sum < target，left++（需要和更大）
4. 如果sum > target，right--（需要和更小）
5. 如果sum == target，找到一组解

**为什么不用哈希表？**
- 哈希表需要O(n)额外空间
- 双指针只需要O(1)空间
- 哈希表不方便处理"找出所有组合"的需求
- 双指针在有序数组上更自然、更容易去重
-----------------------------------------------------
```

- 和是 1 < 2，应该移动哪个指针？为什么？

```
你的回答：
-----------------------------------------------------
模拟过程：

candidate = (-2, -1, 2)，和 = -1 ✗
因为和 < target(2)，应该移动**左指针(left)**
原因：当前和太小，需要更大的数。虽然移动右指针也可以让和变大，但向右移动左指针才能系统地探索所有可能的组合。

完整模拟：
[-2, -1, -1, 0, 1, 2]  target=2
↑i  ↑l            ↑r  sum=-1 < 2 → left++

[-2, -1, -1, 0, 1, 2]  target=2
↑i       ↑l        ↑r  sum=1 < 2 → left++

[-2, -1, -1, 0, 1, 2]  target=2
↑i           ↑l    ↑r  sum=3 > 2 → right--

[-2, -1, -1, 0, 1, 2]  target=2
↑i           ↑l↑r      sum=1 < 2 → left++ (left==right，结束)
-----------------------------------------------------
```

**问题8**: 排序后相同的数会相邻，这如何帮助我们**去重**？具体来说：
- 当 `nums[i] == nums[i-1]` 时，为什么要跳过？
- 当找到一组解后，为什么也要跳过 `nums[left]` 和 `nums[right]` 的重复值？

```
你的回答：
-----------------------------------------------------
去重的关键是：相同的数值只会产生相同的三元组，需要跳过

1. **跳过 nums[i] == nums[i-1]**：
   - nums[i]是三元组的第一个元素
   - 如果nums[i] == nums[i-1]，那么从nums[i-1]开始找的所有组合
     一定包含了从nums[i]开始找的组合
   - 为了避免重复，直接跳过

2. **找到解后跳过重复值**：
   - 当找到一组解(nums[i], nums[left], nums[right])后
   - 如果nums[left]重复，移动left跳过（否则会得到重复的三元组）
   - 如果nums[right]重复，移动right跳过
   - 示例：[-2, 0, 0, 0, 2]，找到(-2, 0, 2)后，中间两个0都要跳过

去重策略确保每个唯一的三元组只出现一次！
-----------------------------------------------------
```

### Step 2: 算子提取

> 目标：掌握双指针找两数之和的模板

**三数之和模板框架**：

```python
def threeSum(nums):
    nums.sort()  # Step 1: 排序
    n = len(nums)
    res = []

    for i in range(n - 2):  # Step 2: 固定第一个数
        # 优化1: 剪枝
        if nums[i] + nums[i + 1] + nums[i + 2] > 0:
            break
        if nums[i] + nums[n - 1] + nums[n - 2] < 0:
            continue

        # Step 3: 去重（跳过重复的i）
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        # Step 4: 双指针找两数之和
        left, right = i + 1, n - 1
        target = -nums[i]

        while left < right:
            curr_sum = nums[left] + nums[right]

            if curr_sum < target:
                left += 1  # 和太小，需要更大的数
            elif curr_sum > target:
                right -= 1  # 和太大，需要更小的数
            else:
                # 找到一组解
                res.append([nums[i], nums[left], nums[right]])

                # Step 5: 跳过重复元素（去重）
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1

                # 移动指针继续寻找
                left += 1
                right -= 1

    return res
```

## 💾 学习笔记沉淀

### 三数之和模板

```python
def threeSum(nums):
    """
    三数之和 - 排序 + 双指针
    时间复杂度: O(n²)
    空间复杂度: O(1) 或 O(k)（k是答案数量）
    """
    nums.sort()
    n = len(nums)
    res = []

    for i in range(n - 2):
        # 剪枝优化
        if nums[i] + nums[i + 1] + nums[i + 2] > 0:
            break
        if nums[i] + nums[n - 1] + nums[n - 2] < 0:
            continue

        # 跳过重复
        if i > 0 and nums[i] == nums[i - 1]:
            continue

        left, right = i + 1, n - 1
        target = -nums[i]

        while left < right:
            curr_sum = nums[left] + nums[right]
            if curr_sum < target:
                left += 1
            elif curr_sum > target:
                right -= 1
            else:
                res.append([nums[i], nums[left], nums[right]])
                # 跳过重复
                while left < right and nums[left] == nums[left + 1]:
                    left += 1
                while left < right and nums[right] == nums[right - 1]:
                    right -= 1
                left += 1
                right -= 1

    return res
```

### 双指针找两数之和模板

```python
def twoSumSorted(nums, target):
    """
    在有序数组中找两数之和（返回下标）
    时间复杂度: O(n)
    空间复杂度: O(1)
    """
    left, right = 0, len(nums) - 1

    while left < right:
        curr_sum = nums[left] + nums[right]
        if curr_sum < target:
            left += 1
        elif curr_sum > target:
            right -= 1
        else:
            return [left, right]

    return None
```

### 记忆锚点

- **核心痛点**：
  1. 三处去重的时机和位置容易混淆
  2. 剪枝条件的判断（特别是第二个continue）
  3. 指针移动的时机（找到解后要跳过重复值）

- **通用模式**：
  固定一个数 + 双指针找两数 = 三数问题
  可以推广到：固定k个数 + 双指针 = k+2数问题

- **关键洞察**：
  排序是双指针的前提，也是去重的基础
  去重要覆盖：固定点、左指针、右指针三个位置

- **易错点**：
  1. i去重时判断条件是 `i > 0 and nums[i] == nums[i-1]`（不是`nums[i] == nums[i+1]`）
  2. 找到解后要先去重再移动指针
  3. 剪枝时边界情况要考虑清楚（nums[i] + nums[i+1] + nums[i+2] > 0）
