**题目描述**:
给定一个大小为 `n` 的数组，找到其中的**多数元素**。多数元素是指在数组中出现次数 **大于 ⌊ n/2 ⌋ 的元素**。

你可以假设数组是非空的，并且给定的数组总是存在多数元素。

**示例**：
```python
输入: [3,2,3]
输出: 3

输入: [2,2,1,1,1,2,2]
输出: 2
```

**关键信息**:
- 多数元素出现次数 > n/2
- 数组非空
- **总是存在**多数元素（保证有解）

### Step 2: 算子提取

> 目标：掌握摩尔投票模板

**摩尔投票模板**：

```python
def majority_element(nums):
    # 1. 初始化候选人和票数
    candidate = None
    count = 0

    # 2. 遍历所有选票
    for num in nums:
        # 3. 如果票数为0，更换候选人
        if count == 0:
            candidate = ___________  # 你答对了：candidate = num
            count = 1

        # 4. 如果当前数是候选人，票数+1，否则-1
        count += 1 if num == candidate else -1

    # 5. 返回最后的候选人（题目保证就是答案）
    return candidate
```

**问题7**: 为什么最后返回的candidate一定是多数元素？请用"n/2"这个性质简单证明。

```
你的回答：
-----------------------------------------------------
简化的“石头剪刀布”或“消耗战”思路：

关键原因：多数元素出现次数 > n/2，意味着剩下的n-超过半数 < n/2。

1. 当count ≠ 0时，每个不同的元素配对会互相抵消一票
2. 完全抵消的话，最多能抵消掉n - n/2 - 1个非多数元素
3. 实际多数元素 > n/2，抵消之后一定会剩下来
4. 因此最后的candidate就是多数元素

或者用投票选举的比喻：
- 每个人投一票，多数派的人超过一半
- 不同派别的人相遇会互相抵消对方的票
- 最后剩下的就是多数派的人
-----------------------------------------------------
```

**问题8**: 如果没有"多数元素一定存在"这个保证，或者想验证结果是否正确，还需要做什么？

**问题12**: 请用大白话描述摩尔投票法的工作原理

```
你的回答：
-----------------------------------------------------
投票选举比喻：

想象我们在竞选一个职位，有很多候选人，但是有一个候选人获得了超过一半的选票。那么我们拿到任意一票，可以进行如下操作：

1. **立旗候选人**：现在没有旗帜（count=0），随便揪个人立旗（candidate=投票结果）
2. **同党支持者**：如果是同党的支持者投的票，就给这个人加票（count++）
3. **反对者**：如果是反对党投的票，就把票数-1（count--），相当于这个人和反对党的人"抵消"了
4. **旗帜倒掉**：当票数减到0时，取下旗帜，找下一个投票人重新立旗

因为超过一半的选票都支持一个候选人，所以无论怎么抵消，这个候选人最后一定会有幸存下来并站着不倒的旗。

更形象一点：一群人打架，有51%以上是红方，其他是蓝方。红方和蓝方一对一消耗，红方人更多，所以最后一定还有红方的人存活。
-----------------------------------------------------

**问题13**: 为什么摩尔投票法能保证找到多数元素？

```
你的回答：
-----------------------------------------------------
因为"多数元素出现次数 > n/2"这个特殊保证：

**官方证明思路**：
设多数元素为x，count(x) > n/2，则count(非x) = n - count(x) < n/2。

设不同的元素可以配对抵消：
- 最坏情况下，每个非x都能找到一个x抵消
- 这需要count(非x)对，会消耗掉count(非x)个x
- 但count(x) > count(非x)，所以x剩余 > 0

**直观理解**：
- 56个人投票，31个人投给A，25个人投给B
- A和B的人一对一抵消，A的队伍还有31-25=6个人剩下来
- 最后一个人就能代表整个选举结果，就是候选人A

因此，经过"抵消"过程后，多数元素一定会"存活"下来成为最后的候选人。

## 💾 学习笔记沉淀

### 摩尔投票算法模板

```python
def majority_element(nums):
    """
    摩尔投票算法 - 查找众数
    时间复杂度: O(n)
    空间复杂度: O(1)
    """
    candidate = None
    count = 0

    for num in nums:
        if count == 0:
            candidate = num
            count = 1
        elif num == candidate:
            count += 1
        else:
            count -= 1

    return candidate
```

### 带验证的完整版本

```python
def majority_element_with_validation(nums):
    # 第一阶段：找出候选人
    candidate = None
    count = 0
    for num in nums:
        if count == 0:
            candidate = num
            count = 1
        else:
            count += 1 if num == candidate else -1

    # 第二阶段：验证候选人
    count = 0
    for num in nums:
        if num == candidate:
            count += 1

    if count > len(nums) // 2:
        return candidate
    else:
        return None  # 没有多数元素
```

### 记忆锚点

- **核心痛点**：找到出现次数超过一半的元素，但不能使用额外空间
- **通用模式**：摩尔投票 = 候选人 + 票数差
- **关键洞察**：超过一半的性质保证了候选人一定能"存活"到最后
- **易错点**：题目是否保证多数元素存在（决定是否需要第二遍验证）

### 复杂度分析

- **时间复杂度**：O(n) - 只需遍历数组一次（或两次，如果需要验证）
- **空间复杂度**：O(1) - 只使用常数级别的额外空间（candidate, count两个变量）
