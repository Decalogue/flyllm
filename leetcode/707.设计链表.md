# 707. 设计链表

**标签**：`设计` `链表` `中等`  
**分类**：链表、模拟  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥 中频

---

## 题目描述

设计链表的实现。你可以选择使用单链表或双链表。单链表中的节点应该具有两个属性：`val` 和 `next`。`val` 是当前节点的值，`next` 是指向下一个节点的指针/引用。

如果要使用双向链表，则还需要一个属性 `prev` 以指示链表中的上一个节点。假设链表中的所有节点下标从 **0** 开始。

实现 `MyLinkedList` 类：

- `MyLinkedList()` 初始化 `MyLinkedList` 对象。
- `int get(int index)` 获取链表中下标 `index` 的节点的值。如果下标无效，返回 `-1` 。
- `void addAtHead(int val)` 将一个值为 `val` 的节点插入到链表中第一个元素之前。在插入完成后，新节点会成为链表的第一个节点。
- `void addAtTail(int val)` 将一个值为 `val` 的节点追加到链表中最后一个元素之后。
- `void addAtIndex(int index, int val)` 将一个值为 `val` 的节点插入到链表中下标为 `index` 的节点之前。如果 `index` 等于链表的长度，那么该节点会被追加到链表的末尾。如果 `index` 比长度更大，该节点将 **不会插入** 到链表中。如果 `index` 小于 `0`，则在头部插入节点。
- `void deleteAtIndex(int index)` 如果下标有效，则删除链表中下标为 `index` 的节点。

### 示例

```
输入：
["MyLinkedList", "addAtHead", "addAtTail", "addAtIndex", "get", "deleteAtIndex", "get"]
[[], [1], [3], [1, 2], [1], [1], [1]]

输出：
[null, null, null, null, 2, null, 3]

解释：
MyLinkedList myLinkedList = new MyLinkedList();
myLinkedList.addAtHead(1);
myLinkedList.addAtTail(3);
myLinkedList.addAtIndex(1, 2);   // 链表变为 1 -> 2 -> 3
myLinkedList.get(1);             // 返回 2
myLinkedList.deleteAtIndex(1);   // 链表变为 1 -> 3
myLinkedList.get(1);             // 返回 3
```

### 提示

- `1 <= val <= 1000`
- 最多调用 `2000` 次 `get`、`addAtHead`、`addAtTail`、`addAtIndex` 和 `deleteAtIndex`

## 📋 面试要点速查（知识卡片）

### 核心思路

**虚拟头节点（哨兵）+ `size`**：`dummy.next` 指向真实链表头；所有「在第 `index` 个节点前插入 / 删除第 `index` 个节点」都统一成**找前驱** `pre`，再改 `pre.next`。用 `size` 在 O(1) 判断 `addAtTail` 与非法下标，避免每次遍历求长度。

**关键词**：`dummy 头` `前驱指针` `size 维护` `index 语义`

### 复杂度速记

- **各操作**：`get` / `addAtIndex` / `deleteAtIndex` 需走到前驱或当前节点，**时间 O(n)**；`addAtHead` / `addAtTail` 若复用 `addAtIndex` 同为 O(n)（本实现）；**额外空间 O(1)**（节点本身不计入额外空间）。

### 记忆口诀

「dummy 占坑在头前，前驱走 index 步；插入删除只动 next，size 跟着加减。」

---

## 解题思路

### 核心思想

- **虚拟头**：真实数据从 `dummy.next` 开始，这样「在位置 0 插入/删除」的前驱永远是 `dummy`，与「在中间操作」代码一致。
- **下标约定**：题目规定第 `0` 个节点是第一个数据节点；`addAtIndex(index, val)` 表示插在**下标为 `index` 的节点之前**；`index == size` 等价于尾插。
- **`addAtIndex` 的负数**：`index < 0` 时按题意视为在头部插入，可先规范为 `index = 0` 再走统一逻辑。

### 算法步骤（单链表 + dummy）

1. 维护 `dummy`、`size`。
2. **定位前驱**：要在下标 `index` 处插入，前驱从 `dummy` 出发走 `index` 步（或封装为 `_get(index - 1)`，并令 **下标 `-1` 表示 `dummy` 本身**）。
3. **插入**：`pre.next = ListNode(val, pre.next)`，`size += 1`。
4. **删除**：校验 `0 <= index < size`，前驱同插入，`pre.next = pre.next.next`，`size -= 1`。
5. **`get`**：校验下标后从第一个数据节点走 `index` 步取 `val`。

### 为什么选择这个方法？

- **优势**：边界少、头插/删与中间同一套「找前驱」；面试写双链表也可，但单链表 + dummy 通常足够。
- **替代方案**：双链表 + 首尾指针可做到部分操作均摊更优，但实现与指针维护更重。

## 代码实现

### 单链表 + 虚拟头（推荐）

```python
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next


class MyLinkedList:
    def __init__(self):
        self.dummy = ListNode()
        self.size = 0

    def _get(self, index: int) -> ListNode | None:
        """
        返回「下标为 index 的节点」。
        约定：index == -1 表示 dummy（即第 0 个节点的前驱）。
        index >= 0 时，从 dummy 走 index+1 步到达该节点。
        """
        if index < -1:
            return None
        cur = self.dummy
        for _ in range(index + 1):
            if cur is None:
                return None
            cur = cur.next
        return cur

    def get(self, index: int) -> int:
        if index < 0 or index >= self.size:
            return -1
        node = self._get(index)
        return node.val if node else -1

    def addAtHead(self, val: int) -> None:
        self.addAtIndex(0, val)

    def addAtTail(self, val: int) -> None:
        self.addAtIndex(self.size, val)

    def addAtIndex(self, index: int, val: int) -> None:
        if index > self.size:
            return
        if index < 0:
            index = 0
        pre = self._get(index - 1)
        if pre is None:
            return
        pre.next = ListNode(val, pre.next)
        self.size += 1

    def deleteAtIndex(self, index: int) -> None:
        if index < 0 or index >= self.size:
            return
        pre = self._get(index - 1)
        if pre and pre.next:
            pre.next = pre.next.next
            self.size -= 1
```

**代码要点**：

- ✅ `_get` 必须允许 **`index == -1` 返回 `dummy`**，否则 `addAtIndex(0)` / `deleteAtIndex(0)` 的前驱为 `None`，头插与删头会静默失败。
- ✅ `addAtIndex` 先处理 `index > self.size` 与 `index < 0`，再统一找 `pre = _get(index - 1)`。
- ✅ `addAtTail` 用 `addAtIndex(self.size, val)` 符合「`index` 等于长度则追加」的题意。

---

## ⚠️ 常见陷阱（面试易错点）

1. **`_get` 把 `index < 0` 一律当成非法**
   - ❌ **错误做法**：`if index < 0: return None`，再用 `pre = _get(index - 1)` 处理插入下标 `0`。
   - ✅ **正确做法**：把「前驱」统一成 `_get(index - 1)` 时，**`-1` 对应 `dummy`**；或不用 `_get`，显式 `pre = dummy` 循环 `index` 次。
   - 💡 **原因**：下标 `0` 的前驱是虚拟头，不是「不存在」。

2. **`addAtIndex` 边界**
   - `index > size` 不插入；`index == size` 尾插；`index < 0` 头插（题意）。

3. **`delete` / `get` 与 `size` 不同步**
   - 每次插入删除必须维护 `size`，否则 `get` 与边界判断会错。

4. **空链表**
   - `dummy.next` 可为 `None`，删除/查找前仍要先判断 `index` 与 `size`。

## 复杂度分析

| 操作 | 时间复杂度 | 说明 |
|------|------------|------|
| `get` | O(n) | 最坏走到尾 |
| `addAtHead` / `addAtTail` / `addAtIndex` | O(n) | 本实现复用 `addAtIndex`，找前驱线性 |
| `deleteAtIndex` | O(n) | 找前驱线性 |

**空间**：除链表节点外 **O(1)**。
