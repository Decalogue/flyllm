# 312. 戳气球

**标签**：`数组` `动态规划` `困难`  
**分类**：区间 DP、逆向思维  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 中频

---

## 题目描述

有 `n` 个气球，编号 `0` 到 `n-1`，每个气球上标有一个数字，这些数字存在数组 `nums` 中。

现在要求你戳破所有的气球。戳破第 `i` 个气球，你可以获得 `nums[left] * nums[i] * nums[right]` 个硬币。这里的 `left` 和 `right` 表示和 `i` 相邻的两个**还没有被戳爆**的气球的下标。当 `left` 或 `right` 不存在时（例如在数组边界），按 `1` 处理。

求所能获得的硬币的**最大总数**。

### 示例 1

```
输入：nums = [3,1,5,8]
输出：167
解释：
nums = [3,1,5,8] --> [3,5,8] --> [3,8] --> [8] --> []
coins =  3*1*5    +   3*5*8   +  1*3*8  + 1*8*1 = 15 + 120 + 24 + 8 = 167
```

### 示例 2

```
输入：nums = [1,5]
输出：10
```

### 提示

- `n == nums.length`
- `1 <= n <= 300`
- `0 <= nums[i] <= 100`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：若按「先戳谁」正向枚举，左右邻居随戳法变化，状态难表示；改为 **区间 DP**，令 `dp[i][j]` 表示**开区间** `(i,j)` 内气球全部戳完的最大得分，并枚举 **区间内最后一个被戳的气球** `k`，则 `k` 被戳时邻居恰为 `arr[i]` 与 `arr[j]`，左右子区间独立。

**关键词**：`区间 DP` `最后戳谁` `边界虚拟 1` `开区间`

### 复杂度速记

- 时间 **O(n³)**：区间 O(n²)，枚举断点 O(n)。  
- 空间 **O(n²)**：`dp` 表；`n≤300` 可过。

### 记忆口诀

「两端补 1，开区间 DP；最后戳 k，得分 `左段 + 中乘积 + 右段`。」

---

## 解题思路

### 为什么正向难做

每戳一个气球，数组收缩，剩余气球的**下标关系**与**相邻关系**都变，若用「当前还剩哪些气球」做 bitmask，**O(2ⁿ)**，`n` 到 300 不可行。

### 逆向：「最后一个戳谁」

想象最终只剩虚拟边界，中间某气球 `k` **最后被戳**。此时它左右相邻的未戳气球就是扩展数组里 `k` 左右两侧**第一个还没被戳的**位置——在「`k` 是 `(i,j)` 开区间内最后一个被戳」的设定下，这两个邻居就是 **`arr[i]`** 与 **`arr[j]`**（`i < k < j`）。

戳 `k` 的得分：**`arr[i] * arr[k] * arr[j]`**。  
开区间 `(i,k)` 与 `(k,j)` 内的气球互不干扰，总分：

\[
dp[i][j] = \max_{i<k<j} \big( dp[i][k] + arr[i]\cdot arr[k]\cdot arr[j] + dp[k][j] \big)
\]

### 扩展数组

在 `nums` 两端各加一个 `1`，得到 `arr = [1] + nums + [1]`，长度为 `n+2`。  
`dp[i][j]` 表示：**只考虑**下标严格在 `i` 与 `j` 之间的气球全部戳光的最大得分；**不包含**端点 `i`、`j`（端点视为「虚拟气球」，永不戳）。

- 若 `j - i <= 1`，开区间内无气球，`dp[i][j] = 0`。  
- 答案：**`dp[0][n+1]`**（即原数组整段夹在两个虚拟 `1` 之间）。

### 填表顺序

按**区间长度**从小到大：先算短区间，再算长区间，保证 `dp[i][k]`、`dp[k][j]` 已求出。

## 代码实现

### 标准写法（二维 DP）

```python
class Solution:
    def maxCoins(self, nums: list[int]) -> int:
        arr = [1] + nums + [1]
        n = len(arr)
        # dp[i][j]：开区间 (i,j) 内全部戳完的最大得分
        dp = [[0] * n for _ in range(n)]
        for length in range(2, n):
            for i in range(n - length):
                j = i + length
                best = 0
                for k in range(i + 1, j):
                    best = max(
                        best,
                        dp[i][k] + arr[i] * arr[k] * arr[j] + dp[k][j],
                    )
                dp[i][j] = best
        return dp[0][n - 1]
```

### 实现细节

- **`length`**：即 `j - i`，从 `2` 开始表示开区间内至少有一个气球可戳（`k` 可取 `i+1`）。  
- **循环上界**：`i` 从 `0` 到 `n - length - 1`，保证 `j = i + length < n`。

---

## ⚠️ 常见陷阱

1. **闭区间写错**  
   - 状态应是**开区间** `(i,j)`；若写成闭区间，转移式与边界要整套改写，易混。

2. **忘记两端补 1**  
   - 题目规定边界外按 `1`，必须显式加入 `arr`，否则最左/最右气球得分算错。

3. **枚举顺序**  
   - 必须先算小区间再算大区间；若随意 `i,j` 双层循环，可能用到未初始化的 `dp`。

4. **与「戳气球顺序模拟」混淆**  
   - 代码里**不**模拟真实戳序列表；`k` 表示「在 `(i,j)` 里最后一个被戳的」，是**计数**意义上的最后，与真实操作顺序一一对应（可证明最优子结构成立）。

## 复杂度分析

- **时间复杂度**：O(n³)，`n` 为扩展后长度，即原长 + 2，量级仍为 O(N³)，`N = len(nums)`。  
- **空间复杂度**：O(n²) 存 `dp`；理论可滚动优化，但本题无必要。

## 总结

312 是**区间 DP** 的代表题：**虚拟边界** + **开区间状态** + **枚举区间内最后一个被操作的元素**，把动态变化的邻居固定为 `arr[i]` 与 `arr[j]`，从而写出清晰转移。面试先讲清「为什么正向难、为什么想最后戳谁」，再写转移与边界即可。
