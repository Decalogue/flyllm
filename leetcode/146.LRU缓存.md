# 146. LRU 缓存

**标签**：`设计` `哈希表` `链表` `双向链表` `中等`  
**分类**：数据结构设计、哈希表、链表  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

## 题目描述

请你设计并实现一个满足 **LRU（最近最少使用）缓存** 约束的数据结构。

实现 `LRUCache` 类：

- `LRUCache(int capacity)`：以正整数 `capacity` 作为容量初始化 LRU 缓存。
- `int get(int key)`：若关键字 `key` 存在于缓存中，返回其值；否则返回 `-1`。
- `void put(int key, int value)`：若 `key` 已存在则更新其值；否则插入。若插入后关键字数量超过 `capacity`，则**逐出最久未使用的关键字**。

`get` 和 `put` 必须以 **O(1)** 平均时间复杂度运行。

### 示例

```
输入
["LRUCache", "put", "put", "get", "put", "get", "put", "get", "get", "get"]
[[2], [1, 1], [2, 2], [1], [3, 3], [2], [4, 4], [1], [3], [4]]

输出
[null, null, null, 1, null, -1, null, -1, 3, 4]

解释
LRUCache lRUCache = new LRUCache(2);
lRUCache.put(1, 1); // 缓存是 {1=1}
lRUCache.put(2, 2); // 缓存是 {1=1, 2=2}
lRUCache.get(1);    // 返回 1，且 1 变为最近使用
lRUCache.put(3, 3); // 逐出 2，缓存是 {1=1, 3=3}
lRUCache.get(2);    // 返回 -1 (未找到)
lRUCache.put(4, 4); // 逐出 1，缓存是 {3=3, 4=4}
lRUCache.get(1);    // 返回 -1 (未找到)
lRUCache.get(3);    // 返回 3
lRUCache.get(4);    // 返回 4
```

### 提示

- `1 <= capacity <= 3000`
- `0 <= key <= 10^4`
- `0 <= value <= 10^5`

## 📋 面试要点速查（知识卡片）

### 核心思路

**哈希表 + 双向链表**：用哈希表实现 key → 节点 的 O(1) 查找；用双向链表维护「使用顺序」，头为最近使用、尾为最久未使用。get 时若存在则把该节点移到头部；put 时若存在则更新并移到头部，若不存在则插到头部，若超容则删掉尾部节点。

**关键词**：`哈希表` `双向链表` `头最近尾最久` `O(1) 查找与移动`

### 复杂度速记

- **get/put**：O(1)——哈希查节点、链表删/插/移到头均为 O(1)。
- **空间**：O(capacity)——哈希 + 链表节点数不超过容量。

### 记忆口诀

「哈希管查找，链表管顺序；头新尾旧，访问就挪头，满了删尾。」

---

## 解题思路

### 核心思想

- **需求**：O(1) 根据 key 取值/更新；O(1) 知道「谁最久未用」并在满时 O(1) 删掉。
- **哈希表**：key → value 或 key → 链表节点，实现 O(1) 查找。
- **顺序**：需要一种结构，支持「把某个元素标成最近使用」「删掉最久未使用的」。  
  - 数组/单链表：删除中间节点或移到一端不是 O(1)。  
  - **双向链表**：已知节点引用时，从中间删掉、接到头部都是 O(1)。因此用**双向链表**维护使用顺序，头 = 最近使用，尾 = 最久未使用；再用**哈希表**存 key → 节点，这样 get/put 都能立刻找到节点并完成「移到头」或「删尾」。

### 数据结构约定

- **双向链表**：`head`（哑节点）→ 最近使用 → … → 最久未使用 → `tail`（哑节点）。新用到的节点插到 `head.next`，逐出时删 `tail.prev`。
- **哈希表**：`cache: key -> DLinkedNode`，节点存 `key, value, prev, next`。存 key 是为了删尾时能从节点反查 key，从而从哈希表里删掉。

### 算法步骤

**get(key)**  
1. 若 key 不在 `cache`，返回 -1。  
2. 否则取节点，将其从链表中移除再插入到头部（标记为最近使用），返回节点值。

**put(key, value)**  
1. 若 key 在 `cache`：更新节点值，并将该节点移到头部。  
2. 若 key 不在：新建节点并插到头部，加入 `cache`；若 `len(cache) > capacity`，删掉 `tail.prev` 对应节点，并从 `cache` 中移除该 key。

**辅助**：`_remove_node(node)`（从链表中摘掉）、`_add_to_head(node)`（插到 head 后），保证「移到头」= 先 remove 再 add_to_head。

## 代码实现

### 写法一：哈希表 + 双向链表（推荐，面试通用）

```python
class DLinkedNode:
    def __init__(self, key: int = 0, value: int = 0):
        self.key = key
        self.value = value
        self.prev = None
        self.next = None


class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.cache = {}  # key -> DLinkedNode
        self.head = DLinkedNode()
        self.tail = DLinkedNode()
        self.head.next = self.tail
        self.tail.prev = self.head

    def get(self, key: int) -> int:
        if key not in self.cache:
            return -1
        node = self.cache[key]
        self._remove_node(node)
        self._add_to_head(node)
        return node.value

    def put(self, key: int, value: int) -> None:
        if key in self.cache:
            node = self.cache[key]
            node.value = value
            self._remove_node(node)
            self._add_to_head(node)
            return
        node = DLinkedNode(key, value)
        self.cache[key] = node
        self._add_to_head(node)
        if len(self.cache) > self.capacity:
            lru = self.tail.prev
            self._remove_node(lru)
            del self.cache[lru.key]

    def _remove_node(self, node: DLinkedNode) -> None:
        node.prev.next = node.next
        node.next.prev = node.prev

    def _add_to_head(self, node: DLinkedNode) -> None:
        node.next = self.head.next
        node.prev = self.head
        self.head.next.prev = node
        self.head.next = node
```

**代码要点**：

- ✅ 双向链表带**哑节点** `head`、`tail`，避免判空，删头/删尾写法统一。
- ✅ 节点必须存 **key**：逐出时通过 `lru.key` 从 `cache` 里删除。
- ✅ 「最近使用」= 插到 `head` 后面；逐出 = 删 `tail.prev`。
- ✅ `_remove_node` 只改指针，不删 `cache`；真正逐出时再 `del self.cache[lru.key]`。

### 写法二：OrderedDict（Python 简洁写法）

Python 的 `collections.OrderedDict` 保持插入顺序，且 `move_to_end(key)` 可把某 key 移到最后。把「最后」视为最近使用，则逐出时删第一个即可（`popitem(last=False)`）。

```python
from collections import OrderedDict

class LRUCache:
    def __init__(self, capacity: int):
        self.capacity = capacity
        self.od = OrderedDict()

    def get(self, key: int) -> int:
        if key not in self.od:
            return -1
        self.od.move_to_end(key)  # 移到最后表示最近使用
        return self.od[key]

    def put(self, key: int, value: int) -> None:
        if key in self.od:
            self.od.move_to_end(key)
        self.od[key] = value
        if len(self.od) > self.capacity:
            self.od.popitem(last=False)  # 删最前面的（最久未用）
```

**注意**：面试时先说「哈希+双向链表」思路并手写，若允许用语言特性可再提 OrderedDict。

## 复杂度分析

| 操作     | 时间复杂度 | 说明 |
|----------|-------------|------|
| get(key) | O(1)        | 哈希查找 + 双向链表摘节点 + 插到头部，均为 O(1)。 |
| put(key, value) | O(1) | 查找/更新/移头/插头/删尾，均为 O(1)。 |
| 空间     | O(capacity) | 哈希表 + 链表节点数不超过 capacity。 |

---

## 易错点

- **节点要存 key**：逐出尾部节点时，需要从节点反查 key 才能从 `cache` 里删掉，否则会漏删。
- **先删再插**：「移到头部」= 先从当前位置 `_remove_node`，再 `_add_to_head`，顺序不能反；否则链表会乱。
- **put 时 key 已存在**：要先更新 value，再移到头，并直接 return，不要重复插入。
- **容量为 0**：题目保证 `capacity >= 1`，一般可不特判；若考虑健壮性，put 时若 capacity 为 0 可不插入。
