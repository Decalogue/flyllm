# 560. 和为 K 的子数组

**标签**：`数组` `哈希表` `前缀和` `中等`  
**分类**：数组与字符串（Hot 100）  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你一个整数数组 `nums` 和一个整数 `k`，请你统计并返回该数组中**连续子数组**的个数，这些子数组的**元素之和**恰好为 `k`。

### 示例 1

```
输入：nums = [1,1,1], k = 2
输出：2
解释：子数组 [1,1] 与 [1,1]（不同区间）共 2 个。
```

### 示例 2

```
输入：nums = [1,2,3], k = 3
输出：2
解释：[1,2] 与 [3]
```

### 提示

- `1 <= nums.length <= 2 * 10^4`
- `-1000 <= nums[i] <= 1000`
- `-10^7 <= k <= 10^7`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：前缀和 `s` 每走一步更新；以当前结尾、和为 `k` 的子数组个数 = 历史前缀和中 **`s - k` 出现次数**；用哈希表统计前缀和频次。

**关键词**：`前缀和` `s - k` `哈希计数` `cnt[0]=1`

### 复杂度速记

- **时间**：O(n)；**空间**：O(n) 哈希表。

### 记忆口诀

「前缀一路加，查表里有没有 s-k；有几次就能接几段。」

---

## 解题思路

### 核心思想

设前缀和 `S[i] = nums[0] + ... + nums[i]`（下标从 0 起）。子数组 `[l..r]` 的和为 `k` 当且仅当：

`S[r] - S[l-1] = k`  ⟺  `S[l-1] = S[r] - k`。

扫描时维护当前前缀和 `s`，在**已出现过的前缀和**里查 `s - k` 出现了多少次，每次累加到答案。等价于：**以当前位置为右端点、和为 k 的子数组个数**。

### 为何初始化 `cnt[0] = 1`？

前缀和为 `0` 出现 1 次，表示「空前缀」。当某个前缀和 `s` 本身等于 `k` 时，需要 `cnt[s - k] = cnt[0]` 计 1，对应子数组 `[0..i]`。

### 算法步骤

1. `cnt = defaultdict(int)`，`cnt[0] = 1`，`s = 0`，`ans = 0`。  
2. 对每个 `x`：`s += x`；`ans += cnt[s - k]`；`cnt[s] += 1`。  
3. 返回 `ans`。

**说明**：数组元素可为负，前缀和**不单调**，不能用双指针缩窗口替代哈希（与全正数滑窗不同）。

---

## 代码实现

```python
from collections import defaultdict


class Solution:
    def subarraySum(self, nums: list[int], k: int) -> int:
        cnt: defaultdict[int, int] = defaultdict(int)
        cnt[0] = 1
        s = 0
        ans = 0
        for x in nums:
            s += x
            ans += cnt[s - k]
            cnt[s] += 1
        return ans
```

---

## ⚠️ 常见陷阱

1. **忘记 `cnt[0] = 1`**  
   - 会漏掉从**下标 0** 开始、和为 `k` 的子数组。

2. **以为数组全正就能滑窗**  
   - 本题有**负数**，前缀和可能回退，双指针 O(n) 滑窗不适用。

3. **与「和 ≥ k」混淆**  
   - 本题要**恰好**等于 `k`，计数逻辑是 `s - k`，不是窗口内调整。

---

## 复杂度分析

- **时间复杂度**：O(n)，每个元素 O(1) 更新哈希。  
- **空间复杂度**：O(n)，不同前缀和个数最坏 O(n)。

和为 K 的子数组是**前缀和 + 哈希频次**的经典题：`ans += cnt[s - k]` 一行点破；务必初始化 `cnt[0]=1`，并牢记**有负数时不能用「全正滑窗」**替代。
