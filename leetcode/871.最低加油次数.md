# 871. 最低加油次数（Minimum Number of Refueling Stops）

**标签**：`贪心` `堆（优先队列）` `动态规划` `困难`  
**分类**：贪心、多维 DP  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 中频

---

## 题目描述

汽车从位置 `0` 出发，目的地为目标位置 `target`；油箱初始油量为 `startFuel`（单位：升）。

有一条从 `0` 到 `target` 的公路，途中有若干加油站。第 `i` 个加油站位于 `stations[i][0]`，可加 `stations[i][1]` 升油（假设每个站只能加一次）。

汽车每消耗 **1 升** 油可以行驶 **1** 单位距离；途中可以随时加油（加满到「拿走的量」为止，本题等价于可加多少拿多少）。

返回从 `0` 到达 `target` 所需的**最少加油次数**；若无法到达，返回 **`-1`**。

### 示例 1

```
输入：target = 100, startFuel = 10, stations = [[10,60],[20,30],[30,30],[60,40]]
输出：2
解释：先到 10（耗 10 油），在 10 加油 60，最远可到 10+60=70；再到 60（耗 50 油），
      在 60 加油 40 后即可到达 100。共加油 2 次。（贪心实现中会在「能到 70」时
      从已路过站中选油量最大者扩展「最远可达」，统计结果同为 2。）
```

### 示例 2

```
输入：target = 100, startFuel = 1, stations = [[10,100]]
输出：-1
解释：到不了 10，无法加油。
```

### 示例 3

```
输入：target = 1, startFuel = 1, stations = []
输出：0
解释：初始油量已够直达目标。
```

### 提示（常见约束）

- `1 <= target, startFuel <= 10^9`
- `0 <= stations.length <= 500`
- `0 <= position_i <= target`
- `1 <= fuel_i < 10^9`（油量极大时用整数）
- 题目保证 `stations` 按位置单调给出（非降序）

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：把「当前能到达的最远里程」记为 `cur`；每当 `cur < target`，在**已经路过的**加油站里，贪心选择**可加油量最大**的那个加一次油（用最大堆维护），直到能到 `target` 或无路可加。

**关键词**：`延后决策` `交换论证` `最大堆` `可达性`

### 复杂度速记

- **贪心 + 堆**：时间 **O(n log n)**（排序 + 每个站最多入堆一次），空间 **O(n)**。  
- **DP**：时间 **O(n²)**，空间 **O(n)**，便于和背包/线性 DP 类比。

### 记忆口诀

「够不到目标就从路过的站里，先拿能加最多的那一站。」

---

## 解题思路

### 核心思想（贪心）

- **状态含义**：用 `cur` 表示：在已选定的若干次加油后，从起点 **0** 出发**最多能开到**的里程（等价于油箱+已加油量支撑的最远连续行驶距离，在「每英里耗 1 升」模型下与「剩余可走英里数」可统一成「最远可达坐标」）。
- **何时加油**：当 `cur < target` 时，必须再加油一次。
- **加哪一站**：所有满足 `position <= cur` 的站都已**路过**，任选其一加油都不会改变「路过」的事实；要使之后尽可能远，应选 **fuel 最大** 的站（交换论证：若最优解在某步选了较小的油而放了较大的油在后面，可通过交换得到不更差的解）。

### 算法步骤（堆）

1. 将 `stations` 按位置排序（题目多已有序，排序保险）。  
2. 小根堆存**负的** `fuel`，实现**大根堆**；指针 `i` 扫描加油站。  
3. `cur = startFuel`，`ans = 0`。  
4. 当 `cur < target`：  
   - 将所有 `stations[i][0] <= cur` 的站的 `fuel` 入堆，`i` 前移；  
   - 若堆空，说明无法再继续扩展，返回 `-1`；  
   - 弹出最大 `fuel` 加到 `cur`，`ans += 1`。  
5. 循环结束返回 `ans`（若初始 `startFuel >= target`，循环不进入，返回 `0`）。

### 方法二：DP

- **状态**：`dp[k]` 表示恰好加油 `k` 次时，从起点出发能到达的**最远里程**（或「在最多 k 次加油下」的最大可达，写法等价需注意初始化）。  
- **转移**：对每个加油站 `(pos, add)`，倒序 `k`：若 `dp[k-1] >= pos`，则 `dp[k] = max(dp[k], dp[k-1] + add)`。  
- **答案**：最小的 `k` 使得 `dp[k] >= target`。

DP 适合说明与「分段资源消耗」类问题的联系；数据范围 `n <= 500` 时 **O(n²)** 可过。

### 为什么贪心正确？

直觉：在必须多走一步才能继续前进时，所有已可达站点里多拿一升油对「延伸里程」的贡献就是那一升的数值，故取最大 `fuel` 最优。形式化可用**交换论证**：若某最优序列在某一步未取当前可达中的最大油，则把该步与「取最大油的那一步」交换，可达终点不会变差。

## 代码实现

### 方法一：贪心 + 堆（推荐）

```python
import heapq


class Solution:
    def minRefuelStops(
        self, target: int, startFuel: int, stations: list[list[int]]
    ) -> int:
        stations.sort()
        pq: list[int] = []  # 存 -fuel，模拟最大堆
        i, n = 0, len(stations)
        cur = startFuel  # 当前能到达的最远位置（从 0 起算）
        ans = 0

        while cur < target:
            while i < n and stations[i][0] <= cur:
                heapq.heappush(pq, -stations[i][1])
                i += 1
            if not pq:
                return -1
            cur += -heapq.heappop(pq)
            ans += 1

        return ans
```

**代码要点**：

- ✅ `cur` 不是「剩余油量」而是**最远可达坐标**，与「每英里 1 升」的加法模型一致。  
- ✅ 只有 `position <= cur` 的站才能入堆，表示已经路过才能加油。  
- ✅ 初始 `startFuel >= target` 时 `while` 不执行，直接返回 `0`。

### 方法二：DP（O(n²)）

```python
class Solution:
    def minRefuelStops(
        self, target: int, startFuel: int, stations: list[list[int]]
    ) -> int:
        n = len(stations)
        # dp[k]：恰好加油 k 次时的最远可达距离
        dp = [0] * (n + 1)
        dp[0] = startFuel
        for pos, add in stations:
            for k in range(n, 0, -1):
                if dp[k - 1] >= pos:
                    dp[k] = max(dp[k], dp[k - 1] + add)

        for k in range(n + 1):
            if dp[k] >= target:
                return k
        return -1
```

**代码要点**：

- ✅ `k` 倒序更新，避免同一站同一轮被用多次（01 背包语义）。  
- ✅ `dp[0] = startFuel` 表示不加站时的可达距离。

## 执行过程示例

`target = 100, startFuel = 10, stations = [[10,60],[20,30],[30,30],[60,40]]`

1. `cur = 10`：入堆 `60`，加油后 `cur = 70`，`ans = 1`。  
2. `cur = 70`：`30`、`30`、`40` 对应位置均 `<= 70`，依次入堆；弹出最大 `40` → `cur = 110 >= 100`，`ans = 2`，结束。

（不同实现入堆顺序可能先弹出 `30` 再弹出 `40`，总数仍为 **2**。）

## ⚠️ 常见陷阱

1. **把 `cur` 当成「剩余油」**  
   - ❌ 与「当前坐标 + 剩余油量」混用易错。  
   - ✅ 统一为「从 0 出发在不考虑未来油站时，**最远能开到哪**」，每次加油把 `fuel` **加到** `cur` 上。

2. **堆用错顺序**  
   - ❌ 弹出最小 `fuel`。  
   - ✅ 要**最大**可加油量；Python 用 `heapq` 时存负数。

3. **边界**  
   - `stations` 为空且 `startFuel < target`：第一次扩展堆为空 → 返回 `-1`。  
   - `startFuel >= target`：返回 `0`，不要先入堆再算。

## 总结

这是一道经典的**贪心 + 优先队列**题，核心是：

1. **延后决策**：只有「够不到目标」时才被迫加油。  
2. **局部最优**：在已路过的站中选 **fuel** 最大的。  
3. **实现**：维护「最远可达 `cur`」，用最大堆选油；或用 DP 二次确认正确性。

**面试建议**：先说 O(n²) DP 展示思路，再优化到 heap O(n log n)；能口述交换论证或「为何每次取最大」即可得分。
