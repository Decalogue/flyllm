# 111. 二叉树的最小深度

**标签**：`BFS` `DFS` `树` `广度优先搜索` `深度优先搜索` `简单`  
**分类**：二叉树、遍历、层次扩展  
**难度**：⭐ 简单  
**频率**：🔥🔥 中频

---

## 题目描述

给定一个二叉树，找出其**最小深度**。

**最小深度**的定义：从根结点到**最近叶子结点**的路径上的结点个数（含根与叶）。

**叶子结点**：左右孩子均为空的结点。

### 示例 1

```
输入：root = [3,9,20,null,null,15,7]
输出：2
解释：最短路径为 3 → 9，共 2 个结点。
```

### 示例 2

```
输入：root = [2,null,3,null,4,null,5,null,6]
输出：5
解释：树呈右链，叶子在最底，路径 2→3→4→5→6，深度为 5。
```

### 示例 3

```
输入：root = []
输出：0
```

### 提示

- 树中结点数的范围是 `[0, 10^5]`  
- `-1000 <= Node.val <= 1000`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：最小深度是「根到**叶子**的最短路径」；BFS **第一次遇到叶子**时的层数即为答案；DFS 时要注意**只有一侧子树**时不能用 `1 + min(左, 右)` 把空子树当 0 误用。

**关键词**：`叶子定义` `BFS 最早层` `单子树特判` `与最大深度的区别`

### 复杂度速记

- **时间**：O(n)，每个结点最多访问常数次。  
- **空间**：BFS 队列最坏 O(n)；DFS 递归栈最坏 O(n)（链状树）。

### 记忆口诀

「最小深度看叶子，一层一层 BFS；递归别乱取 min，一边空了走另一边。」

---

## 解题思路

### 核心思想

- **定义**：深度按「结点个数」计，且终点必须是**叶子**。  
- **BFS**：按层扩展，**最先弹出的无左右孩子的结点**所在层深就是最小深度，与「无权图最短路」同构。  
- **DFS**：当前结点若为叶子返回 1；若**只有左或只有右**，答案在**非空**子树里，应返回 `1 + minDepth(那侧)`；若两侧都有，返回 `1 + min(左深, 右深)`。

**关键点：**

- 与 [104. 二叉树的最大深度](https://leetcode.cn/problems/maximum-depth-of-binary-tree/) 不同：最大深度可写 `1 + max(l, r)`；最小深度在**一侧为空**时不能把空子树深度当成 0 再取 min。  
- 空树深度为 0。  
- BFS 在「极浅叶子」存在时可以**提前终止**，不必遍历整棵树（最坏仍要 O(n)）。

### 算法步骤（BFS）

1. 若 `root` 为空，返回 `0`。  
2. 队列存放 `(结点, 当前深度)`，根入队，深度为 `1`。  
3. 弹出队首：若为叶子（无左右孩子），返回当前深度。  
4. 否则将非空孩子入队，深度为 `当前深度 + 1`。  
5. 重复直到找到叶子。

### 算法步骤（DFS）

1. 若 `root` 为空，返回 `0`。  
2. 若为叶子，返回 `1`。  
3. 若只有左子树：`return 1 + minDepth(root.left)`。  
4. 若只有右子树：`return 1 + minDepth(root.right)`。  
5. 否则：`return 1 + min(minDepth(root.left), minDepth(root.right))`。

### 为什么两种都可以？

- **BFS**：语义与「最短步数到叶子」一致，代码直观，易讲清给面试官。  
- **DFS**：一次遍历、无队列；需注意单子树分支，避免套错最大深度的模板。

## 代码实现

### 写法一：BFS（推荐 ⭐）

```python
from collections import deque


class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def minDepth(self, root: TreeNode | None) -> int:
        """
        层序扩展，第一次遇到叶子即返回当前深度。
        时间复杂度: O(n)，空间复杂度: O(n)（队列）
        """
        if root is None:
            return 0
        q = deque([(root, 1)])
        while q:
            node, d = q.popleft()
            if node.left is None and node.right is None:
                return d
            if node.left is not None:
                q.append((node.left, d + 1))
            if node.right is not None:
                q.append((node.right, d + 1))
        return 0
```

**代码要点：**

- ✅ 用「是否叶子」判断返回，而不是「是否无孩子可入队」的模糊条件。  
- ✅ 空树在入队前就返回 0。  
- ✅ 左右孩子分别判断再入队，避免 `None` 入队。

### 写法二：DFS 递归

```python
class Solution:
    def minDepth(self, root: TreeNode | None) -> int:
        """
        单子树时必须沿非空侧向下，不能 1 + min(左,右) 当一侧为 0。
        时间复杂度: O(n)，空间复杂度: O(h)，h 为树高，最坏 O(n)
        """
        if root is None:
            return 0
        if root.left is None and root.right is None:
            return 1
        if root.left is None:
            return 1 + self.minDepth(root.right)
        if root.right is None:
            return 1 + self.minDepth(root.left)
        return 1 + min(self.minDepth(root.left), self.minDepth(root.right))
```

### 版本对比

| 对比项 | BFS | DFS |
|--------|-----|-----|
| **时间** | O(n) | O(n) |
| **额外空间** | 队列 O(n) | 栈 O(h)，链状 O(n) |
| **易错点** | 较少 | 易写成最大深度式子 |
| **面试表述** | 「第一次碰到叶子」很清晰 | 需强调单子树分支 |

### 执行过程示例（BFS）

`root = [3,9,20,null,null,15,7]`：

```
队列初始: [(3,1)]
弹出 3，非叶子，入队 (9,2), (20,2)
弹出 9，是叶子 → 返回深度 2
```

## ⚠️ 常见陷阱（面试易错点）

1. **把「最小深度」当成 `1 + min(左深, 右深)` 一路用到底**  
   - ❌ **错误做法**：一侧为空、另一侧很深时，空侧深度按 0 参与 min，得到错误的小深度。  
   - ✅ **正确做法**：仅当**左右都存在**时才 `1 + min`；只有一侧时 `1 + minDepth(非空侧)`。  
   - 💡 **原因**：最小深度终点必须是**叶子**；根若只有一棵子树，根本身不是叶子，最短路径必须走到那棵子树里的叶子。

2. **与最大深度代码混用**  
   - 最大深度：`return 0 if not root else 1 + max(l, r)`。  
   - 最小深度：不能简单把 `max` 换成 `min`，必须保留单子树判断。

3. **边界**  
   - 空树：0。  
   - 单结点：1。

## 复杂度分析

### 时间复杂度

- **BFS / DFS**：均为 O(n)，每个结点访问常数次。

### 空间复杂度

- **BFS**：队列规模最坏 O(n)（例如满树最后一层）。  
- **DFS**：递归栈 O(h)，平衡树约 O(log n)，链状树 O(n)。

### 复杂度对比表

| 方法 | 时间 | 空间 | 备注 |
|------|------|------|------|
| BFS | O(n) | O(n) | 推荐，语义贴题 |
| DFS | O(n) | O(h) | 代码短，注意分支 |

## 关键点

1. **题目中的「深度」与「到叶子」**  
   - 路径两端必须是根与**叶子**，不能停在只有一棵子树的中间结点。

2. **为何 BFS 适合**  
   - 按层递增，第一次到达的叶子即深度最小，与最短路思想一致。

3. **追问：若改成「到任意 null 子指针的最短边数」？**  
   - 定义变化后结论不同；本题以官方定义（到叶子、结点计数）为准。

## 💬 面试问题（模拟面试场景）

### 基础问题

1. **Q：最小深度和最大深度在代码上最大的区别是什么？**  
   - A：最大深度可统一写成 `1 + max(左,右)`；最小深度在一侧为空时不能把空当作深度 0 去取 min，必须只在非空子树里继续算。

2. **Q：BFS 为什么第一次遇到叶子就可以停？**  
   - A：BFS 按层扩展，层号从小到大；第一次访问到的叶子一定属于层数最小的那一层。

### 进阶问题

1. **Q：若树很大但保证很宽很浅，选 BFS 还是 DFS？**  
   - A：深度很小时 BFS 队列宽度可能很大；但本题求最浅叶子，BFS 往往更早结束。DFS 空间 O(h)，在浅而宽时栈更省，但最坏仍要扫全树。
