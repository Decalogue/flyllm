# 94. 二叉树的中序遍历

**标签**：`栈` `树` `深度优先搜索` `简单`  
**分类**：二叉树、遍历  
**难度**：⭐ 简单  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你二叉树的根节点 `root`，返回其节点值的**中序遍历**。

**中序遍历**：先**中序**遍历左子树，再访问**根**结点，再**中序**遍历右子树。顺序记作：**左 → 根 → 右**。对二叉搜索树（BST）而言，中序遍历得到的是**按键值递增**的序列。

### 示例 1

```
输入：root = [1,null,2,3]
输出：[1,3,2]
```

### 示例 2

```
输入：root = []
输出：[]
```

### 示例 3

```
输入：root = [1]
输出：[1]
```

### 提示

- 树中结点数目范围：`[0, 100]`
- `-100 <= Node.val <= 100`

**进阶**：递归解法很简单，能否用**迭代**完成？

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：递归按「左根右」；迭代时用栈：沿左子树一路压栈到底，弹出时访问该结点，再转向其右子树（右子树仍是「先一路向左」）。

**关键词**：`中序` `左根右` `一路向左压栈` `弹栈访问再走右`

### 复杂度速记

- **时间**：O(n)，每个结点访问常数次。  
- **空间**：递归 O(h)；迭代栈最坏 O(h)；Morris 为 O(1) 额外空间（会临时改树指针，需理解恢复过程）。h 为树高。

### 记忆口诀

「中序先左后根：指针顺着左枝全入栈，弹出再记根，最后钻进右边 subtree。」

---

## 解题思路

### 核心思想

中序是 **DFS** 的一种访问顺序，与定义一致即可写出递归。迭代时难点在于：**根要在左子树全部处理完后才输出**，因此不能像前序那样「一入栈就访问」；典型做法是维护当前指针 `cur` 与栈 `st`：

- 只要 `cur` 非空，就把 `cur` 压栈并 `cur = cur.left`（把自己和整条左链存起来，表示「左子树还没处理完」）。
- `cur` 为空时说明左子树到头，弹栈得到「当前子树的根」，访问之，再 `cur = cur.right` 去处理右子树。

循环条件为 `while cur or st`：右子树可能为空，需靠栈里未完成的祖先继续。

### 算法步骤（迭代）

1. `res = []`，`st = []`，`cur = root`。  
2. 当 `cur` 或 `st` 非空：  
   - 内层 `while cur`：`st.append(cur)`，`cur = cur.left`。  
   - `cur = st.pop()`，将 `cur.val` 加入 `res`。  
   - `cur = cur.right`。  
3. 返回 `res`。

### 与前序、后序的对比

- **前序**（[144](144.二叉树的前序遍历.md)）：根先输出，迭代常「先压右再压左」从栈顶模拟。  
- **中序**：根在左之后，故需「左链压栈 + 弹栈访问 + 转右」。  
- **后序**（[145](145.二叉树的后序遍历.md)）：根最后，迭代写法通常更绕（双栈或标记），勿与中序模板混用。

## 代码实现

### 写法一：递归（最直观）

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def inorderTraversal(self, root: TreeNode | None) -> list[int]:
        res: list[int] = []

        def dfs(n: TreeNode | None) -> None:
            if not n:
                return
            dfs(n.left)
            res.append(n.val)
            dfs(n.right)

        dfs(root)
        return res
```

**要点**：访问顺序严格为左 → 根 → 右，与定义一致。

### 写法二：迭代 + 栈（推荐应对进阶）

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def inorderTraversal(self, root: TreeNode | None) -> list[int]:
        res: list[int] = []
        st: list[TreeNode] = []
        cur: TreeNode | None = root
        while cur or st:
            while cur:
                st.append(cur)
                cur = cur.left
            cur = st.pop()
            res.append(cur.val)
            cur = cur.right
        return res
```

**要点**：`while cur` 把左链全部压栈；弹栈时刻即为「左子树已处理完」，此时访问根，再进入右子树。

### 写法三：Morris 中序遍历（O(1) 额外空间，进阶）

利用叶子结点的空 `right` 指针临时指向中序后继，避免用栈；遍历结束需把树还原。

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def inorderTraversal(self, root: TreeNode | None) -> list[int]:
        res: list[int] = []
        cur: TreeNode | None = root
        while cur:
            if cur.left is None:
                res.append(cur.val)
                cur = cur.right
            else:
                pred = cur.left
                while pred.right and pred.right is not cur:
                    pred = pred.right
                if pred.right is None:
                    pred.right = cur
                    cur = cur.left
                else:
                    pred.right = None
                    res.append(cur.val)
                    cur = cur.right
        return res
```

**要点**：`pred` 为 `cur` 在中序下的前驱；第一次线索化时走向左子树，第二次说明左子树已遍历完，访问 `cur` 并拆线索。

---

## ⚠️ 常见陷阱

1. **循环条件写成 `while st`**  
   - ❌ 当 `cur` 已移到右孩子、但栈一时为空且右子树非空时，会提前退出。  
   - ✅ 使用 `while cur or st`。

2. **把前序的「先压右再压左」套到中序**  
   - 中序必须先处理完左子树再访问根，**一路向左压栈**才是正解。

3. **Morris 漏拆线索**  
   - 第二次经过结点时必须 `pred.right = None`，否则树结构被破坏。

4. **空树**  
   - `root` 为 `None` 时，`cur` 与 `st` 皆空，不进入循环，返回 `[]`。

## 复杂度分析

| 写法 | 时间 | 额外空间 | 说明 |
|------|------|----------|------|
| 递归 | O(n) | O(h) | 调用栈深度为树高。 |
| 迭代栈 | O(n) | O(h) | 左链最长为 h。 |
| Morris | O(n) | O(1) | 每个边最多走两次，常数额外指针。 |

最坏（链状树）h = n；平衡树 h = O(log n)。

## 关键点

1. **BST 与中序**：若题目隐含 BST，中序遍历常用来**有序输出**或校验 BST 性质。  
2. **与前序 144、后序 145**：三种遍历 DFS 时间均为 O(n)，差异在**访问根的时机**与迭代实现细节。  
3. **Morris 适用场景**：面试官追问「不用栈、O(1) 空间」时再展开，需能口述线索化与恢复。
