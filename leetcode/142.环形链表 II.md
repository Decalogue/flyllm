# 142. 环形链表 II

**标签**：`链表` `双指针` `哈希表` `中等`  
**分类**：链表、快慢指针（Floyd 判环）、环入口  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频  

---

## 题目描述

给定一个链表的头节点 `head`，返回链表开始**入环的第一个节点**。如果链表无环，则返回 `null`。

说明：不允许修改给定链表；链表可能在你未知的某个内部节点处成环。

### 示例 1

```
可视化：
3 → 2 → 0 → -4
        ↑     |
        └─────┘
（-4 指向 2，环从节点 2 开始）

对于本题 API：链表结构如上，返回值为值为 2 的那个节点引用。
```

### 示例 2

```
1 → 2
↑   |
└───┘

环从节点 1 开始，返回值为 1 的节点。
```

### 示例 3

```
1 → 2 → null

无环，返回 null。
```

### 提示

- 链表中节点的数目范围在 `[0, 10^4]` 内。  
- `-10^5 <= Node.val <= 10^5`  
- `pos`（若存在环）为环起始节点在链表中的下标（面试/手写测试时常用）。

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：先用**快慢指针**判断并找到环上相遇点；再令一指针回**头结点**，两指针**同速**前进，相遇处即**环入口**。

**关键词**：`Floyd 判环` `slow/fast` `二次相遇` `a + b = nC`

### 复杂度速记

| 方法 | 时间 | 空间 |
|------|------|------|
| Floyd（双指针） | O(n) | O(1) |
| 哈希记录已访问结点 | O(n) | O(n) |

### 记忆口诀

「快追慢相遇在环上；一头回到起点，同步走撞到入口。」

---

## 解题思路

### 核心思想

1. **第一阶段**：`slow` 每次一步、`fast` 每次两步。若存在环，二者必在环上某点相遇；若无环，`fast` 会先抵达 `null`。  
2. **第二阶段**：相遇后令 `p = head`，`p` 与 `slow` 每次各走一步，再次相遇点即为**环的第一个节点**。

### 数学证明（为什么第二阶段成立）

设：

- 从 `head` 到环入口的链长为 **a**；  
- 从入口沿环前进到快慢**第一次相遇点**的距离为 **b**；  
- 环长为 **C**。

则第一次相遇时，`slow` 走过 **a + b**；`fast` 走过 **a + b + nC**（在环上多绕了 **n** 圈），且 `fast` 路程是 `slow` 的 **2** 倍：

\[
2(a + b) = a + b + nC \implies a + b = nC \implies a = nC - b.
\]

含义：从**相遇点**再沿环前进 **a** 步，等价于先走 **nC - b**，恰好落在**环入口**（因为从相遇点走到入口沿环的“正向”距离是 **C - b**，配合整圈 **nC** 对齐后就是入口）。

因此：指针 **A** 从 `head` 走 **a** 步到入口；指针 **B** 从**相遇点**同速也走 **a** 步，也会到入口，二者在入口相遇。

### 方法对比

| 方法 | 做法 | 面试怎么说 |
|------|------|------------|
| **Floyd** | 快慢相遇 + 一头回 head 同速走 | 空间 O(1)，必会推导 a、b、C 关系 |
| **哈希表** | 遍历，`visited` 里第一次重复即入口 | 好写，空间 O(n)，适合追问「有没有更省空间」 |

### 易错点

- **相遇后**必须让其中一指针回到 **`head`**，再与另一指针**同样每次一步**，不能继续用「一步 vs 两步」。  
- 无环时应在**第一阶段**就返回，不要进入第二阶段。  
- 「相遇点不一定是入口」——只是推导桥梁，入口由第二阶段求出。

## 代码实现

### 方法一：Floyd 双指针（⭐ 面试首选）

```python
class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None


class Solution:
    def detectCycle(self, head: ListNode | None) -> ListNode | None:
        slow = fast = head
        while fast and fast.next:
            slow, fast = slow.next, fast.next.next
            if slow is fast:
                p = head
                while p is not slow:
                    p, slow = p.next, slow.next
                return p
        return None
```

**要点**：

- `while fast and fast.next` 同时保证 `fast`、`fast.next` 非空，避免无环时空指针。  
- 第一阶段用 **`is`** 比较结点对象（题目要求返回结点引用）。  
- 第二阶段 `p` 与 `slow` 同步移动直至相等，该结点即为环入口。

### 方法二：哈希表（直观、占空间）

```python
class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None


class Solution:
    def detectCycle(self, head: ListNode | None) -> ListNode | None:
        seen: set[ListNode] = set()
        while head:
            if head in seen:
                return head
            seen.add(head)
            head = head.next
        return None
```

## 复杂度

- **Floyd**：时间 **O(n)**（指针绕环与链长线性相关），空间 **O(1)**。  
- **哈希**：时间 **O(n)**，空间 **O(n)**。

## 与 141 的关系

- **141**：只判**是否有环**——快慢相遇即可 `return True`。  
- **142**：还要**定位环起点**——需第二阶段「一头回 `head`、同步走」。可先写 141 再扩展为 142，面试常连环追问。
