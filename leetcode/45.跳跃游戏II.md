## 题目描述

给定一个长度为 n 的 0 索引整数数组 nums。初始位置在下标 0。

每个元素 nums[i] 表示从索引 i 向后跳转的最大长度。换句话说，如果你在索引 i 处，你可以跳转到任意 (i + j) 处：

- 0 <= j <= nums[i] 且
- i + j < n

返回到达 n - 1 的最小跳跃次数。测试用例保证可以到达 n - 1。

### 示例 1

```
输入: nums = [2,3,1,1,4]
输出: 2
解释: 跳到最后一个位置的最小跳跃数是 2。
     从下标为 0 跳到下标为 1 的位置，跳 1 步，然后跳 3 步到达数组的最后一个位置。
```

### 示例 2

```
输入: nums = [2,3,0,1,4]
输出: 2
```

### 提示

- 1 <= nums.length <= 10^4
- 0 <= nums[i] <= 1000
- 题目保证可以到达 n - 1

### 核心思路

**一句话总结**：从右到左DP记录每个位置到终点的最少跳跃次数，或者用贪心算法从左到右跟踪当前能到达的最远位置。

**关键词**：`动态规划` `贪心算法` `最少跳跃次数`

### 复杂度速记

- **时间复杂度**：
  - DP解法：O(n²) - 每个位置需要检查所有可跳位置
  - 贪心解法：O(n) - 一次遍历
- **空间复杂度**：
  - DP解法：O(n) - DP数组
  - 贪心解法：O(1) - 只使用几个变量

### 记忆口诀

"从右到左DP找最小，贪心一次遍历更高效"

## 解题思路

### 核心思想

**方法1：动态规划（从右到左）**

- `dp[i]` 表示从位置 i 跳到终点 (n-1) 的最少跳跃次数
- 从右到左遍历，对于每个位置 i，查看它能跳到的所有位置，取最小值
- 初始状态：`dp[n-1] = 0`（终点不需要跳）

**方法2：贪心算法（推荐）**

- 维护当前能到达的最远位置 `max_reach`
- 维护当前跳跃的边界 `end`
- 当到达边界时，增加跳跃次数，更新边界为当前能到达的最远位置

### 算法步骤

#### 方法1：DP（从右到左）

1. 初始化 `dp` 数组，长度为 n，所有值设为无穷大
2. `dp[n-1] = 0`（终点不需要跳）
3. 从 `n-2` 到 `0` 遍历：
  - 对于位置 i，检查所有可跳位置 `[i+1, i+nums[i]]`
  - `dp[i] = min(dp[j] for j in range(i+1, i+nums[i]+1)) + 1`
4. 返回 `dp[0]`

#### 方法2：贪心算法

1. 初始化 `jumps = 0`（跳跃次数）
2. 初始化 `max_reach = 0`（当前能到达的最远位置）
3. 初始化 `end = 0`（当前跳跃的边界）
4. 遍历数组（不遍历最后一个元素）：
  - 更新 `max_reach = max(max_reach, i + nums[i])`
  - 当 `i == end` 时（到达当前边界）：
    - `jumps += 1`
    - `end = max_reach`
5. 返回 `jumps`

### 为什么选择这个方法？

- **DP优势**：思路直观，容易理解
- **DP缺点**：时间复杂度O(n²)，对于大数据量较慢
- **贪心优势**：时间复杂度O(n)，空间复杂度O(1)，最优解
- **适用场景**：机考时优先用贪心，时间更少

## 完整代码实现

### 方法1：动态规划（从右到左）

```python
class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        # dp[i] 表示从位置i跳到终点的最少跳跃次数
        dp = [float('inf')] * n
        dp[n - 1] = 0  # 终点不需要跳
        
        # 从右到左遍历
        for i in range(n - 2, -1, -1):
            # 从位置i最多能跳到 i + nums[i]
            max_jump = min(i + nums[i], n - 1)
            # 在可跳范围内找到最小值
            for j in range(i + 1, max_jump + 1):
                dp[i] = min(dp[i], dp[j] + 1)
        
        return dp[0]
```

**复杂度分析：**

- 时间复杂度：O(n²)，对于每个位置需要检查所有可跳位置
- 空间复杂度：O(n)，DP数组

### 方法2：贪心算法（推荐）⭐

```python
class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        if n <= 1:
            return 0 
        
        jumps = 0  # 跳跃次数
        max_reach = 0  # 当前能到达的最远位置
        end = 0  # 当前跳跃的边界
        
        # 遍历到倒数第二个位置即可
        for i in range(n - 1):
            # 更新能到达的最远位置
            max_reach = max(max_reach, i + nums[i])
            
            # 到达当前边界，需要进行下一次跳跃
            if i == end:
                jumps += 1
                end = max_reach
                
                # 如果已经能到达终点，提前结束
                if max_reach >= n - 1:
                    break
        
        return jumps
```

**复杂度分析：**

- 时间复杂度：O(n)，一次遍历
- 空间复杂度：O(1)，只使用常数空间

### 方法3：贪心算法（简化版）

```python
class Solution:
    def jump(self, nums: List[int]) -> int:
        n = len(nums)
        jumps = 0
        end = 0
        farthest = 0
        
        for i in range(n - 1):
            farthest = max(farthest, i + nums[i])
            if i == end:
                jumps += 1
                end = farthest
        
        return jumps
```

## 算法图解

### DP方法示例：nums = [2,3,1,1,4]

```
索引:  0  1  2  3  4
nums:  2  3  1  1  4
dp:    ∞  ∞  ∞  ∞  0

从右到左：
i=3: dp[3] = min(dp[4]) + 1 = 0 + 1 = 1
i=2: dp[2] = min(dp[3]) + 1 = 1 + 1 = 2
i=1: dp[1] = min(dp[2], dp[3], dp[4]) + 1 = min(2,1,0) + 1 = 1
i=0: dp[0] = min(dp[1], dp[2]) + 1 = min(1,2) + 1 = 2

结果：dp[0] = 2
```

### 贪心方法示例：nums = [2,3,1,1,4,1]

```
i=0: max_reach = 2, end = 0
     到达边界，jumps=1, end=2

i=1: max_reach = max(2, 1+3) = 4, end = 2
     未到达边界

i=2: max_reach = max(4, 2+1) = 4, end = 2
     到达边界，jumps=2, end=4

i=3: max_reach = max(4, 3+1) = 4, end = 4
     未到达边界

i=4: max_reach = max(4, 4+4) = 8, end = 4
     到达边界, jumps=3, end=8

结果：jumps = 3
```

## 常见错误

### ❌ 错误1：理解错误

```python
# 错误：以为nums[i]表示固定跳nums[i]步
for i in range(len(nums)):
    jump(nums[i])  # 错误理解
```

### ❌ 错误2：DP方向错误

```python
# 错误：从左到右DP，需要记录所有可能路径
# 这样会超时或逻辑复杂
```

### ✅ 正确思路

- DP从右到左：终点状态已知，向前推导
- 贪心从左到右：每次选择能跳最远的位置

## 总结

从右到左DP是一个很好的解法。但机考时建议用贪心算法，因为：

1. ✅ 时间复杂度更优：O(n) vs O(n²)
2. ✅ 空间复杂度更优：O(1) vs O(n)
3. ✅ 代码更简洁

**记住核心思想：**

- DP：`dp[i] = min(dp[所有可跳位置]) + 1`
- 贪心：每次到达边界时，选择当前能跳最远的位置作为新边界
