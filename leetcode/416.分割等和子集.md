# 416. 分割等和子集

**标签**：`数组` `动态规划` `中等`  
**分类**：0-1 背包、子集和  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你一个**非空**数组 `nums`，其中元素均为**正整数**。判断能否将数组**分成两个子集**，使两个子集的元素之和**相等**。

- 每个元素必须**恰好**属于其中一个子集（不重复、不遗漏）。  
- 若可以，返回 `true`；否则返回 `false`。

### 示例 1

```
输入：nums = [1,5,11,5]
输出：true
解释：可分为 [1,5,5] 与 [11]，和均为 11。
```

### 示例 2

```
输入：nums = [1,2,3,5]
输出：false
解释：总和 11 为奇数，无法均分。
```

### 提示

- `1 <= nums.length <= 200`
- `1 <= nums[i] <= 100`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：两子集和相等 ⇔ 存在子集和为 **`sum(nums) / 2`**；总和为奇数直接 `false`。把每个数看成**体积=价值=nums[i]** 的物品，问能否恰好装满容量 **`target = sum/2`** 的背包 → **0-1 背包**可达性 `dp[j]`。

**关键词**：`和的一半` `0-1 背包` `一维 DP 倒序`

### 复杂度速记

- **时间**：O(n × target)，`target = sum/2`，本题 `sum ≤ 200×100`，可接受。  
- **空间**：O(target) 一维布尔数组。

### 记忆口诀

「先判奇偶，再凑半和；内层从大到小，防同一数用两次。」

---

## 解题思路

### 等价转化

设总和为 `S`。若两子集和均为 `T`，则 `2T = S`，故 **`S` 必须为偶数**，且问题等价于：是否存在子集，其元素和为 **`T = S // 2`**。

### 0-1 背包建模

- **物品**：每个 `nums[i]` 最多选一次。  
- **容量**：`target = S // 2`。  
- **状态**：`dp[j]` 表示是否可用已考虑的若干个数**凑出和 `j`**。  
- **边界**：`dp[0] = True`（和为 0 总可达）。  
- **转移**：对每个数 `x`，若 `dp[j - x]` 已为真，则 `dp[j]` 可为真：  
  `dp[j] = dp[j] or dp[j - x]`（在 `j >= x` 时）。

### 为何内层要「从大到小」枚举 `j`

一维数组滚动时，若 **`j` 从小到大**更新，同一轮里会用**刚加入的 `x` 再次更新更大下标**，等价于 **`x` 被用了多次**，违反 0-1 约束。  
**`j` 从 `target` 递减到 `x`**，保证 `dp[j - x]` 来自**上一轮**（尚未放入当前 `x`）的状态。

### 可选剪枝

- `max(nums) > target`：最大元已超过目标和，不可能（除非单独成子集且另一边也能凑——实际上若 `max > target` 且总和为 `2*target`，则最大元无法放入「和为 target」的子集，直接 `false`）。

## 代码实现

### 写法一：一维布尔 DP（推荐）

```python
class Solution:
    def canPartition(self, nums: list[int]) -> bool:
        s = sum(nums)
        if s % 2:
            return False
        t = s // 2
        dp = [False] * (t + 1)
        dp[0] = True
        for x in nums:
            for j in range(t, x - 1, -1):
                dp[j] = dp[j] or dp[j - x]
        return dp[t]
```

### 写法二：二维 DP（便于讲清「前 i 个数能否凑 j」）

```python
class Solution:
    def canPartition(self, nums: list[int]) -> bool:
        s = sum(nums)
        if s % 2:
            return False
        t = s // 2
        n = len(nums)
        # dp[i][j]：考虑前 i 个数能否凑出和 j
        dp = [[False] * (t + 1) for _ in range(n + 1)]
        for i in range(n + 1):
            dp[i][0] = True
        for i in range(1, n + 1):
            x = nums[i - 1]
            for j in range(1, t + 1):
                dp[i][j] = dp[i - 1][j]
                if j >= x:
                    dp[i][j] = dp[i][j] or dp[i - 1][j - x]
        return dp[n][t]
```

**空间**：O(n × t)；可滚动为一维即写法一。

### 写法三：位运算压缩（进阶）

把 `dp` 看成集合「当前可达的和」，每加一个 `x`：`reachable |= reachable << x`。需注意 Python 中大整数仍受 `target` 限制，实现时可用 `& ((1 << (t + 1)) - 1)` 截断高于 `t` 的位（或判断 `t` 位是否为 1）。面试以写法一为主。

---

## ⚠️ 常见陷阱

1. **只判奇偶忘记 DP**  
   - 偶数和也可能无法分割，如 `[1,2,5]`，和为 8 但无法凑出 4。

2. **一维 DP 正序枚举 `j`**  
   - 会变成**完全背包**语义，同一元素可被「用多次」，答案错误。

3. **空数组 / 单元素**  
   - 题目保证非空；长度为 1 时无法分成两个非空子集，一般理解为 `false`（与「两子集和相等」在力扣测试下一致）。

4. **与「子集个数」题混淆**  
   - 本题只要**是否存在**；若问方案数，转移会不同。

---

## 复杂度分析

- **一维 DP**：时间 **O(n × target)**，空间 **O(target)**，`target = sum(nums) / 2`。  
- **二维 DP**：空间 **O(n × target)**，理解用。

在提示范围内，`sum(nums) ≤ 200 × 100 = 20000`，`target ≤ 10000`，可行。
