# 3. 无重复字符的最长子串

**标签**：`哈希表` `字符串` `滑动窗口` `中等`  
**分类**：双指针、滑动窗口  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给定字符串 `s`，找出其中**不含重复字符**的**最长连续子串**的长度。

- **子串**：`s` 中连续的一段；本题要求这段里**每个字符最多出现一次**（即窗口内互不相同）。  
- 若 `s` 为空，长度为 `0`。

### 示例 1

```
输入：s = "abcabcbb"
输出：3
解释：最长无重复子串为 "abc"，长度为 3。
```

### 示例 2

```
输入：s = "bbbbb"
输出：1
解释：任意单个字符构成的子串均无重复，最长长度为 1。
```

### 示例 3

```
输入：s = "pwwkew"
输出：3
解释：最长无重复子串为 "wke"，长度为 3。
注意：子串必须是连续的，"pwke" 是子序列而非子串。
```

### 提示

- `0 <= s.length <= 5 * 10^4`
- `s` 由英文字母、数字、符号和空格组成

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：维护 `[left, right]` 滑动窗口，右端每次右移一位；若 `s[right]` 在窗口内**已出现过**，则把 **`left` 挪到该字符上次出现位置的下一个**，保证窗口内无重复；全程用 **`right - left + 1` 更新最大长度**。

**关键词**：`滑动窗口` `字符 → 最后下标` `left = max(left, last[c] + 1)`

### 复杂度速记

- **时间**：O(n)，`left`、`right` 均只向前移动，各至多 n 步。  
- **空间**：O(min(n, |Σ|))，哈希表存出现过的字符种类（|Σ| 为字符集大小）。

### 记忆口诀

「右扩一格，重复就把左拉到上次后面；窗口长度随时刷最大。」

---

## 解题思路

### 为何是滑动窗口

无重复是**「窗口内性质」**：若 `[left, right]` 合法，则右端加入新字符后，要么仍合法，要么**仅因新字符与窗口内某次重复**而不合法。不合法时，**左端不必从 0 重试**，只需把 `left` 移到**第一次能消掉重复**的位置——即**重复字符上次出现下标 + 1**，其左侧全部丢弃，窗口仍连续。

### 用「最后出现位置」字典

- `last[c]`：字符 `c` 在遍历过程中**最近一次**的下标。  
- 当扫描到 `s[right] = c` 时：若 `c` 曾在**当前窗口内**出现过，则必有 `last[c] >= left`，此时令 **`left = last[c] + 1`**。  
- **必须用 `max(left, last[c] + 1)` 或先判断 `last[c] >= left`**：若 `c` 上次出现在**已滑出窗口的左侧**，`last[c] < left`，则**不能**把 `left` 错误左移（本题代码用 `if c in last and last[c] >= left` 处理）。

### 更新顺序

先根据重复收缩 `left`，再 **`last[c] = right`**，最后用当前窗口长度更新答案。

### 其他写法（口述）

- **`collections.Counter` + while 左移**：右加字符，若计数 > 1 则不断 `left++` 直到合法；最坏仍 O(n)（双指针均单向）。  
- **ASCII 小表**：若字符集很小，可用 **长度为 128 的数组** 代替字典，常数更小。

## 代码实现

### 写法一：哈希表记最后下标（推荐）

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        last: dict[str, int] = {}
        left = 0
        ans = 0
        for right, c in enumerate(s):
            if c in last and last[c] >= left:
                left = last[c] + 1
            last[c] = right
            ans = max(ans, right - left + 1)
        return ans
```

### 写法二：`left = max(left, last.get(c, -1) + 1)` 合并分支

未出现过可用 `-1` 表示，`last.get(c, -1) + 1` 与 `left` 取 max 等价于「仅当重复在窗口内才右移 `left`」。

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        last: dict[str, int] = {}
        left = 0
        ans = 0
        for right, c in enumerate(s):
            left = max(left, last.get(c, -1) + 1)
            last[c] = right
            ans = max(ans, right - left + 1)
        return ans
```

**说明**：`last.get(c, -1) + 1` 在 `c` 首次出现时值为 `0`，`max(left, 0)` 不改变 `left`；当 `c` 在窗口内重复时，`last[c] + 1` 会大于当前 `left`，从而收缩窗口。

### 写法三：定长数组（字符集为 ASCII 时）

```python
class Solution:
    def lengthOfLongestSubstring(self, s: str) -> int:
        last = [-1] * 128
        left = 0
        ans = 0
        for right, c in enumerate(s):
            x = ord(c)
            if last[x] >= left:
                left = last[x] + 1
            last[x] = right
            ans = max(ans, right - left + 1)
        return ans
```

---

## ⚠️ 常见陷阱

1. **忘记 `last[c] >= left`**  
   - 仅用 `left = last[c] + 1` 会把 `left` 拽到**已不在窗口内的旧重复**之后，窗口错误缩短。必须保证**重复位置仍在当前 `[left, right]` 内**。

2. **与「子序列」混淆**  
   - 示例 `"pwwkew"` 中 `"pwke"` 不是连续子串，不能算。

3. **空串**  
   - `left=0`，循环不进入，`ans=0`，正确。

4. **Unicode**  
   - 若语言/题目含宽字符，数组开 128 不够，应退回**哈希表**。

## 复杂度分析

- **时间复杂度**：O(n)，每个下标最多被 `left`、`right` 各经过一次。  
- **空间复杂度**：O(min(n, |Σ|))，存储已出现字符的最后位置。

## 总结

3 题是**滑动窗口入门**：右指针扫描，`last` 记下标，重复且**在窗口内**则 `left` 跳到 `last[c]+1`，维护**无重复**不变量并更新最大长度。面试写写法一或写法二均可；追问「为何 O(n)」答**双指针单调右移**。
