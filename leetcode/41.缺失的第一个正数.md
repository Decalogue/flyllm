# 41. 缺失的第一个正数

**标签**：`数组` `哈希表` `困难`  
**分类**：数组、原地哈希  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 中高频

---

## 题目描述

给你一个未排序的整数数组 `nums`，请你找出其中**没有出现的最小的正整数**。

要求：**时间复杂度 O(n)**，且**只使用常数级别额外空间**。

### 示例 1

```
输入：nums = [1,2,0]
输出：3
解释：范围 [1,2] 中的数字都在数组中，最小未出现正数为 3。
```

### 示例 2

```
输入：nums = [3,4,-1,1]
输出：2
解释：1 在数组中，但 2 没有出现。
```

### 示例 3

```
输入：nums = [7,8,9,11,12]
输出：1
解释：最小的正整数 1 没有出现。
```

### 提示

- `1 <= nums.length <= 10^5`
- `-2^31 <= nums[i] <= 2^31 - 1`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**结论**：答案一定在 `[1, n+1]` 中（n 为数组长度）。若 1～n 都出现则答案为 n+1。

**O(1) 空间做法（原地哈希）**：用下标表示「是否出现」——若正整数 x 在 `[1, n]` 内，就把它放到位置 `x-1`（即 `nums[x-1] = x`）。遍历一遍把能放的都放到「正确位置」，再扫一遍，第一个满足 `nums[i] != i+1` 的 i，答案就是 `i+1`；若都满足则答案为 `n+1`。

**关键词**：`答案在 [1,n+1]` `原地哈希` `nums[x-1]=x` `下标当桶`

### 复杂度速记

- **时间**：O(n)——每个位置上的数最多被交换到正确位置一次。
- **空间**：O(1)——仅用常数额外变量（在原数组上操作）。

### 记忆口诀

「答案必在 1 到 n+1；把值 x 放到下标 x-1，再扫谁不对谁就是答案。」

---

## 解题思路

### 核心思想

- **答案范围**：最小未出现正数一定在 `[1, n+1]`。因为若 1～n 都出现，答案就是 n+1；否则答案在 1～n 之间。
- **原地标记**：不额外开桶，用数组下标 0～n-1 表示 1～n 是否出现。理想状态是「若 1 存在则放在 index0，若 2 存在则放在 index1」，即 `nums[i] == i+1`。因此对每个位置 i，若 `nums[i]` 在 `[1, n]` 且当前 `nums[nums[i]-1] != nums[i]`，就把 `nums[i]` 换到下标 `nums[i]-1` 处（用 swap），并**继续看当前 i 上的新值**（用 while），直到当前值不在 [1,n] 或已在正确位置。最后从左到右找第一个 `nums[i] != i+1`，返回 `i+1`；若都相等则返回 `n+1`。

### 算法步骤

1. 设 `n = len(nums)`。
2. 对 `i` 从 0 到 n-1：  
   `while 1 <= nums[i] <= n and nums[nums[i]-1] != nums[i]`：交换 `nums[i]` 与 `nums[nums[i]-1]`（把 `nums[i]` 放到下标 `nums[i]-1`）。
3. 对 `i` 从 0 到 n-1：若 `nums[i] != i+1`，返回 `i+1`。
4. 返回 `n+1`。

---

## 代码实现

### 写法一：原地哈希（O(1) 空间，推荐）

```python
from typing import List

class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        n = len(nums)
        for i in range(n):
            while 1 <= nums[i] <= n and nums[nums[i] - 1] != nums[i]:
                j = nums[i] - 1
                nums[j], nums[i] = nums[i], nums[j]
        for i in range(n):
            if nums[i] != i + 1:
                return i + 1
        return n + 1
```

**代码要点**：

- ✅ 用 **while** 而不是 if：交换后当前 `nums[i]` 可能仍是 [1,n] 内的新值，需要继续放到正确位置。
- ✅ 条件 `nums[nums[i]-1] != nums[i]` 避免重复交换（已在正确位置则跳过），同时避免死循环。
- ✅ 只关心 [1, n] 内的数，超出范围或非正数留在「错误」位置，第二遍扫描时会被判为 `nums[i] != i+1` 或占位后不影响答案。

### 写法二：哈希集合（O(n) 空间，不满足题面常数空间）

```python
class Solution:
    def firstMissingPositive(self, nums: List[int]) -> int:
        nums_set = set(nums)
        for i in range(1, len(nums) + 1):
            if i not in nums_set:
                return i
        return len(nums) + 1
```

思路简单：答案在 [1, n+1]，从 1 开始枚举第一个不在集合里的数。**空间 O(n)**，若面试明确要求 O(1) 空间需用写法一。

---

## 复杂度分析

| 写法     | 时间   | 空间   | 说明 |
|----------|--------|--------|------|
| 原地哈希 | O(n)   | O(1)   | 每个数最多被交换到正确位置一次；仅常数额外变量。 |
| 哈希集合 | O(n)   | O(n)   | set 存全部元素，不满足「常数额外空间」。 |

---

## 易错点

- **用 while 而不是 if**：交换一次后，当前格子可能又得到一个 [1,n] 的数，需要继续放置，否则会漏排。
- **交换条件**：必须写 `nums[nums[i]-1] != nums[i]`，避免同一对无限交换（两数相等时）以及已就位时多余操作。
- **答案上界**：最小未出现正数最大为 `n+1`（当 1～n 都出现时），所以第二段扫描后若都满足 `nums[i]==i+1`，返回 `n+1`。
