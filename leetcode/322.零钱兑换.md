# 322. 零钱兑换

**标签**：`广度优先搜索` `数组` `动态规划` `中等`  
**分类**：完全背包、最短路（BFS）  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你一个整数数组 `coins`，表示不同面额的**硬币**；另给一个整数 `amount`，表示要凑出的**总金额**。

每种面额的硬币可以取**任意多枚**（完全背包）。若存在若干枚硬币面值之和恰好等于 `amount`，返回所用硬币的**最少枚数**；若**无法凑出**，返回 **`-1`**。

你可以认为每种硬币有**无限个**。

### 示例 1

```
输入：coins = [1,2,5], amount = 11
输出：3
解释：11 = 5 + 5 + 1，共 3 枚。
```

### 示例 2

```
输入：coins = [2], amount = 3
输出：-1
```

### 示例 3

```
输入：coins = [1], amount = 0
输出：0
```

### 提示

- `1 <= coins.length <= 12`
- `1 <= coins[i] <= 2^31 - 1`
- `0 <= amount <= 10^4`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：`dp[x]` 表示凑出金额 `x` 的**最少硬币数**；对每种硬币 `c`，若 `x >= c`，则 `dp[x] = min(dp[x], dp[x-c] + 1)`；初始 `dp[0]=0`，其余为无穷大。

**关键词**：`完全背包` `最优化` `线性 DP` `也可用 BFS 最短路`

### 复杂度速记

- **DP**：时间 **O(amount × |coins|)**，空间 **O(amount)**。  
- **BFS**：状态为当前金额，边为 `+c`，最短路即最少枚数；同阶复杂度量级。

### 记忆口诀

「凑 x：拆成先凑 x−c 再加一枚 c，对所有 c 取最小。」

---

## 解题思路

### 与「贪心」的区别

若硬币为**标准货币体系**（如 1,5,10），贪心有时可行；**任意面额**时贪心**不总最优**（面试常追问反例），应使用 **DP** 或 **BFS**。

### 方法一：DP（完全背包）

- **状态**：`dp[x]` = 和为 `x` 的最少硬币个数，不可达用「无穷大」表示。  
- **边界**：`dp[0] = 0`（金额 0 不需要硬币）。  
- **转移**：对每个 `x = 1..amount`，枚举 `c in coins`，若 `c <= x`：  
  `dp[x] = min(dp[x], dp[x - c] + 1)`。

「完全背包」体现在：同一枚 `c` 可反复用，通过从小到大的 `x` 自然允许同一 `c` 贡献多次。

### 方法二：BFS

把金额看作结点，`0` 为起点；从当前金额 `u` 连边到 `u + c`（`c in coins`），权为 1。求 `0` 到 `amount` 的**最短路径**（边数最少即枚数最少）。`amount` 达 `10^4` 时与 DP 同量级，可作为口述备选。

### 无穷大取值

需满足 `INF + 1` 不溢出且 `INF` 大于任何合法答案。`amount ≤ 10^4` 时可用 **`amount + 1`** 或 **`10**9`** 作为 `INF`。

## 代码实现

### 写法一：DP（`INF = amount + 1`，推荐）

```python
class Solution:
    def coinChange(self, coins: list[int], amount: int) -> int:
        inf = amount + 1
        dp = [inf] * (amount + 1)
        dp[0] = 0
        for x in range(1, amount + 1):
            for c in coins:
                if c <= x:
                    dp[x] = min(dp[x], dp[x - c] + 1)
        return dp[amount] if dp[amount] < inf else -1
```

### 写法二：原笔记写法（`10**9` 作 INF）

```python
class Solution:
    def coinChange(self, coins: list[int], amount: int) -> int:
        dp = [0] + [10**9] * amount
        for x in range(1, amount + 1):
            for c in coins:
                if c <= x:
                    dp[x] = min(dp[x], dp[x - c] + 1)
        return dp[amount] if dp[amount] < 10**9 else -1
```

### 写法三：先枚举硬币（完全背包另一种循环顺序）

与写法一等价（本题求最值，两种常见顺序均可）。

```python
class Solution:
    def coinChange(self, coins: list[int], amount: int) -> int:
        inf = amount + 1
        dp = [inf] * (amount + 1)
        dp[0] = 0
        for c in coins:
            for x in range(c, amount + 1):
                dp[x] = min(dp[x], dp[x - c] + 1)
        return dp[amount] if dp[amount] < inf else -1
```

### 写法四：BFS（最短路）

```python
from collections import deque


class Solution:
    def coinChange(self, coins: list[int], amount: int) -> int:
        if amount == 0:
            return 0
        dist = [-1] * (amount + 1)
        dist[0] = 0
        q = deque([0])
        while q:
            u = q.popleft()
            for c in coins:
                v = u + c
                if v > amount or dist[v] != -1:
                    continue
                dist[v] = dist[u] + 1
                if v == amount:
                    return dist[v]
                q.append(v)
        return -1
```

## ⚠️ 常见陷阱

1. **`amount == 0`**  
   - 应返回 **`0`**，不是 `-1`。

2. **用 `0` 表示不可达**  
   - 会与「0 枚凑出 0」混淆；不可达应用 **INF** 或 **-1 距离未访问**（BFS）。

3. **01 背包顺序写反**  
   - 若错误地写成「先金额再硬币但内层从大到小」等，可能把每枚硬币限制为只用一次；本题是**完全背包**，需保证 `dp[x-c]` 已包含多次使用同一 `c` 的情况（写法一、三均正确）。

4. **与「方案数」混淆**  
   - [518. 零钱兑换 II](https://leetcode.cn/problems/coin-change-2/) 求**组合数**，转移是**求和**而非 `min`，模型不同。

## 复杂度分析

- **DP**：时间 **O(n × amount)**，`n = len(coins)`；空间 **O(amount)**。  
- **BFS**：最坏同样要扩展大量状态，与 **O(n × amount)** 同量级理解即可。

## 总结

322 是**完全背包求最小件数**：`dp[x] = min_c(dp[x-c]+1)`，边界 `dp[0]=0`，不可达输出 `-1`。面试主写 **DP**；追问「别的做法」可答 **BFS 最短路**。与 518 区分：**最少数** vs **方案数**。
