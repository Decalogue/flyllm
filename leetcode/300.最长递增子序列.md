# 300. 最长递增子序列

**标签**：`数组` `二分查找` `动态规划` `中等`  
**分类**：线性 DP、耐心排序、tails 数组  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你一个整数数组 `nums`，返回其中**最长严格递增子序列**的长度。

- **子序列**：由原数组删除若干元素（可不删）得到，**不改变剩余元素的相对顺序**。  
- **严格递增**：子序列中后一项必须**严格大于**前一项（相等不算递增）。

### 示例 1

```
输入：nums = [10,9,2,5,3,7,101,18]
输出：4
解释：最长递增子序列是 [2,3,7,101]，长度为 4。
```

### 示例 2

```
输入：nums = [0,1,0,3,2,3]
输出：4
```

### 示例 3

```
输入：nums = [7,7,7,7,7]
输出：1
```

### 提示

- `1 <= nums.length <= 2500`
- `-10^4 <= nums[i] <= 10^4`

**进阶**：能否用 **O(n log n)** 时间复杂度求解？

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结（二分）**：维护数组 `tails`，`tails[k]` 表示长度为 **`k+1`** 的递增子序列的**最小末尾元素**；对每个 `x` 用 **`bisect_left(tails, x)`** 找到插入位置，能接在后面则更新或扩展 `tails`，最后长度为答案。

**一句话总结（DP）**：`dp[i]` 表示**以 `nums[i]` 结尾**的最长递增子长度；`dp[i] = max(dp[j]+1)`，其中 `j < i` 且 `nums[j] < nums[i]`。

**关键词**：`tails` `bisect_left` `严格递增` `O(n log n)` / `O(n^2) DP`

### 复杂度速记

- **DP**：时间 **O(n²)**，空间 **O(n)**。  
- **二分 + tails**：时间 **O(n log n)**，空间 **O(n)**。

### 记忆口诀

「以 i 结尾看左边更小；tails 保长度 k 的最小尾巴，二分插。」

---

## 解题思路

### 与 [491. 非递减子序列](491.非递减子序列.md) 的区别

- **300**：只问**长度**，且要**严格递增**（`>`）。  
- **491**：枚举**所有**合法子序列，且为**非递减**（`>=`）。模型与复杂度都不同。

### 方法一：O(n²) DP

- **`dp[i]`**：以 `nums[i]` **结尾**的最长递增子序列长度。  
- **转移**：`dp[i] = 1 + max(dp[j] | j < i 且 nums[j] < nums[i])`，若没有更小的 `j` 则取 `1`。  
- **答案**：`max(dp)`。

正确性：任何以 `i` 结尾的递增序列，其倒数第二个元素必是某个 `j < i` 且 `nums[j] < nums[i]`，故可归纳。

### 方法二：O(n log n) — `tails` + 二分

维护递增数组 `tails`（实际是「各长度下的最小末尾」）：

- 遍历 `x = nums[i]`，在 `tails` 上 **`bisect_left(tails, x)`** 得位置 `pos`。  
- 若 `pos == len(tails)`，说明 `x` 比当前所有末尾都大，**最长长度 +1**，`tails.append(x)`。  
- 否则 **`tails[pos] = x`**：同样长度下用更小的末尾，有利于后面接上更多数。

**为何用 `bisect_left`**：本题要**严格递增**，相等不能接在同一序列末尾；`left` 会把 `x` 放到第一个 `>= x` 的位置，相等时替换而不是延长。

若题目改为**非递减** LIS，应改用 **`bisect_right`**。

### 「耐心排序」直观

把 `nums` 想象成发牌：每堆顶只能放比顶小的牌（递增堆的建模与 tails 对偶）。堆数等于 LIS 长度——与 `tails` 长度一致。

## 代码实现

### 写法一：tails + 二分（推荐，满足进阶）

```python
import bisect


class Solution:
    def lengthOfLIS(self, nums: list[int]) -> int:
        tails: list[int] = []
        for x in nums:
            i = bisect.bisect_left(tails, x)
            if i == len(tails):
                tails.append(x)
            else:
                tails[i] = x
        return len(tails)
```

### 详解：`tails` 与 `bisect_left`（写法一主循环）

#### `tails` 表示什么

- `tails` 在算法进行过程中**始终按值递增排序**（可当作有序数组用二分）。  
- **`tails[k]`** 的语义：在所有长度恰好为 **`k+1`** 的**严格递增**子序列里，**最后一个元素能取到的最小值**。  
- **`len(tails)`** 就是当前能构造出的**最长**递增子序列的长度（最终答案）。

直觉：同样都是「长度为 L 的递增链」，末尾越小，后面越容易接上更大的数，所以只保留**最小末尾**就够了。

#### `bisect.bisect_left(tails, x)` 在做什么

Python 里 `bisect_left(a, x)` 假设 `a` **已升序**，返回下标 **`i`**，满足：

- 把 `x` 插在 **`i` 这个位置**时，插入后序列仍升序；  
- 等价描述：**`i` 是第一个满足 `tails[i] >= x` 的位置**；若所有元素都 `< x`，则 **`i = len(tails)`**。

因此：

- **`i == len(tails)`**：说明 `x` 比当前所有末尾都大，可以接在**最长链**后面 → **最长长度 +1**，`append(x)`。  
- **否则**：存在长度为 **`i+1`** 的链，其当前最小末尾是 `tails[i]`，且 `tails[i] >= x`。用 **`x` 替换 `tails[i]`** 后，仍是长度 `i+1` 的链，但末尾**更小或相等**，不劣于原来，且**严格递增**要求下用 `x` 替换相等或更大的末尾是正确的更新。

#### 为何本题用 `bisect_left` 而不是 `bisect_right`

本题要**严格递增**：新元素不能和链尾**相等**。  
- `bisect_left` 找到第一个 **`>= x`** 的位置：若链尾已有 `x`，会落在那个位置并把旧值换成 `x`（相等时相当于不延长长度），**不会**错误地把长度加一。  
- 若误用 `bisect_right`（第一个 **`> x`**），当 `tails` 里已有 `x` 时可能插到更后，语义变成「非递减」链的维护，与**严格** LIS 不一致。

#### 逐行对应

| 代码 | 含义 |
|------|------|
| `for x in nums` | 按原数组顺序处理（子序列必须保序） |
| `i = bisect_left(tails, x)` | `x` 应作为「长度为 `i+1` 的链」的新末尾候选位置 |
| `append` | `x` 延长最长链 |
| `tails[i] = x` | 在长度 `i+1` 上收紧为更小的末尾 |

#### 极简走表（`nums = [3, 1, 2]`）

| `x` | `tails` 前 | `i` | 操作后 `tails` |
|-----|------------|-----|----------------|
| 3 | `[]` | 0 | `[3]` |
| 1 | `[3]` | 0 | `[1]` |
| 2 | `[1]` | 1 | `[1,2]` |

答案 `len(tails) = 2`（如子序列 `[1,2]`）。

---

### 写法二：O(n²) DP

```python
class Solution:
    def lengthOfLIS(self, nums: list[int]) -> int:
        n = len(nums)
        dp = [1] * n
        for i in range(1, n):
            for j in range(i):
                if nums[j] < nums[i]:
                    dp[i] = max(dp[i], dp[j] + 1)
        return max(dp)
```

## ⚠️ 常见陷阱

1. **非递减 vs 严格递增**  
   - 300 是**严格**递增；`tails` 解法必须用 **`bisect_left`**。非递减版本用 **`bisect_right`**。

2. **`tails` 不是某条真实 LIS**  
   - `tails` 只保证长度正确，一般**不能**直接当最长子序列输出；要还原路径需额外记录前驱。

3. **空数组**  
   - 题目 `n >= 1`，无需特判；若扩展题允许空串，应返回 `0`。

4. **把贪心当万能**  
   - 「每次选最小能接的」之类不严谨的贪心易错；`tails` 维护的是**各长度最小末尾**，有完整证明。
