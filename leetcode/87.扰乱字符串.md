# 87. 扰乱字符串

**标签**：`字符串` `递归` `记忆化搜索` `动态规划` `困难`  
**分类**：字符串、分治、区间 DP  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥 低频

---

## 题目描述

使用下面描述的算法可以扰乱字符串 `s` 得到字符串 `t`：

1. 如果字符串的长度为 1，算法停止。
2. 如果字符串的长度大于 1，执行下列操作：
   - 在一个随机下标处将字符串分割成两个非空子字符串，记为 `x` 和 `y`。即 `s = x + y`（`+` 表示拼接）。
   - 随机决定是要「交换两个子字符串」还是「保持这两个子字符串的顺序不变」。即执行 `s = y + x` 或保持 `s = x + y`。
   - 对 `x` 和 `y` 继续执行这个算法。

给你两个长度相同的字符串 `s1` 和 `s2`，判断是否 **存在** 一种扰乱方案使 `s2` 是 `s1` 的扰乱结果。

### 示例 1

```
输入：s1 = "great", s2 = "rgeat"
输出：true
解释：一种可能方案 —— great → gr/eat（分割）→ 保持 → 子串递归扰乱后可得 rgeat。
```

### 示例 2

```
输入：s1 = "abcde", s2 = "caebd"
输出：false
```

### 示例 3

```
输入：s1 = "a", s2 = "a"
输出：true
```

### 提示

- `s1.length == s2.length`
- `1 <= s1.length <= 30`
- `s1` 和 `s2` 由小写英文字母组成

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：按长度递归（或区间 DP）枚举每一种「分割 + 是否交换」的组合；务必先用 **字符 multiset** 剪枝。

**关键词**：`分治` `递归树` `两种配对方式` `排序/频数剪枝` `记忆化`

### 复杂度速记

- **时间复杂度**：最坏指数级，剪枝 + 记忆化后实际可过 \(n \le 30\)；区间 DP 为 \(O(n^4)\) 量级（常数与实现相关）。
- **空间复杂度**：记忆化栈与缓存 \(O(n^3)\) 条目量级（与实现相关）。

### 记忆口诀

「先比 multiset，相等再看分；左对左、右对右；或左对尾、右对头。」

### 代码模板（递归 + 记忆化）

```python
@cache
def dfs(a, b):
    if a == b:
        return True
    if Counter(a) != Counter(b):
        return False
    n = len(a)
    for i in range(1, n):
        if dfs(a[:i], b[:i]) and dfs(a[i:], b[i:]):
            return True
        if dfs(a[:i], b[n - i :]) and dfs(a[i:], b[: n - i]):
            return True
    return False
```

## 解题思路

### 核心思想

扰乱过程在任意一步都是：**把当前串切成非空两段**，对两段**独立**继续扰乱，然后拼起来；拼的时候这两段的**左右顺序**可以**交换**也可以**不换**。

因此判断 `s1` 能否扰乱成 `s2` 等价于：存在某个分割长度 \(i\)（\(1 \le i < n\)），使得下面两种情况之一成立：

1. **不交换整体左右块**：  
   `s1[:i]` 扰乱成 `s2[:i]`，且 `s1[i:]` 扰乱成 `s2[i:]`。
2. **交换整体左右块**（原 `s1` 的左块对应扰乱后 `s2` 的**右**块）：  
   `s1[:i]` 扰乱成 `s2[n-i:]`，且 `s1[i:]` 扰乱成 `s2[:n-i]`。

边界：`n == 1` 时当且仅当 `s1 == s2`。

### 必要剪枝

- 若 `sorted(s1) != sorted(s2)`（或用 `Counter`），则两串字符 multiset 不同，**直接 false**。这是面试里必说的优化。
- 若 `s1 == s2`，**直接 true**。

### 算法步骤（递归）

1. 若 `s1 == s2`，返回 `True`。
2. 若 `Counter(s1) != Counter(s2)`，返回 `False`。
3. 令 `n = len(s1)`，枚举分割 \(i = 1, \dots, n-1\)：
   - 若 `isScramble(s1[:i], s2[:i])` 且 `isScramble(s1[i:], s2[i:])`，返回 `True`。
   - 若 `isScramble(s1[:i], s2[n-i:])` 且 `isScramble(s1[i:], s2[:n-i])`，返回 `True`。
4. 枚举结束返回 `False`。

对 `(s1, s2)` 做 **记忆化**（或改为区间 DP），避免重复子问题。

### 区间 DP 思路（可选）

令 `dp[len][i][j]` 表示 `s1[i:i+len]` 是否能扰乱成 `s2[j:j+len]`。  
枚举分割 `k`（左段长度），同样分「不交换 / 交换」两种转移。适合不喜欢写超长字符串切片的场景。

### 为什么选择这个方法？

- **优势**：直接对齐题意的递归结构，最易讲清「交换」对应的下标配对。
- **劣势**：裸递归指数爆炸，必须通过 **频数剪枝 + 记忆化**（或区间 DP）控制复杂度。

## 代码实现

### 方法一：记忆化搜索（推荐 ⭐）

```python
from collections import Counter
from functools import cache


class Solution:
    def isScramble(self, s1: str, s2: str) -> bool:
        @cache
        def dfs(a: str, b: str) -> bool:
            if a == b:
                return True
            if Counter(a) != Counter(b):
                return False
            n = len(a)
            for i in range(1, n):
                # 不交换：左对左、右对右
                if dfs(a[:i], b[:i]) and dfs(a[i:], b[i:]):
                    return True
                # 交换：s1 左 i 对应 s2 右 i 段
                if dfs(a[:i], b[n - i :]) and dfs(a[i:], b[: n - i]):
                    return True
            return False

        return dfs(s1, s2)
```

**说明**：`Counter` 每次新建有开销；面试可改为「26 字母频次元组」或「排序字符串」作为缓存键的一部分，或单独写 `same_freq(a,b)`。

### 方法二：区间 DP（O(n⁴) 量级）

```python
class Solution:
    def isScramble(self, s1: str, s2: str) -> bool:
        n = len(s1)
        if sorted(s1) != sorted(s2):
            return False

        # dp[l][i][j]: s1[i:i+l] 能否扰乱成 s2[j:j+l]
        dp = [[[False] * n for _ in range(n)] for _ in range(n + 1)]
        for i in range(n):
            for j in range(n):
                dp[1][i][j] = s1[i] == s2[j]

        for length in range(2, n + 1):
            for i in range(n - length + 1):
                for j in range(n - length + 1):
                    for k in range(1, length):
                        if dp[k][i][j] and dp[length - k][i + k][j + k]:
                            dp[length][i][j] = True
                            break
                        if dp[k][i][j + length - k] and dp[length - k][i + k][j]:
                            dp[length][i][j] = True
                            break

        return dp[n][0][0]
```

## ⚠️ 常见陷阱

1. **只判断字符集合是否相同，忽略次数** —— 必须用 multiset（`Counter` / 排序 / 频次数组）。
2. **交换时子串下标写反** —— 左段长度 \(i\) 时，与 `s2` 配对的是 `s2[n-i:]` 与 `s2[:n-i]`，长度分别是 \(i\) 与 \(n-i\)。
3. **忘记剪枝** —— 无剪枝的纯递归在大数据上会超时。
4. **记忆化键用列表** —— 需用不可变类型；对整段子串做 `cache` 最简单，也可改为 `(i, j, len)` 三维状态。

## 总结

- 本质是 **分治 + 两种拼接方式**（对应是否交换左右块）。
- **频数 / 排序剪枝** 与 **记忆化** 是过题与面试表达的重点。
- 掌握递归版本即可应对大部分面试；区间 DP 可作为「非递归」补充写法。
