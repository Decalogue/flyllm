## 题目描述

给定一个整数数组 `temperatures`，表示每天的温度，返回一个数组 `answer`，其中 `answer[i]` 表示对于第 i 天，要等几天才会出现更高的温度。若之后都不会升高，则该位置为 0。

### 示例 1

```
输入：temperatures = [73,74,75,71,69,72,76,73]
输出：[1,1,4,2,1,1,0,0]
解释：第 0 天 73，第 1 天 74 更高 → 等 1 天；第 2 天 75，第 6 天 76 更高 → 等 4 天；以此类推。
```

### 示例 2

```
输入：temperatures = [30,40,50,60]
输出：[1,1,1,0]
```

### 示例 3

```
输入：temperatures = [30,60,90]
输出：[1,1,0]
```

### 提示

- 1 <= temperatures.length <= 10^5
- 30 <= temperatures[i] <= 100

## 📋 面试要点速查（知识卡片）

### 核心思路
**一句话总结**：对每个位置 i，找**右侧第一个比 `temperatures[i]` 大的位置** j，则 `answer[i] = j - i`；用**单调递减栈**（栈内温度从底到顶递减）一次遍历，遇到更大温度就弹栈并记录天数差。

**关键词**：`单调栈` `右侧第一个更大` `下一个更高温度`

### 复杂度速记
- **时间复杂度**：O(n) - 每个下标最多入栈、出栈各一次
- **空间复杂度**：O(n) - 单调栈

### 记忆口诀
「单调递减栈，遇大就弹栈；弹栈时记答案，天数差 i - 栈顶。」

### 代码模板（可直接套用）

```python
# 单调递减栈：栈内存下标，栈内对应温度单调递减
# 当前温度比栈顶大 → 栈顶的「下一个更高温度」就是当前天，answer[栈顶] = i - 栈顶
stack = []
res = [0] * len(temperatures)
for i in range(len(temperatures)):
    while stack and temperatures[stack[-1]] < temperatures[i]:
        idx = stack.pop()
        res[idx] = i - idx
    stack.append(i)
return res
```

## 解题思路

### 核心思想

对每一天 i，需要找到**右边第一个比 `temperatures[i]` 大的位置** j，则 `answer[i] = j - i`；若不存在则为 0。这是典型的「**下一个更大元素**」问题。

**关键点：**
- **单调递减栈**：栈内存下标，从栈底到栈顶对应的温度**单调递减**。当遇到 `temperatures[i]` 比栈顶大时，说明栈顶那天找到了「下一个更高温度」就是第 i 天，弹栈并设 `answer[栈顶] = i - 栈顶`。
- 初始时 `answer` 全为 0，遍历结束后仍留在栈里的下标表示之后没有更高温度，保持 0 即可，无需哨兵。

### 算法步骤

1. 初始化 `answer = [0] * n`。
2. 维护**单调递减栈**（存下标），从左到右遍历 `temperatures`。
3. 当 `temperatures[i] > temperatures[stack[-1]]` 时：
   - 栈顶下标 `idx` 的「下一个更高温度」为第 i 天，`answer[idx] = i - idx`。
   - 弹栈，重复直到栈为空或栈顶对应温度 ≥ `temperatures[i]`。
4. 将 i 入栈。
5. 遍历结束后返回 `answer`。

### 为什么选择这个方法？

- **优势**：一次遍历 O(n) 为每个位置找到右侧第一个更大，避免暴力对每个 i 向右扫。
- **适用场景**：「下一个更大/更小元素」、区间最值相关。
- **替代方案**：暴力对每个 i 向右找第一个更大，O(n²)；或从右往左用栈预处理「下一个更大」数组再填答案，多一轮但思路直观。

## 代码实现

### 单调栈（推荐 ⭐）

```python
from typing import List

class Solution:
    def dailyTemperatures(self, temperatures: List[int]) -> List[int]:
        """
        单调递减栈：栈内温度从底到顶递减，遇到更大温度则栈顶找到「下一个更高」的天数差。
        时间复杂度: O(n)
        空间复杂度: O(n)
        """
        stack = []
        res = [0] * len(temperatures)
        for i in range(len(temperatures)):
            while stack and temperatures[stack[-1]] < temperatures[i]:
                idx = stack.pop()
                res[idx] = i - idx
            stack.append(i)
        return res
```

**代码要点**：
- ✅ 栈内下标对应温度单调递减，遇到更大就弹栈并填 `res[idx] = i - idx`。
- ✅ `res` 初始为 0，未弹出的下标自然保持 0（之后没有更高温度）。
- ✅ 先写 `res[stack[-1]]` 再 `pop`，顺序不要反。

### 执行过程示例

以 `temperatures = [73,74,75,71,69,72,76,73]` 为例：

- i=0：栈 [0]。
- i=1：74 > 73，弹 0，res[0]=1；栈 [1]。
- i=2：75 > 74，弹 1，res[1]=1；栈 [2]。
- i=3,4：栈 [2,3]、[2,3,4]。
- i=5：72 > 69，弹 4，res[4]=1；72 > 71，弹 3，res[3]=2；栈 [2,5]。
- i=6：76 > 72，弹 5，res[5]=1；76 > 75，弹 2，res[2]=4；栈 [6]。
- i=7：73 不大于 76，栈 [6,7]。结束后 res = [1,1,4,2,1,1,0,0]。

## ⚠️ 常见陷阱（面试易错点）

1. **弹栈时先填答案再 pop**
   - ❌ 先 `stack.pop()` 再 `res[pop_val]=...` 会用到错误的索引（pop 后栈顶已变）。
   - ✅ 先用 `idx = stack[-1]` 或先 `res[stack[-1]] = i - stack[-1]` 再 `stack.pop()`。

2. **比较方向**
   - ❌ 写成「栈顶大于当前」才弹栈，会变成找「下一个更小」。
   - ✅ 找**下一个更高**：当前 **大于** 栈顶时弹栈，即 `temperatures[i] > temperatures[stack[-1]]`。

3. **边界情况**
   - 空数组：不进入循环，返回 []（若题目保证 n>=1 可忽略）。
   - 单元素：栈 [0]，无弹栈，res = [0]，正确。

## 复杂度分析

### 时间复杂度
- 每个下标最多入栈一次、出栈一次，总操作 O(n)。

### 空间复杂度
- 单调栈最多 O(n) 个下标。

### 复杂度对比表

| 方法           | 时间复杂度 | 空间复杂度 | 备注     |
|----------------|------------|------------|----------|
| 单调栈（推荐） | O(n)       | O(n)       | 一次遍历 |
| 暴力           | O(n²)      | O(1)       | 每个 i 向右扫 |

## 关键点

1. **为什么是单调「递减」栈？**
   - 我们要找的是「右边第一个**更大**」：当当前温度比栈顶大，栈顶才找到答案并弹栈；栈内保持递减，才能保证每次弹栈时当前 i 就是栈顶的「下一个更高」位置。

2. **与 84 题的区别**
   - 84：找「左右第一个**更小**」→ 单调**递增**栈，且需哨兵。
   - 739：找「右边第一个**更大**」→ 单调**递减**栈，答案未弹出的默认为 0，不需哨兵。

## 💬 面试问题（模拟面试场景）

### 基础问题

1. **Q: 这道题的核心思路是什么？**
   - A: 对每一天 i 找右边第一个比当天温度高的位置 j，answer[i] = j - i；用单调递减栈，遇到更大温度就弹栈并记录天数差。

2. **Q: 为什么用单调递减栈？**
   - A: 栈内温度递减，遇到更大的温度时，当前下标就是栈顶的「下一个更高温度」所在天，弹栈并填 answer[栈顶] = i - 栈顶即可。

3. **Q: 需要哨兵吗？**
   - A: 不需要。answer 初始为 0，遍历结束后还在栈里的下标表示之后没有更高温度，保持 0 即可。

### 进阶问题

1. **Q: 若要求「左边第一个更大」怎么做？**
   - A: 从左往右扫时用单调递减栈，弹栈时当前 i 是栈顶的「右边第一个更大」；若改为「左边第一个更大」，可从右往左扫，或从左往右扫时在弹栈时记录「栈顶的左边第一个更大」为当前栈顶的下一个元素（需在弹栈前看新栈顶）。
