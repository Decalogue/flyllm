# 1. 两数之和

**标签**：`数组` `哈希表` `简单`  
**分类**：数组与字符串（Hot 100）  
**难度**：⭐ 简单  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给定一个整数数组 `nums` 和一个整数目标值 `target`，请你在该数组中找出**和为目标值 `target`** 的那**两个**整数，并返回它们的**数组下标**。

你可以假设每种输入只会对应一个答案。但是，数组中**同一个元素在答案里不能重复出现**。

你可以按任意顺序返回答案。

### 示例 1

```
输入：nums = [2,7,11,15], target = 9
输出：[0,1]
解释：因为 nums[0] + nums[1] == 9 ，返回 [0, 1] 。
```

### 示例 2

```
输入：nums = [3,2,4], target = 6
输出：[1,2]
```

### 示例 3

```
输入：nums = [3,3], target = 6
输出：[0,1]
```

### 提示

- `2 <= nums.length <= 10^4`
- `-10^9 <= nums[i] <= 10^9`
- `-10^9 <= target <= 10^9`
- **只会存在一个有效答案**

### 进阶

你可以想出一个时间复杂度小于 `O(n^2)` 的算法吗？

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：一边扫数组，一边用**哈希表**记下「值 → 下标」；当前数为 `x` 时查表里是否已有 `target - x`。

**关键词**：`一次遍历` `补数 complement` `值不能重复用同一索引`

### 复杂度速记

- **时间**：O(n)  
- **空间**：O(n) 哈希表

### 记忆口诀

「当前缺谁？表里找补数；找不到就先记下自己。」

---

## 解题思路

### 核心思想

对下标 `i`，需要找是否存在 `j < i` 使得 `nums[j] + nums[i] == target`，即 `nums[j] == target - nums[i]`。把已扫描过的 `nums[j]` 放进哈希表，即可 O(1) 查询。

### 算法步骤

1. 初始化空字典 `seen`。  
2. 枚举 `i, x = nums[i]`：  
   - 若 `target - x` 在 `seen` 中，返回 `[i, seen[target - x]]`（顺序任意）。  
   - 否则 `seen[x] = i`。  
3. 题目保证有解，循环内必返回。

### 为何「先查后存」？

若先存当前再查，会误用**同一索引**两次（例如 `[3,3]`、`target=6`）。必须先查补数，再把当前值写入表。

---

## 代码实现

### 哈希表（推荐）

```python
class Solution:
    def twoSum(self, nums: list[int], target: int) -> list[int]:
        seen: dict[int, int] = {}
        for i, x in enumerate(nums):
            need = target - x
            if need in seen:
                return [i, seen[need]]
            seen[x] = i
        return []
```

### 执行过程示例

以 `nums = [2, 7, 11, 15]`, `target = 9` 为例：

```
i=0, x=2:
  need = 7，不在 seen
  seen = {2: 0}

i=1, x=7:
  need = 2，在 seen，seen[2]=0
  返回 [1, 0]
```

---

## ⚠️ 常见陷阱

1. **先写入再查找**  
   - 会导致同一元素被用两次，违反题意。

2. **返回值顺序**  
   - 题目允许任意顺序，`[i, j]` 与 `[j, i]` 均可。

3. **暴力 O(n²)**  
   - 能通过小数据，面试应主动给出哈希 O(n)。

---

## 复杂度分析

| 方法 | 时间 | 空间 |
|------|------|------|
| 哈希一次遍历 | O(n) | O(n) |
| 暴力两重循环 | O(n²) | O(1) |

---

## 关键点

1. **哈希表键**：数值；**值**：首次出现的下标。  
2. 本题**保证唯一解**；变体可能有多组解或无解，需改返回约定。

## 其他方法（暴力，不推荐）

```python
class Solution:
    def twoSum(self, nums: list[int], target: int) -> list[int]:
        n = len(nums)
        for i in range(n):
            for j in range(i + 1, n):
                if nums[i] + nums[j] == target:
                    return [i, j]
        return []
```

## 总结

两数之和是**哈希表降低查找复杂度**的入门题：把「找另一个数」变成「找补数是否在已访问集合里」，一遍扫描即可，也是数组与字符串类题的常见套路起点。
