# 141. 环形链表

**标签**：`链表` `双指针` `哈希表` `简单`  
**分类**：链表、快慢指针（Floyd 判环）  
**难度**：⭐ 简单  
**频率**：🔥🔥🔥 高频  

---

## 题目描述

给你一个链表的头节点 `head`，判断链表中**是否存在环**。

- 若存在环，返回 `true`；否则返回 `false`。  
- 环的定义：链表中某个节点的 `next` 指针指向**链表前面**的某个节点（或自己），使链表形成环。

说明：不允许修改链表；额外使用 **O(1)** 内存是常见追问（对应 Floyd 解法）。

### 示例 1

```
3 → 2 → 0 → -4
        ↑     |
        └─────┘

存在环，返回 true。
```

### 示例 2

```
1 → 2 → null

无环，返回 false。
```

### 示例 3

```
1 → 1（自环）

存在环，返回 true。
```

### 提示

- 链表中节点数目范围是 `[0, 10^4]`。  
- `-10^5 <= Node.val <= 10^5`  
- `pos` 为环起始下标（`-1` 表示无环），测试用构造器常用。

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：**快慢指针**同起点出发，`fast` 每次两步、`slow` 每一步；若有环必在环上**相遇**，无环则 `fast` 先走到空。

**关键词**：`Floyd 判环` `slow/fast` `追及` `线性时间与常数空间`

### 复杂度速记

| 方法 | 时间 | 空间 |
|------|------|------|
| Floyd（双指针） | O(n) | O(1) |
| 哈希记录已访问结点 | O(n) | O(n) |

### 记忆口诀

「快指针追慢指针，进环必相撞；链表到头没撞上就没环。」

---

## 解题思路

### 核心思想

1. `slow`、`fast` 均从 `head` 出发。  
2. 每轮：`slow = slow.next`，`fast = fast.next.next`。  
3. **有环**：`fast` 最终会进入环并「套圈」追上 `slow`，出现 `slow is fast`。  
4. **无环**：链表有限，`fast` 或 `fast.next` 会为 `None`，循环结束返回 `false`。

### 为什么有环一定会相遇？

进入环后，`fast` 相对 `slow` 每轮**追近 1 步**（速度差），环长有限，故**必然在有限步内相遇**。无环时 `fast` 无法无限前进而不掉出链表，故不会与 `slow` 在「结点」上无限错开——只会提前结束。

### 方法对比

| 方法 | 做法 | 面试怎么说 |
|------|------|------------|
| **Floyd** | 快慢指针，相遇即有环 | 空间 O(1)，141/142 同款框架 |
| **哈希 / 集合** | 遍历看 `node` 是否出现过 | 好理解，需 O(n) 额外空间 |

### 易错点

- 循环条件写 **`while fast and fast.next`**，避免 `fast.next` 为空时再访问 `fast.next.next`。  
- 用 **`is`** 判断结点对象是否同一引用，不要用 `==`（一般 `ListNode` 未重载相等时也可，但面试里明确「结点引用」更稳）。  
- **不要**只移动 `fast` 不检查：每轮应在移动**之后**或按标准模板在移动中检查相遇。

### 进阶：找环入口

本题只问**有没有环**。若要返回**入环第一个结点**，见 **[142. 环形链表 II](./142.环形链表 II.md)**（第二阶段：`p = head` 与 `slow` 同速再走直到相遇）。

## 代码实现

### 方法一：Floyd 快慢指针（⭐ 面试首选）

```python
class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None


class Solution:
    def hasCycle(self, head: ListNode | None) -> bool:
        slow = fast = head
        while fast and fast.next:
            slow, fast = slow.next, fast.next.next
            if slow is fast:
                return True
        return False
```

**要点**：

- 起点都设为 `head`，不要预读一步（否则边界易错）。  
- 相遇即 `True`；出循环则 `False`。

### 方法二：哈希表

```python
class ListNode:
    def __init__(self, x):
        self.val = x
        self.next = None


class Solution:
    def hasCycle(self, head: ListNode | None) -> bool:
        seen: set[ListNode] = set()
        while head:
            if head in seen:
                return True
            seen.add(head)
            head = head.next
        return False
```

## 复杂度

- **Floyd**：时间 **O(n)**（有环时上界与头到环距离及环长有关，仍为线性量级），空间 **O(1)**。  
- **哈希**：时间 **O(n)**，空间 **O(n)**。
