# 191. 位 1 的个数

**标签**：`位运算` `简单`  
**分类**：位技巧、汉明重量  
**难度**：⭐ 简单  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给定一个无符号整数 `n`，统计其**二进制表示中数位为 `1` 的个数**（也称为[汉明重量](https://zh.wikipedia.org/wiki/%E6%B1%89%E6%98%8E%E9%87%8D%E9%87%8F)）。

### 示例 1

```
输入：n = 11（二进制 00000000000000000000000000001011）
输出：3
```

### 示例 2

```
输入：n = 128（二进制 00000000000000000000000010000000）
输出：1
```

### 示例 3

```
输入：n = 2147483644（0x7FFFFFFC，含 30 个 1）
输出：30
```

### 提示

- 输入为**长度为 32** 的二进制串（力扣常见为 32 位无符号语义）。

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：反复执行 `n &= n - 1`，每次消掉**最低位的 1**，消几次就有几个 1。

**关键词**：`n & (n-1)` `lowbit` `O(1) 单次` `O(k) k=1的个数`

### 复杂度速记

- **时间**：O(k)，k 为二进制中 1 的个数，最坏 O(32) 即 O(1)（固定位宽）。  
- **空间**：O(1)。

### 记忆口诀

「`n-1` 借位翻转尾部，`n` 与一下少一个 1。」

---

## 解题思路

### 核心思想

对 `n > 0`，`n - 1` 会把 `n` 的**最低位的 1** 变成 0，并将其右侧全部变成 1。因此：

`n & (n - 1)` 的结果恰好**去掉了最低位的那个 1**。

循环直到 `n == 0`，循环次数即为 1 的个数。

### 为何正确？

每次操作严格减少一个 1，且不会误删高位，故计数等于原数中 1 的个数。

### 其他写法（了解即可）

- **逐位检查**：`while n: c += n & 1; n >>= 1`（在部分语言需注意无符号右移）。  
- **查表 / 内置**：生产环境可用；面试优先写位运算技巧。

---

## 代码实现

### 写法一：`n & (n - 1)`（推荐）

```python
class Solution:
    def hammingWeight(self, n: int) -> int:
        c = 0
        while n:
            n &= n - 1
            c += 1
        return c
```

### 写法二：逐位

```python
class Solution:
    def hammingWeight(self, n: int) -> int:
        c = 0
        for _ in range(32):
            c += n & 1
            n >>= 1
        return c
```

**说明**：力扣 Python 中 `n` 按题目给定范围处理；若语言里存在**有符号**与**算术右移**问题，需用无符号右移（本题 Python 写法二在 32 次循环下通常与题意一致）。

---

## ⚠️ 常见陷阱

1. **负数**  
   - Java/C++ 等需注意 `int` 是否为补码、是否要用 `>>>`。Python 本题一般按无符号 32 位理解即可。

2. **与 338 混淆**  
   - [338. 比特位计数](338.比特位计数.md) 要对 `0..n` **每个数**都数 1，用 DP；本题只数**一个** `n`，用 `n & (n-1)` 或循环 32 位即可。

3. **死循环**  
   - `while n` 在 `n` 最终会变 0；若错误写成 `n >>= 1` 但未限制位数，有符号负数可能出问题——固定位数循环更安全。

## 复杂度分析

| 方法 | 时间 | 空间 |
|------|------|------|
| `n & (n-1)` | O(k)，k≤32 | O(1) |
| 固定 32 次移位 | O(32)=O(1) | O(1) |
