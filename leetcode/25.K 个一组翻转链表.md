# 25. K 个一组翻转链表

**标签**：`链表` `递归` `困难`  
**分类**：链表、双指针  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 中高频

---

## 题目描述

给你链表的头节点 `head`，请你**每 k 个节点一组**进行翻转，并返回修改后的链表。

- k 是正整数，且小于或等于链表长度。
- 若节点总数不是 k 的整数倍，**最后不足 k 个的节点保持原有顺序**。
- 不能只改节点值，需要**实际进行指针/节点交换**。

### 示例 1

```
输入：head = [1,2,3,4,5], k = 2
输出：[2,1,4,3,5]
```

### 示例 2

```
输入：head = [1,2,3,4,5], k = 3
输出：[3,2,1,4,5]
```

### 提示

- 链表中的节点数目为 n
- `1 <= k <= n <= 5000`
- `0 <= Node.val <= 1000`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**哑节点 + 分组反转**：用哑节点统一头边界；每轮用 `pre` 指向「当前分组前一个节点」，从 `pre` 走 k 步得到本组尾 `tail`，若不足 k 个则直接返回；记 `suf = tail.next`（下一组头），对 `[head, tail]` 做反转，再令 `pre.next = 新头`、`新尾.next = suf`，然后 `pre` 和 `head` 移到下一组起点继续。

**关键词**：`哑节点` `pre/tail/suf` `分组内反转再接好前后`

### 复杂度速记

- **时间**：O(n)——每个节点最多被遍历、被反转各一次。
- **空间**：O(1)——仅用常数额外指针（不含递归栈）。

### 记忆口诀

「哑节点当 pre，每轮先找 tail 再找 suf；反转 [head,tail] 后 pre 接新头、新尾接 suf，再推进到下一组。」

---

## 解题思路

### 核心思想

- 每 k 个为一组，组内做一次「206 式」反转；组与组之间用 `pre`（当前组前一个节点）和 `suf`（当前组下一个节点）接好。
- **找本组尾**：从 `pre` 出发走 k 步得到 `tail`；若中途遇到 `None`，说明不足 k 个，保持原样并返回。
- **反转区间**：对从 `head` 到 `tail` 的这段链表反转，返回新头、新尾（即原来的 `tail`、`head`）。
- **接好前后**：`pre.next = 新头`，`新尾.next = suf`；然后 `pre` 移到新尾，`head = pre.next` 作为下一组的头，继续循环。

### 算法步骤

1. 哑节点 `res`，`res.next = head`，`pre = res`。
2. 当 `head` 非空：
   - `tail = pre`，循环 k 次 `tail = tail.next`，若某次 `tail` 为 `None` 则不足 k 个，返回 `res.next`。
   - `suf = tail.next`（下一组第一个节点）。
   - 对 `[head, tail]` 做反转，得到新头、新尾（即 `tail`、`head` 互换角色）。
   - `pre.next = 新头`，`新尾.next = suf`。
   - `pre = 新尾`，`head = pre.next`，进入下一组。
3. 返回 `res.next`。

### reverse(head, tail) 的写法

与 206 相同逻辑：`pre = None`，`cur = head`，循环直到 `pre == tail`（即把 head 到 tail 都反转完），返回 `(tail, head)` 作为新头、新尾。

## 代码实现

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next

class Solution:
    def reverse(self, head: ListNode, tail: ListNode):
        """反转 [head, tail] 这一段，返回 (新头, 新尾)，即 (tail, head)。"""
        cur, pre = head, None
        while pre != tail:
            tmp = cur.next
            cur.next = pre
            pre = cur
            cur = tmp
        return tail, head

    def reverseKGroup(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:
        # 哑节点，便于统一处理首组；pre 为当前分组的前一个节点
        res = ListNode()
        res.next = head
        pre = res

        while head:
            tail = pre
            for _ in range(k):
                tail = tail.next
                if not tail:
                    return res.next  # 不足 k 个，保持原顺序
            suf = tail.next  # 下一组的第一个节点

            # 翻转本组 [head, tail]，得到新头、新尾
            head, tail = self.reverse(head, tail)

            # 接好：前一组的尾巴连本组新头，本组新尾连下一组头
            pre.next = head
            tail.next = suf

            # 移到下一组：pre 为本组新尾，head 为下一组头
            pre = tail
            head = pre.next

        return res.next
```

**代码要点**：

- ✅ `reverse` 的循环条件 `pre != tail` 保证恰好反转 [head, tail] 这一段，返回 `(tail, head)` 即新头、新尾。
- ✅ 先找 `tail` 再取 `suf = tail.next`，反转后用 `pre.next = head`、`tail.next = suf` 接好。
- ✅ 不足 k 个时直接 `return res.next`，最后一段保持不反转。

---

## 复杂度分析

| 项目 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 每个节点被遍历常数次，反转时每个节点被访问常数次。 |
| 空间 | O(1) | 仅哑节点和若干指针（若用递归实现 reverse 则栈 O(k)）。 |

---

## 易错点

- **找 tail 时判空**：`for _ in range(k)` 里每次 `tail = tail.next` 后要检查 `if not tail`，否则不足 k 个时会越界或死循环。
- **反转区间含义**：`reverse(head, tail)` 反转的是闭区间 [head, tail]，循环条件 `pre != tail` 表示要反转到 tail 为止（包含 tail）。
- **下一组起点**：反转后本组新尾是原来的 `head`，所以 `pre = tail` 实际是 `pre = 新尾`，`head = pre.next` 才是下一组的头（即原来的 `suf`）。
