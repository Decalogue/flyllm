# 169. 多数元素

**标签**：`数组` `哈希表` `分治` `摩尔投票` `简单`  
**分类**：数组与字符串（Hot 100）  
**难度**：⭐ 简单  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给定一个大小为 `n` 的数组 `nums`，返回其中的**多数元素**。

**多数元素**：在数组中出现次数**严格大于** `⌊ n/2 ⌋` 的元素。

你可以假设数组**非空**，且**总是存在**多数元素。

### 示例 1

```
输入：nums = [3,2,3]
输出：3
```

### 示例 2

```
输入：nums = [2,2,1,1,1,2,2]
输出：2
```

### 提示

- `n == nums.length`
- `1 <= n <= 5 * 10^4`
- `-10^9 <= nums[i] <= 10^9`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：**摩尔投票**：维护一个候选和计数；相同则 `+1`，不同则 `-1`；计数归零就换候选；因多数元素超过一半，最后剩下的候选即答案。

**关键词**：`Boyer-Moore 投票` `抵消` `O(n) O(1)`

### 复杂度速记

- **摩尔投票**：时间 O(n)，额外空间 O(1)。  
- **哈希计数**：时间 O(n)，空间 O(n)。

### 记忆口诀

「同伙加票，异己减票；票清零就换人，过半者站到最后。」

---

## 解题思路

### 摩尔投票（推荐）

把不同数值看成「互相对票」：当前候选与遍历值相同则支持票 `+1`，不同则 `-1`。若票数为 0，说明当前候选被「全抵消」，换一个候选从 1 票重新开始。

设真实众数为 `x`，出现 `m` 次，且 `m > n/2`。最坏情况下，每个 `x` 都与一个非 `x` 配对抵消，净剩票数至少 `m - (n - m) = 2m - n > 0`，故最终 `candidate` 必为 `x`。

### 其他思路

- **排序**：排序后下标 `n//2` 处必是众数；O(n log n)。  
- **哈希**：统计频次，找超过 `n//2` 的键；O(n) 空间。

---

## 代码实现

### 摩尔投票（题目保证存在众数）

```python
class Solution:
    def majorityElement(self, nums: list[int]) -> int:
        candidate = None
        count = 0
        for x in nums:
            if count == 0:
                candidate = x
                count = 1
            elif x == candidate:
                count += 1
            else:
                count -= 1
        return candidate
```

### 若不保证存在众数（需第二遍验证）

```python
class Solution:
    def majorityElement(self, nums: list[int]) -> int | None:
        candidate = None
        count = 0
        for x in nums:
            if count == 0:
                candidate = x
                count = 1
            elif x == candidate:
                count += 1
            else:
                count -= 1
        if candidate is None:
            return None
        cnt = sum(1 for x in nums if x == candidate)
        return candidate if cnt > len(nums) // 2 else None
```

---

## ⚠️ 常见陷阱

1. **不保证众数存在时直接返回 candidate**  
   - 若无超过 `n/2` 的元素，投票结果可能是**任意**残留值，必须**再数一遍**验证。

2. **与「众数」严格大于一半混淆**  
   - 本题是 `> ⌊n/2⌋`，不是「最多」或「≥ n/2」。

3. **229 题（超过 n/3）**  
   - 需**两个**候选与两套计数，且最后同样要验证；不能照搬 169 单候选代码。

## 总结

多数元素是**摩尔投票**的模板题：利用「超过一半」在配对抵消后仍占优，实现 O(n)、O(1)；变体题先问清是否**保证存在**与**阈值是 n/2 还是 n/k**。
