# 301. 删除无效的括号

**标签**：`广度优先搜索` `回溯` `字符串` `困难`  
**分类**：字符串、括号、搜索  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 中频

---

## 题目描述

给你一个由 `'('`、`')'` 和**小写英文字母**组成的字符串 `s`。

从字符串中删除**最少数量**的无效括号，使得输入字符串有效。返回**所有可能的结果**。答案可以按**任意顺序**返回。

**有效括号串**定义与 [20. 有效的括号](20.有效的括号.md) 相同；本题字符串中可能夹杂字母，字母顺序与数量**不变**，只考虑删或不删括号。

### 示例 1

```
输入：s = "()())()"
输出：["(())()","()()()"]
```

### 示例 2

```
输入：s = "(a)())()"
输出：["(a())()","(a)()()"]
```

### 示例 3

```
输入：s = ")("
输出：[""]
```

### 提示

- `1 <= s.length <= 20`
- `s` 由小写英文字母、`'('`、`')'` 组成
- 数据规模较小，但**解的数量**可能较多，需注意去重与剪枝

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：**BFS**：逐层删 1 个括号，第一次出现合法串的整层即为「最少删除数」，收集该层所有合法串；或 **DFS**：先统计**至少要删几个左括号、几个右括号**，再回溯删括号并去重。

**关键词**：`最少删除` `BFS 按层` `DFS 计数剪枝` `同层去重`

### 复杂度速记

- **BFS**：指数级状态，题目 `n≤20` 可接受；每层扩展 O(n) 种删法。  
- **DFS**：同样指数，靠「必须删的个数」剪枝。

### 记忆口诀

「BFS 一层层删，首现合法就收口；DFS 先数该删几个括号再搜。」

---

## 解题思路

### 合法性判定

遍历字符串，遇 `'('` 计数 `+1`，遇 `')'` 计数 `-1`；过程中 `<0` 非法；结束须 `=0`。字母跳过。

### 方法一：BFS（易想）

1. 队列初始放入 `s`，`visited` 去重。  
2. 每层：弹出当前串，若合法则加入答案，并标记「本层已找到合法串」。  
3. 若本层**已有合法串**，不再向下一层扩展（保证**最少**删除数）。  
4. 否则对每个位置尝试删掉 `'('` 或 `')'`，新串未访问则入队。  
5. 重复直到某层出现合法串或队列为空。

### 方法二：DFS + 预处理删除个数

1. 从左扫一遍，统计**必须删除的右括号数** `rm`（多余的 `)`）；再统计**必须删除的左括号数** `lm`（扫完仍多出的 `(`）。  
2. DFS：从下标 `start` 开始，尝试删或不删；已删 `lm` 个 `(`、`rm` 个 `)` 且串合法则加入集合。  
3. 同层**连续相同字符**只删第一个，避免重复结果（经典去重）。

## 代码实现

### 写法一：BFS（推荐口述）

```python
from collections import deque


class Solution:
    def removeInvalidParentheses(self, s: str) -> list[str]:
        def valid(t: str) -> bool:
            c = 0
            for ch in t:
                if ch == "(":
                    c += 1
                elif ch == ")":
                    c -= 1
                    if c < 0:
                        return False
            return c == 0

        ans: list[str] = []
        vis = {s}
        q = deque([s])
        found = False
        while q:
            for _ in range(len(q)):
                t = q.popleft()
                if valid(t):
                    ans.append(t)
                    found = True
                if found:
                    continue
                for i in range(len(t)):
                    if t[i] not in "()":
                        continue
                    nxt = t[:i] + t[i + 1 :]
                    if nxt not in vis:
                        vis.add(nxt)
                        q.append(nxt)
            if found:
                break
        return ans
```

### 写法二：DFS（先算最少要删的 `left_rem` / `right_rem`）

从左到右扫一遍：遇 `(` 则「待匹配左括号」`+1`；遇 `)` 若前面有未匹配 `(` 则抵消，否则说明多了一个 `)`，**`right_rem += 1`**。扫完后剩余的「待匹配左括号」个数即为 **`left_rem`**。DFS 时只在「还可以删」的前提下分支**删**或**留**，并用 **`open_cnt`** 表示当前路径上未匹配的 `(` 数；遇 `)` 若要保留必须先 `open_cnt > 0`。结果用 **`set`** 去重（也可用「连续相同括号只删第一个」在搜索中剪枝）。

```python
class Solution:
    def removeInvalidParentheses(self, s: str) -> list[str]:
        left_rem = right_rem = 0
        for c in s:
            if c == "(":
                left_rem += 1
            elif c == ")":
                if left_rem > 0:
                    left_rem -= 1
                else:
                    right_rem += 1

        res: set[str] = set()

        def dfs(idx: int, lr: int, rr: int, open_cnt: int, path: list[str]) -> None:
            if idx == len(s):
                if lr == 0 and rr == 0 and open_cnt == 0:
                    res.add("".join(path))
                return
            c = s[idx]
            if c not in "()":
                path.append(c)
                dfs(idx + 1, lr, rr, open_cnt, path)
                path.pop()
                return
            if c == "(":
                if lr > 0:
                    dfs(idx + 1, lr - 1, rr, open_cnt, path)
                path.append("(")
                dfs(idx + 1, lr, rr, open_cnt + 1, path)
                path.pop()
            else:
                if rr > 0:
                    dfs(idx + 1, lr, rr - 1, open_cnt, path)
                if open_cnt > 0:
                    path.append(")")
                    dfs(idx + 1, lr, rr, open_cnt - 1, path)
                    path.pop()

        dfs(0, left_rem, right_rem, 0, [])
        return list(res)
```

## ⚠️ 常见陷阱

1. **只要一个解**  
   - 本题要**所有**最少删除方案，不能只返回第一个合法串。

2. **重复结果**  
   - 如 `"(()"` 删不同位置的 `(` 可能得到相同串，需 `set` 或**同层连续相同字符剪枝**。

3. **误删字母**  
   - 只能删除 `'('` 或 `')'`，字母必须保留。

4. **与 20 题关系**  
   - 合法性判定相同；301 是「最少删掉几个括号」的**搜索**问题。

---

## 复杂度分析

- **状态数**：最坏指数级（与 `n` 与括号分布有关）。  
- **BFS**：每层最多 O(n) 个分支，深度最多 n，实际由 `visited` 控制。  
- **题目**：`n ≤ 20`，暴力搜索可接受。

## 总结

301 是**括号 + 最小编辑 + 输出所有方案**：**BFS 按层**最贴合「最少删除」语义；**DFS** 需先算可删括号数量再剪枝，并处理去重。面试优先写 BFS 全流程，再补充 DFS 思路即可。
