# 128. 最长连续序列

**标签**：`哈希表` `并查集` `数组` `中等`  
**分类**：哈希、数组  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥 中高频

---

## 题目描述

给定一个未排序的整数数组 `nums`，找出**数字连续**的最长序列（不要求序列元素在原数组中连续）的长度。  
请你设计并实现**时间复杂度为 O(n)** 的算法解决此问题。

### 示例 1

```
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
```

### 示例 2

```
输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9
解释：最长连续序列是 [0,1,2,3,4,5,6,7,8]，长度为 9。
```

### 示例 3

```
输入：nums = [1,0,1,2]
输出：3
解释：最长连续序列是 [0,1,2]，长度为 3。
```

### 提示

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

用**哈希集合**存所有数，**只从「连续序列的起点」开始往后数**：若 `n - 1` 不在集合里，则 `n` 可作为起点，用 `while n+1 in set` 向后扩展，得到以 `n` 为起点的连续段长度。所有起点枚举完，取最大长度。

**关键**：每个数字最多被「作为起点数一次 + 在别人延伸时被访问一次」，总访问 O(n)，故整体 O(n)。

**关键词**：`哈希集合` `只枚举起点` `n-1 不在集合`

### 复杂度速记

- **时间**：O(n)——每个数至多被访问 2 次（作为起点一次、被延伸经过一次）。
- **空间**：O(n)——哈希集合存全部数字。

### 记忆口诀

「先入集合再枚举，n-1 不在才当起点；从起点往后加一数长度，取最大。」

---

## 解题思路

### 核心思想

- **暴力**：对每个数，往左往右扩展找连续段——同一段会被重复枚举多次，无法 O(n)。
- **优化**：只对「连续序列的最小值」（即**起点**）做一次完整延伸。如何判断起点？**若 `n - 1` 不在数组里，则 `n` 可以是某段连续序列的起点**；否则 `n` 一定在「以比它小的数为起点」的那段里，不必再数。

因此：先把所有数放进 `set`，遍历集合（或原数组去重后），当 `n - 1 not in set` 时，从 `n` 开始用 `while n+1 in set` 向后数，更新最大长度。

### 算法步骤

1. 将 `nums` 转为集合 `nums_set`，去重且 O(1) 查询。
2. 初始化 `res = 0`。
3. 对集合中每个数 `n`：
   - 若 `n - 1` 在集合中，说明 `n` 不是某段的最小值，跳过（避免重复数同一段）。
   - 若 `n - 1` 不在集合中，则 `n` 为某段起点：令 `cur = n`，`length = 0`，循环 `while cur in nums_set`：`length += 1`，`cur += 1`。
   - `res = max(res, length)`。
4. 返回 `res`。

### 为什么是 O(n)？

- 每个连续段只会被「起点」那一次循环完整走完。
- 每个数字 `x` 最多参与两次：一次是作为起点被枚举，一次是在以更小数为起点的延伸中被 `cur` 扫到。故总访问次数为 O(n)。

## 代码实现

```python
from typing import List

class Solution:
    def longestConsecutive(self, nums: List[int]) -> int:
        """
        哈希集合 + 只从序列起点扩展。
        若 n-1 不在集合中，则 n 为某连续段起点，向后数到不在集合为止。
        时间 O(n)，空间 O(n)。
        """
        nums_set = set(nums)
        res = 0

        for n in nums_set:
            # 只从「连续序列的最小值」开始数，避免同一段被重复计算
            if n - 1 not in nums_set:
                cur = n
                length = 0
                while cur in nums_set:
                    length += 1
                    cur += 1
                res = max(res, length)

        return res
```

**代码要点**：

- ✅ 遍历**集合** `nums_set` 即可，无需遍历原数组（避免重复元素重复枚举）。
- ✅ 必须判断 `n - 1 not in nums_set`，否则会重复统计同一段。
- ✅ 内层 `while` 每轮 `cur` 递增，每个数最多进入一次，保证 O(n)。

## 复杂度分析

| 项目     | 复杂度 | 说明 |
|----------|--------|------|
| 时间     | O(n)   | 建 set O(n)；每个数最多被起点枚举 1 次、被延伸访问 1 次，合计 O(n)。 |
| 空间     | O(n)   | 哈希集合存储所有不同数字。 |

---

## 易错点

- **遍历对象**：应遍历 `set(nums)` 或去重后的序列，若遍历原 `nums`，重复值会作为「起点」被多次枚举，逻辑正确但多做了无用功（同一段算多次），仍可 AC，但面试时说明「遍历集合更清晰」更好。
- **起点判断**：必须用 `n - 1 not in nums_set`。若写成 `n + 1 not in nums_set` 就变成「只从终点」数，也可以，但习惯上从最小值（起点）数更直观。
- **空数组**：`nums = []` 时，`res` 保持 0 即可，无需特判。
