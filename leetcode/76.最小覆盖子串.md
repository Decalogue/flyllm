# 76. 最小覆盖子串

**标签**：`字符串` `滑动窗口` `哈希表` `困难`  
**分类**：双指针、滑动窗口、字符串  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你两个字符串 `s` 和 `t`，返回 `s` 中涵盖 `t` 中**所有字符**（含**重复次数**）的**最短子串**。若不存在满足条件的子串，返回空字符串 `""`。

- **覆盖**：`t` 中每个字符出现的次数，必须在所选 `s` 的连续子串中**不少于**对应次数（多出来的字符可以留在窗口里）。

### 示例 1

```
输入：s = "ADOBECODEBANC", t = "ABC"
输出："BANC"
解释：子串 "BANC" 包含 'A'、'B'、'C' 各至少 1 个，且在所有满足条件的子串中最短。
```

### 示例 2

```
输入：s = "a", t = "a"
输出："a"
```

### 示例 3

```
输入：s = "a", t = "aa"
输出：""
解释：t 中需要两个 'a'，s 只有一个，无法满足。
```

### 提示

- `m == s.length`，`n == t.length`
- `1 <= m, n <= 10^5`
- `s` 和 `t` 由英文字母组成

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：右指针扩展窗口直到**覆盖** `t`，再左指针尽量**收缩**仍保持覆盖，全程记录最短区间。

**关键词**：`滑动窗口` `need 与 window` `有效覆盖 valid` `先扩后缩`

### 复杂度速记

- **时间复杂度**：O(|s| + |t|) — 每个下标最多被左、右指针各扫一次（均摊）。
- **空间复杂度**：O(|Σ|) — 字符集大小，实际为哈希表存出现过的字母种类，常数级。

### 记忆口诀

「右扩凑齐 need，左缩保覆盖；齐了就量长，断档再右扩。」

### 代码模板（滑动窗口 + 满足条件）

```python
need = Counter(t)           # 每种字符需要几个
window = defaultdict(int)   # 当前窗口内计数
valid = 0                   # 有多少种字符已经「个数达标」
required = len(need)        # 需要达标的字符种类数

for right in range(len(s)):
    # 右端入窗，更新 window，必要时 valid += 1
    while valid == required:
        # 左端出窗尝试收缩，更新最短答案
        # 左端出窗，必要时 valid -= 1
```

## 解题思路

### 核心思想

用**双指针**维护 `s` 上的一个连续区间 `[left, right]`。先移动 `right` 扩大窗口，直到窗口内各字符出现次数**不少于** `t` 的要求；再移动 `left` 缩小窗口，在仍满足要求的前提下尽量变短，并更新全局最短答案。`right` 走到尽头即结束。

**关键点：**

- `t` 中有**重复**时，必须用**次数**比较，不能只判断「出现过」。
- 「满足覆盖」可化为：对 `need` 中每一种字符 `c`，都有 `window[c] >= need[c]`。用 `valid` 表示当前已有多少种字符达到等号（`window[c] == need[c]` 作为从「未达标」到「达标」的触发）。
- 左指针每移一次，若某种字符从「达标」变成「不足」，`valid` 减 1，窗口不再合法，需继续右扩。

### 算法步骤

1. 统计 `need = Counter(t)`，记 `required = len(need)`（不同字符种类数）。
2. `left = 0`，`window` 为空，`valid = 0`。`start, length` 用于记最短子串起点与长度。
3. `right` 从 `0` 到 `len(s)-1`：
   - 将 `s[right]` 纳入 `window`，若某字符 `c` 在 `need` 中且 `window[c] == need[c]`，则 `valid += 1`。
4. 当 `valid == required` 时，窗口已覆盖 `t`：
   - 若当前区间更短，更新 `start` 与 `length`。
   - 将 `s[left]` 移出窗口：`window[d] -= 1`；若 `d` 在 `need` 中且移出后 `window[d] < need[d]`，则 `valid -= 1`；`left += 1`。
   - 重复步骤 4 直到 `valid < required`，再回到步骤 3 扩 `right`。
5. 若从未找到合法窗口，返回 `""`；否则返回 `s[start:start+length]`。

### 为什么选择这个方法？

- **优势**：线性时间，思路与「子串包含某类性质」类题统一，可迁移到 438、567 等。
- **适用场景**：主串很长、目标串由字符频约束时。
- **替代方案**：暴力 O(|s|²) 枚举子串不可接受；仅哈希不加滑窗会超时。

## 代码实现

### 滑动窗口（推荐 ⭐）

```python
from collections import Counter, defaultdict


class Solution:
    def minWindow(self, s: str, t: str) -> str:
        """
        need：t 中各字符需求次数；window：当前窗口内次数。
        valid：已有多少种字符达到「恰好满足 need 中的次数」（窗口内该字符个数 >= need 时，
        在从 <need 增到 ==need 时 valid+1；左移导致 <need 时 valid-1）。
        时间复杂度: O(len(s) + len(t))
        空间复杂度: O(|字母表|)
        """
        if not s or not t or len(s) < len(t):
            return ""

        need = Counter(t)
        window = defaultdict(int)
        valid = 0
        required = len(need)

        start, length = 0, float("inf")
        left = 0

        for right in range(len(s)):
            c = s[right]
            window[c] += 1
            if c in need and window[c] == need[c]:
                valid += 1

            while valid == required:
                if right - left + 1 < length:
                    start = left
                    length = right - left + 1
                d = s[left]
                window[d] -= 1
                if d in need and window[d] < need[d]:
                    valid -= 1
                left += 1

        return "" if length == float("inf") else s[start : start + length]
```

**代码要点**：

- ✅ `valid` 按**字符种类**统计「是否已满足该种类的个数要求」，不是窗口内总字符数。
- ✅ 右扩时 `window[c] == need[c]` 才 `valid += 1`（从未达标到刚好达标）；左缩后若 `window[d] < need[d]`，该种类不再达标，`valid -= 1`。
- ✅ 收缩循环用 `while valid == required`，在同一 `right` 下尽可能左移，得到以当前 `right` 为右端点的最短合法左端点。

### 写法二：用「欠缺量」代替 valid（思路等价）

部分题解用 `missing` 表示「相对 `t` 还缺多少个字符」或逐类判断，本质仍是维护窗口是否覆盖 `need`，面试任选一种写顺手的即可。

## ⚠️ 常见陷阱（面试易错点）

1. **忽略 `t` 中重复字符**
   - ❌ **错误做法**：只判断 `t` 中字母是否都在窗口里出现过。
   - ✅ **正确做法**：用**次数** `need[c]` 与 `window[c]` 比较。
   - 💡 **原因分析**：如 `t = "aa"`，窗口里只有一个 `a` 不算覆盖。

2. **`valid` 与「种类数」混淆**
   - ❌ **错误做法**：`valid` 每次右扩加 1。
   - ✅ **正确做法**：仅当某字符从**不足**变为**恰好达到** `need[c]` 时，该种类算「达标」，种类达标数加 1；左移导致**低于** `need[c]` 时减 1。

3. **边界情况**
   - **`len(s) < len(t)`**：不可能覆盖，直接 `""`。
   - **答案为空**：全程未出现 `valid == required`，返回 `""`。

## 复杂度分析

### 时间复杂度

- **O(|s| + |t|)**：`need` 建表 O(|t|)；`right` 扫一遍 `s`，`left` 只前进不后退，每个位置最多被左、右各访问一次。

### 空间复杂度

- **O(|Σ|)**：`need` 与 `window` 的键集大小不超过字符集；英文大小写字母时最多 52 类。

### 复杂度对比表

| 方法 | 时间复杂度 | 空间复杂度 | 备注 |
|------|------------|------------|------|
| 滑动窗口 | O(\|s\| + \|t\|) | O(\|Σ\|) | 标准解 |
| 暴力枚举子串 | O(\|s\|²) 或更高 | O(1) 额外 | 超时 |

## 关键点

1. **为何是「== need[c]」时 valid+1，而不是 ≥？**  
   每种字符只需在「从不够到够」时记一次达标；多于 `need[c]` 仍算已覆盖该种类，直到左移把个数降到 `need[c]` 以下才撤销。

2. **与「无重复字符最长子串」有何不同？**  
   那题窗口内要求**互异**（或出现次数约束不同）；本题窗口要**包含**目标 multiset，是「下界」约束，收缩条件相反。

3. **面试要点**  
   能说清 `need / window / valid / required` 四者含义，并能手推一例（如 `S=ADOBECODEBANC`, `T=ABC`）。

## 💬 面试问题（模拟面试场景）

### 基础问题

1. **Q: 为什么双指针不会漏掉最优解？**  
   - A: 对每个固定的 `right`，我们在合法前提下把 `left` 拉到最右，得到以该 `right` 结尾的最短合法窗口；全局最短必出现在某一组 `(left, right)` 上，因此不会漏。

2. **Q: 时间复杂度为什么是线性的？**  
   - A: `left` 与 `right` 均只向右移动，各自最多走 |s| 步。

### 进阶问题

1. **Q: 若字符集极大（如 Unicode）怎么分析空间？**  
   - A: 仍为窗口与 need 中出现过的字符种类数，题目常给常数字母表。

2. **Q: 变种——只要求字符种类相同、不要求顺序？**  
   - A: 仍是 multiset 覆盖，框架不变；若变成「排列」则不是子串问题。

## 总结

这是一道**滑动窗口 + 字符频**的经典题，核心是：

1. **右扩**直到窗口覆盖 `t` 的频数要求；**左缩**在仍覆盖的前提下取最短。  
2. 用 **`valid` + `required`** 维护「多少种字符已达标」，避免每次 O(|Σ|) 检查。  
3. 复杂度 **O(|s|+|t|)**，空间 O(|Σ|)。

**面试建议**：

- 🎯 先写 `need` 与窗口更新，再写 `valid` 的增减条件，最后写收缩与答案更新。  
- 🎯 主动举 `t` 含重复字符的例子，展示边界。
