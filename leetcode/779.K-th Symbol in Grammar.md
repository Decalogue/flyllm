# 779. K-th Symbol in Grammar

**标签**：`递归` `二叉树隐式建模` `位运算` `简单` `LeetCode`  
**分类**：`递归 / 数学规律`  
**难度**：⭐ 简单  
**频率**：🔥🔥 中频

---

## 题目描述

我们构造一个语法表：

- 第 1 行：`0`
- 后续每一行由上一行每个字符替换得到：
  - `0 -> 01`
  - `1 -> 10`

给定 `n` 和 `k`，返回第 `n` 行第 `k` 个字符（`k` 从 1 开始）。

---

## 📋 面试要点速查（知识卡片）

### 核心思路
**一句话总结**：第 `n` 行第 `k` 位，等价于从根 `0` 走到第 `k` 个叶子路径上“翻转次数”的奇偶性。  

**关键词**：`父子映射` `奇偶翻转` `bitCount`

### 复杂度速记
- **递归解法**：时间 `O(n)`，空间 `O(n)`（递归栈）
- **位运算解法**：时间 `O(1)`（固定字长）/ `O(log k)`（按位计数），空间 `O(1)`

### 记忆口诀
`左继承，右翻转；统计一路翻几次。`

---

## 解题思路

### 方法一：递归（面试首选，逻辑最直观）

把每一行看成一棵满二叉树的一层展开：

- 父节点是 `0`，左子是 `0`，右子是 `1`
- 父节点是 `1`，左子是 `1`，右子是 `0`

因此：

- 第 `n` 行第 `k` 位的父节点是第 `n-1` 行第 `(k+1)//2` 位
- 如果 `k` 是奇数（左孩子），值与父相同
- 如果 `k` 是偶数（右孩子），值与父相反

递归边界：`n == 1` 时答案是 `0`。

---

### 方法二：位运算（最短代码）

经典结论：

- 第 `n` 行第 `k` 位（在合法范围内）等于 `popcount(k-1) % 2`

直觉解释：`k-1` 的二进制中 1 的个数，表示从根走到该位置过程中“走右边”的次数；每走右边翻转一次，翻转奇数次为 `1`，偶数次为 `0`。

---

## 代码实现

### 解法一：递归

```python
class Solution:
    def kthGrammar(self, n: int, k: int) -> int:
        # 边界：第一行只有一个 0
        if n == 1:
            return 0

        parent = self.kthGrammar(n - 1, (k + 1) // 2)
        # 奇数位：左孩子，和父亲相同；偶数位：右孩子，和父亲相反
        return parent if k % 2 == 1 else 1 - parent
```

**代码要点**：
- ✅ 关键是父节点下标 `(k + 1) // 2`
- ✅ 奇偶决定是否翻转
- ✅ 不需要构造整行字符串

### 解法二：位运算

```python
class Solution:
    def kthGrammar(self, n: int, k: int) -> int:
        # Python 3.8+ 可用 int.bit_count()
        return (k - 1).bit_count() & 1
```

**代码要点**：
- ✅ `n` 在公式里不显式出现（只要输入保证 k 在第 n 行范围内）
- ✅ 结果只看 `(k-1)` 的 1 的个数奇偶

## 执行过程示例

以 `n = 4, k = 5` 为例：

- `k=5`（奇数）看父节点 `(5+1)//2 = 3`
- `k=3`（奇数）看父节点 `(3+1)//2 = 2`
- `k=2`（偶数）看父节点 `(2+1)//2 = 1`
- 到达 `n=1` 返回 `0`
- 回溯：第 2 层是右孩子，翻转为 `1`；第 3 层左孩子保持 `1`；第 4 层左孩子保持 `1`

答案是 `1`。

---

## ⚠️ 常见陷阱（面试易错点）

1. **下标从 1 开始，不是从 0 开始**
   - ❌ 把父节点写成 `k // 2`
   - ✅ 正确是 `(k + 1) // 2`

2. **奇偶与翻转关系写反**
   - ❌ 奇数翻转、偶数不翻转
   - ✅ 奇数（左）不翻转，偶数（右）翻转

3. **试图构造整行字符串**
   - ❌ 第 `n` 行长度是 `2^(n-1)`，会爆内存
   - ✅ 只做下标反推或位运算

## 复杂度分析

### 递归解法
- **时间复杂度**：`O(n)`（每层递归一次）
- **空间复杂度**：`O(n)`（递归栈）

### 位运算解法
- **时间复杂度**：`O(1)`（固定字长机器）或 `O(log k)`（按位统计）
- **空间复杂度**：`O(1)`

---

## 关键点（面试追问高频）

1. **为什么可以只看 `k-1` 的 bitCount？**
   - 二进制路径中每个 `1` 对应一次“走右子树”，即一次翻转。

2. **为什么位运算里不需要 `n`？**
   - 位置 `k` 的路径结构由 `k` 自身决定；`n` 仅用于保证输入合法（`1 <= k <= 2^(n-1)`）。

3. **和“构造字符串取第 k 位”有什么本质区别？**
   - 前者利用结构规律做索引映射；后者做显式展开，复杂度指数级。

## 总结

这题是“**结构化递归 + 第 k 位查询**”模板题：

1. 不构造整行，只做索引映射；
2. 先掌握父子关系递归写法；
3. 再记住 `ans = popcount(k-1) % 2` 的位运算捷径。

面试时优先讲递归推导，再补一句位运算结论，完整度最高。
