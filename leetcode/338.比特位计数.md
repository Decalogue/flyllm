# 338. 比特位计数

**标签**：`位运算` `动态规划` `简单`  
**分类**：位运算、线性 DP  
**难度**：⭐ 简单  
**频率**：🔥🔥 中高频

---

## 题目描述

给你一个整数 `n`，对于 `0 ≤ i ≤ n` 的每个 `i`，计算其二进制表示中 **`1` 的个数**（汉明重量），返回长度为 `n + 1` 的数组 `ans`，其中 `ans[i]` 即为 `i` 的二进制中 `1` 的个数。

### 示例 1

```
输入：n = 2
输出：[0,1,1]
解释：
0 --> 0
1 --> 1
2 --> 10
```

### 示例 2

```
输入：n = 5
输出：[0,1,1,2,1,2]
解释：
0 --> 0    （0 个 1）
1 --> 1
2 --> 10
3 --> 11
4 --> 100
5 --> 101
```

### 提示

- `0 <= n <= 10^5`

**进阶**：时间复杂度 **O(n)** 内完成，这是容易做到的；能否用 **O(n log n)** 以外的更巧妙方式？（本题标准即为线性 DP / 位运算技巧。）

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：`i` 的二进制比 `i >> 1` 多出的只是**最低位**：`dp[i] = dp[i >> 1] + (i & 1)`。

**关键词**：`右移一位` `最低位` `O(n) 递推`

### 复杂度速记

- **时间**：O(n)，每个 `i` 常数时间。  
- **空间**：O(n) 存答案数组（题目要求），除答案外可 O(1) 额外空间。

### 记忆口诀

「右移丢掉最低位，再加上当前位是 0 还是 1。」

---

## 解题思路

### 核心思想

观察：`i` 与 `i // 2`（即 `i >> 1`）在二进制上只差**最后一位**。

- `i >> 1` 的二进制与 `i` 去掉最低位后相同，故 `1` 的个数相差仅由最低位决定。  
- 最低位为 1 则 `i & 1 == 1`，否则为 0。

因此 `dp[0] = 0`，对 `i ≥ 1`：`dp[i] = dp[i >> 1] + (i & 1)`。

### 另一种等价递推（lowbit）

`i & (i - 1)` 会**去掉** `i` 的最低位的 `1`，故：

`dp[i] = dp[i & (i - 1)] + 1`

同样 O(n)，常数可能略大，但思路常用于「扩展」类位运算题。

### 算法步骤

1. 创建长度为 `n + 1` 的数组 `dp`，`dp[0] = 0`。  
2. 对 `i` 从 `1` 到 `n`：`dp[i] = dp[i >> 1] + (i & 1)`。  
3. 返回 `dp`。

---

## 代码实现

### 写法一：右移 + 最低位（最常用）

```python
class Solution:
    def countBits(self, n: int) -> list[int]:
        dp = [0] * (n + 1)
        for i in range(1, n + 1):
            dp[i] = dp[i >> 1] + (i & 1)
        return dp
```

### 写法二：`i & (i - 1)` 消去 lowbit

```python
class Solution:
    def countBits(self, n: int) -> list[int]:
        dp = [0] * (n + 1)
        for i in range(1, n + 1):
            dp[i] = dp[i & (i - 1)] + 1
        return dp
```

**要点**：`i & (i - 1)` 在 `i > 0` 时至少比 `i` 小，且递推无环。

---

## ⚠️ 常见陷阱

1. **边界**  
   - `n = 0` 时答案为 `[0]`，循环从 `1` 开始即可。

2. **误以为要对每个 `i` 做 O(log i) 逐位统计**  
   - 可以，但总复杂度 O(n log n)；题目期望利用**相邻数字**关系做到 O(n)。

3. **与 191 混淆**  
   - [191. 位 1 的个数](https://leetcode.cn/problems/number-of-1-bits/) 只求**一个**整数的 1 的个数；本题要对 `0..n` **每个**都算一遍，适合 DP。

## 复杂度分析

| 方法 | 时间 | 额外空间（除输出） |
|------|------|---------------------|
| 逐 i 暴力数 bit | O(n log n) | O(1) |
| DP 右移 | O(n) | O(1) |
| DP lowbit | O(n) | O(1) |

## 总结

本题是**位运算 + 线性 DP** 的典型题：利用 `i` 与 `i >> 1` 的**一位之差**递推，可在 O(n) 内填完 `0..n` 所有数的 popcount，实现简单、常考。
