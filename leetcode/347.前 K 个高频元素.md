# 347. 前 K 个高频元素

**标签**：`堆` `哈希` `桶排序` `中等`  
**分类**：数组、频数统计、TopK  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频  

---

## 题目描述

给你一个整数数组 `nums` 和一个整数 `k`，请你返回其中出现频率前 `k` 高的元素。你可以按**任意顺序**返回答案。

### 示例 1

```
输入：nums = [1,1,1,2,2,3], k = 2
输出：[1,2]
解释：1 出现 3 次，2 出现 2 次，3 出现 1 次。前 2 高频为 1 与 2。
```

### 示例 2

```
输入：nums = [1], k = 1
输出：[1]
```

### 提示

- `1 <= nums.length <= 10^5`
- `k` 的取值范围是 `[1, 数组中不同元素的个数]`
- `-10^4 <= nums[i] <= 10^4`
- 题目数据保证答案**唯一**（即前 `k` 个高频元素是确定的）

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：先**统计每个数的出现次数**，再按次数选出**最大的 k 个**（堆、排序或桶）。

**关键词**：`Counter` `TopK` `小根堆维护 k 个` `频数桶`

### 复杂度速记

| 方法 | 时间 | 空间 |
|------|------|------|
| 哈希 + 按频排序 | O(n + u log u) | O(u) |
| 哈希 + `nlargest` / 大小为 k 的堆 | O(n + u log k) | O(u) |
| 哈希 + 桶（下标=频数） | O(n) | O(n) |

其中 `n = len(nums)`，`u` 为不同元素个数（`u ≤ n`）。

### 记忆口诀

「先数数，再按次数做 TopK；桶排序频数有上限 n。」

---

## 解题思路

### 核心思想

1. 遍历 `nums`，得到 `值 → 出现次数`（`Counter` 或手写哈希表）。  
2. 在「不同数值」这一层做 TopK：**按频数从大到小**取 `k` 个键。

### 算法步骤

1. `c = Counter(nums)`。  
2. 从 `c` 的键里选出频数最高的 `k` 个（堆、排序、或桶）。  
3. 返回这 `k` 个数组成的列表（顺序任意）。

### 方法对比

| 方法 | 做法 | 面试怎么说 |
|------|------|------------|
| **排序** | `sorted(c.keys(), key=c.get, reverse=True)[:k]` | 最直观，u 不大时够用 |
| **`nlargest`** | 对键用 `key=c.__getitem__` 取前 k 大 | 等价于大小为 k 的堆思路，代码短 |
| **小根堆** | 堆里存 `(频数, 值)`，超过 k 就弹最小 | 经典 TopK 模板，强调堆时写这个 |
| **桶排序** | 下标 `i` 表示频数 `i`，桶里放数值列表 | 频数 ∈ [1, n]，线性时间 |

## 代码实现

### 方法一：`Counter` + `heapq.nlargest`（推荐手写 ⭐）

```python
import heapq
from collections import Counter


class Solution:
    def topKFrequent(self, nums: list[int], k: int) -> list[int]:
        c = Counter(nums)
        return heapq.nlargest(k, c.keys(), key=c.__getitem__)
```

**要点**：

- `key=c.__getitem__` 等价于 `lambda x: c[x]`，按**频数**比较键。  
- 时间 O(n + u log k)，空间 O(u)。

### 方法二：小根堆维护 k 个「高频候选」

```python
import heapq
from collections import Counter


class Solution:
    def topKFrequent(self, nums: list[int], k: int) -> list[int]:
        c = Counter(nums)
        h: list[tuple[int, int]] = []
        for x, f in c.items():
            heapq.heappush(h, (f, x))
            if len(h) > k:
                heapq.heappop(h)
        return [x for _, x in h]
```

**要点**：

- 堆按**频数**比较；堆大小始终 ≤ k，弹出的是当前堆里频数最小者。  
- 最后堆里剩的就是全局频数最高的 k 个（可 unsorted，题目允许任意顺序）。

### 方法三：桶排序（O(n)）

```python
from collections import Counter


class Solution:
    def topKFrequent(self, nums: list[int], k: int) -> list[int]:
        c = Counter(nums)
        buckets: list[list[int]] = [[] for _ in range(len(nums) + 1)]
        for x, f in c.items():
            buckets[f].append(x)
        ans: list[int] = []
        for i in range(len(buckets) - 1, 0, -1):
            for x in buckets[i]:
                ans.append(x)
                if len(ans) == k:
                    return ans
        return ans
```

**要点**：

- 任意元素出现次数最多为 `n`，故桶长 `n+1`，下标 `0` 不用。  
- 从高频桶往低频扫，凑满 `k` 个即返回。

### 执行过程示例（桶思路）

`nums = [1,1,1,2,2,3], k = 2`：

- `c = {1:3, 2:2, 3:1}`  
- `buckets[3]=[1], buckets[2]=[2], buckets[1]=[3]`  
- 从 `i=3` 取 `1`，再从 `i=2` 取 `2`，得 `[1,2]`。

## ⚠️ 常见陷阱

1. **和 215 题混淆**  
   - 215 是按**数值大小**找第 k 大元素；本题是按**出现次数**找 TopK **数值**。  
   - 比较键不同：一个是数组元素值，一个是 `Counter` 的频数。

2. **用大根堆时的元素设计**  
   - Python `heapq` 是小根堆。若要「频数大的优先」，可存 `(-count, num)`，或直接用上面的「维护 k 个的小根堆」。

3. **返回值顺序**  
   - 题目允许任意顺序；若面试官要求按频数降序输出，需在得到 k 个数后再按 `c[x]` 排一下。

## 复杂度分析

| 方法 | 时间复杂度 | 空间复杂度 |
|------|------------|------------|
| 哈希 + `nlargest` | O(n + u log k) | O(u) |
| 哈希 + 小根堆 size k | O(n + u log k) | O(u) |
| 哈希 + 按频排序 | O(n + u log u) | O(u) |
| 哈希 + 桶 | O(n) | O(n)（桶与答案） |

统计次数遍历一次为 O(n)；后续复杂度由「不同元素个数 u」与选取方式决定。
