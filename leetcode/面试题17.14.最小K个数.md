# 面试题 17.14. 最小 K 个数

**标签**：`堆` `快速选择` `排序` `TopK` `中等`  
**分类**：数组、第 K 小 / TopK  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥 中频  

> 出处：《程序员面试金典》第 17 章；与 [剑指 Offer 40. 最小的 k 个数](https://leetcode.cn/problems/zui-xiao-de-kge-shu-lcof/) 同类。

---

## 题目描述

给定整数数组 `arr` 和整数 `k`，请返回其中**最小的 k 个数**。若题目未要求顺序，答案中元素**任意排列**均可；若要求升序，可在得到 k 个数后再排序。

### 示例 1

```
输入：arr = [3, 2, 1], k = 2
输出：[1, 2]（或 [2, 1]，视题目是否要求有序）
```

### 示例 2

```
输入：arr = [0, 1, 2, 1], k = 1
输出：[0]
```

### 提示（常见约束）

- `1 <= k <= arr.length <= 10^5`（量级因平台而异）
- 元素可为负数；存在重复元素
- 注意 `k == len(arr)` 时即返回全体（或拷贝）

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：求「最小的 k 个」等价于先定位**第 k 小分界**（快排划分），或用**大小为 k 的大根堆**在线维护当前最小的 k 个。

**关键词**：`TopK` `大根堆` `快速选择` `partition`

### 复杂度速记

| 方法 | 时间 | 空间 |
|------|------|------|
| 全局排序 | O(n log n) | O(1) 或 O(n) 视是否拷贝 |
| 大根堆（容量 k） | O(n log k) | O(k) |
| 快速选择 + 取前 k | 平均 O(n)，最坏 O(n²) | O(1) 原地 / O(log n) 栈 |

### 与「第 K 大」的关系

- [215. 数组中的第 K 大元素](<215.数组中的第 K 大元素.md>)：找**一个**分界值；本题要**k 个**数，划分到下标 `k-1` 后取 `arr[:k]` 即可（顺序任意时可不排序）。

---

## 解题思路

### 方法一：大根堆（推荐口述「稳定工程味」）

维护元素个数不超过 `k` 的**大根堆**（堆顶是当前 k 个里最大的）。遍历数组：若堆未满则加入；若满且当前元素小于堆顶，则弹出堆顶再压入当前元素。结束后堆中即为最小的 k 个（堆无序，若需升序再 `sorted(heap)`）。

### 方法二：快速选择（推荐强调「平均线性」）

仿快排 `partition`：随机或固定基准，将 `< pivot` 放左侧。若 pivot 最终下标 `m == k-1`，则左半部分含最小的 k 个；若 `m > k-1` 在左半继续；若 `m < k-1` 在右半找剩下的个数。最后取 `arr[:k]`。与 215 题「找第 k 大」同一套划分框架，只是比较方向与索引换算不同。

### 方法三：排序

`sorted(arr)[:k]`，最简单，面试常作为 baseline，再问优化。

## 代码实现

### 方法一：大根堆（Python `heapq` 用小根堆模拟大根堆）

`heapq` 只有小根堆，存 `(-x, x)` 或统一存负数即可。

```python
import heapq


class Solution:
    def smallestK(self, arr: list[int], k: int) -> list[int]:
        if k == 0:
            return []
        # 小根堆存 -x，等价于大根堆存 x，堆顶为当前 k 个里的最大
        h: list[int] = []
        for x in arr:
            if len(h) < k:
                heapq.heappush(h, -x)
            elif x < -h[0]:
                heapq.heapreplace(h, -x)
        return [-t for t in h]
```

若要求输出升序：`return sorted(-t for t in h)`。

**时间复杂度**：O(n log k)。**空间复杂度**：O(k)。

---

### 方法二：快速选择（原地，平均 O(n)）

```python
import random


class Solution:
    def smallestK(self, arr: list[int], k: int) -> list[int]:
        if k == 0 or not arr:
            return []

        def partition(lo: int, hi: int) -> int:
            p = random.randint(lo, hi)
            arr[p], arr[hi] = arr[hi], arr[p]
            pivot = arr[hi]
            i = lo
            for j in range(lo, hi):
                if arr[j] <= pivot:
                    arr[i], arr[j] = arr[j], arr[i]
                    i += 1
            arr[i], arr[hi] = arr[hi], arr[i]
            return i

        lo, hi = 0, len(arr) - 1
        target = k - 1
        while True:
            m = partition(lo, hi)
            if m == target:
                return arr[:k]
            if m < target:
                lo = m + 1
            else:
                hi = m - 1
```

**说明**：划分后 `[0, k)` 内元素均 ≤ 第 k 小位置上的值，且恰为最小的 k 个（含重复时 multiset 意义下成立）。若题目要求结果升序，再对 `arr[:k]` 排序，额外 O(k log k)。

**时间复杂度**：平均 O(n)，最坏 O(n²)（可通过随机基准缓解）。**空间复杂度**：O(1) 额外空间（递归实现为 O(log n) 栈）。

---

### 方法三：排序（Baseline）

```python
class Solution:
    def smallestK(self, arr: list[int], k: int) -> list[int]:
        return sorted(arr)[:k]
```

**时间复杂度**：O(n log n)。**空间复杂度**：O(n)（排序中间数组，视实现而定）。

---

## ⚠️ 常见陷阱

1. **堆的方向搞反**  
   求「最小的 k 个」要用**大根堆**淘汰堆内较大者；若用小根堆保留全体再弹 k 次，复杂度变差。

2. **快速选择后误以为全局有序**  
   `arr[:k]` 仅保证这 k 个数是整体最小的 k 个，**内部未必有序**。

3. **边界**  
   `k == 0`、`k == len(arr)`、全相同元素，划分仍应正确；注意题目是否允许修改原数组（不允许则需拷贝再选）。

## 复杂度对比表

| 方法 | 时间 | 空间 | 备注 |
|------|------|------|------|
| 排序 | O(n log n) | O(n) | 实现最短 |
| 大根堆 | O(n log k) | O(k) | k ≪ n 时划算 |
| 快速选择 | 平均 O(n) | O(1) | 可原地，最坏需随机化 |

## 总结

- **TopK 最小**：大根堆容量 k，或快速选择把分界推到 `k-1`。  
- 面试顺序：先说排序 baseline → 堆 O(n log k) → 快选平均 O(n)。  
- 看清输出是否要**有序**，再决定是否对长度为 k 的结果排序。
