# 145. 二叉树的后序遍历

**标签**：`栈` `树` `深度优先搜索` `简单`  
**分类**：二叉树、遍历  
**难度**：⭐ 简单  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你二叉树的根结点 `root`，返回其结点值的**后序遍历**。

**后序遍历**：先**后序**遍历左子树，再**后序**遍历右子树，最后访问**根**结点。顺序记作：**左 → 右 → 根**。

### 示例 1

```
输入：root = [1,null,2,3]
输出：[3,2,1]
```

### 示例 2

```
输入：root = []
输出：[]
```

### 示例 3

```
输入：root = [1]
输出：[1]
```

### 提示

- 树中结点数目范围：`[0, 100]`  
- `-100 <= Node.val <= 100`

**进阶**：递归解法很简单，能否用**迭代**完成？

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：递归严格「左右根」；迭代常见两路——(1) 先做一次「根右左」的栈遍历再**整体反转**；(2) 单栈 + **上一次访问结点**判断右子树是否已处理完，再访问根。

**关键词**：`后序` `左右根` `根右左反转` `last / prev 指针`

### 复杂度速记

- **时间**：O(n)，每个结点访问常数次。  
- **空间**：递归 O(h)；迭代栈 O(h)；「反转法」额外 O(n) 存遍历序列。h 为树高。

### 记忆口诀

「后序根垫后：要么根右左再倒序，要么盯紧右子树有没有刚走完。」

---

## 解题思路

### 核心思想

后序是 **DFS**，根**最后**输出，因此不能像前序那样「一弹出就访问」。常见实现：

1. **递归**：与定义一致，面试先写这个最稳。  
2. **根右左 + 反转**：前序是「根左右」，若改成「根右左」（栈里**先压左再压右**，使右先出），得到序列反转后正好是「左右根」。  
3. **单栈 + `last`**：一路向左压栈；栈顶无左可走时，若**右子树为空或右子树已处理完**（`last` 是右孩子），则访问根并弹栈；否则转向右子树。`last` 记录**最近一次写入答案的结点**。

### 算法步骤（反转法）

1. 若 `root` 为空，返回 `[]`。  
2. `res = []`，`st = [root]`。  
3. 当栈非空：弹出 `n`，将 `n.val` 追加到 `res`；若 `n.left` 非空入栈；若 `n.right` 非空入栈（**先左后右**，出栈顺序为**先右后左**，整体遍历顺序为**根右左**）。  
4. `return res[::-1]`。

### 与前序、中序的对比

- **前序**（[144](144.二叉树的前序遍历.md)）：根最先，栈模拟时**先压右再压左**。  
- **中序**（[94](94.二叉树的中序遍历.md)）：左链压栈，弹栈时访问根，再转右。  
- **后序**：根最后；迭代勿套用中序「一路向左」模板，除非配合 `last` 或标记。

## 代码实现

### 写法一：递归（最直观）

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def postorderTraversal(self, root: TreeNode | None) -> list[int]:
        res: list[int] = []

        def dfs(n: TreeNode | None) -> None:
            if not n:
                return
            dfs(n.left)
            dfs(n.right)
            res.append(n.val)

        dfs(root)
        return res
```

**要点**：访问顺序严格为左 → 右 → 根。

### 写法二：迭代 — 根右左再反转（代码短 ⭐）

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def postorderTraversal(self, root: TreeNode | None) -> list[int]:
        if not root:
            return []
        res: list[int] = []
        st: list[TreeNode] = [root]
        while st:
            n = st.pop()
            res.append(n.val)
            if n.left:
                st.append(n.left)
            if n.right:
                st.append(n.right)
        return res[::-1]
```

**要点**：与前序「先右后左」相反，这里**先左后右**，得到 **根 → 右 → 左** 的访问顺序写入 `res`，反转即为后序。

### 写法三：迭代 — 单栈 + `last`（不反转，O(h) 额外栈）

```python
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right


class Solution:
    def postorderTraversal(self, root: TreeNode | None) -> list[int]:
        res: list[int] = []
        st: list[TreeNode] = []
        last: TreeNode | None = None
        cur: TreeNode | None = root
        while st or cur:
            if cur:
                st.append(cur)
                cur = cur.left
            else:
                peek = st[-1]
                if peek.right and last is not peek.right:
                    cur = peek.right
                else:
                    res.append(peek.val)
                    last = st.pop()
        return res
```

**要点**：`last` 表示**上一个已加入 `res` 的结点**；仅当「无右子树」或「右子树已在上一轮处理完」时才能访问当前根。

### 执行过程示例（反转法）

`root = [1,null,2,3]`：根 `1`，`1.right = 2`，`2.left = 3`。

| 步骤 | 弹出 | 入栈（先左后右） | `res`（根右左） |
|------|------|------------------|-----------------|
| 1 | `1` | 无左；压入 `2` | `[1]` |
| 2 | `2` | 压入 `3` | `[1,2]` |
| 3 | `3` | 无 | `[1,2,3]` |

`res[::-1]` → `[3,2,1]`，即后序。

## ⚠️ 常见陷阱

1. **与前序入栈顺序混淆**  
   - 前序要 **先压右再压左**（先访问左子树）。  
   - 反转法求后序要 **先压左再压右**（先走右子树，得到根右左）。二者**相反**，勿背混。

2. **单栈写法里过早访问根**  
   - ❌ 栈顶无左孩子就立刻 `pop` 并输出。  
   - ✅ 若存在右孩子且右子树尚未遍历完（`last is not peek.right`），应先 `cur = peek.right`。

3. **空树**  
   - 直接返回 `[]`；反转法中勿向栈压入 `None`。

4. **把中序模板硬套到后序**  
   - 中序是「左根右」的一路向左 + 弹栈访问；后序根在最后，除非使用 `last` / 双栈 / 标记，否则不能照搬。

## 复杂度分析

| 写法 | 时间 | 额外空间 | 说明 |
|------|------|----------|------|
| 递归 | O(n) | O(h) | 调用栈。 |
| 根右左 + 反转 | O(n) | O(h) 栈 + O(n) 结果缓冲 | 反转再拷贝一次，总体 O(n) 空间。 |
| 单栈 + `last` | O(n) | O(h) | 仅输出数组，无整段反转。 |

最坏链状树 h = n；平衡树 h = O(log n)。

---

## 关键点

1. **后序与删除、依赖顺序**：若子树处理完才能处理根（如自底向上算高度、收集路径），后序思想很常见。  
2. **N 叉树**：LeetCode [590](https://leetcode.cn/problems/n-ary-tree-postorder-traversal/) 同理，先遍历所有孩子再根。  
3. **面试选型**：先写递归；追问迭代时优先 **反转法**（好写），再问 **O(h) 栈** 再写 `last` 版。
