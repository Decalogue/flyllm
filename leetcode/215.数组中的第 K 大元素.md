# 215. 数组中的第 K 大元素

**标签**：`堆` `快速选择` `分治` `中等`  
**分类**：数组、TopK / 第 K 大  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频  

[力扣题目链接](https://leetcode.cn/problems/kth-largest-element-in-an-array/)

---

## 题目描述

给定整数数组 `nums` 和整数 `k`，请返回数组中第 **`k` 个最大的元素**。

请注意，你需要找的是数组在排序后第 `k` 个最大的元素，而不是第 `k` 个不同的元素。

你必须设计并实现时间复杂度为 `O(n)` 的算法解决此问题吗？——**不必须**；面试中通常要求能说清多种做法及复杂度。

### 示例 1

```
输入：nums = [3,2,1,5,6,4], k = 2
输出：5
```

### 示例 2

```
输入：nums = [3,2,3,1,2,4,5,5,6], k = 4
输出：4
```

### 提示

- `1 <= k <= nums.length <= 10^5`
- `-10^4 <= nums[i] <= 10^4`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：第 `k` 大 = 升序排序后下标 **`n - k`** 处的值；可用**快速选择**一次划到该下标，或用**大小为 `k` 的小根堆**保留当前最大的 `k` 个数。

**关键词**：`TopK` `小根堆` `快速选择` `partition` `下标换算 n-k`

### 复杂度速记

| 方法 | 时间 | 空间 |
|------|------|------|
| 排序取倒数第 k | O(n log n) | O(1)~O(n) |
| 小根堆容量 k | O(n log k) | O(k) |
| 快速选择（随机基准） | 平均 O(n)，最坏 O(n²) | O(1) 原地 |

### 记忆口诀

「第 k 大 → 升序里倒数第 k 个 → 下标 `n-k`；堆留 k 个大的用小根堆。」

---

## 解题思路

### 下标换算（必会）

设 `n = len(nums)`，升序排序后第 `k` 大元素在下标 **`n - k`**（0 起始）。  
因此「找第 k 大」等价于「找**第 `(n-k+1)` 小**」或「划分后落在下标 **`n-k`** 上的那个数」。

### 方法一：快速选择（原地，平均线性）

对区间做快排式 `partition`：随机基准，把 `≤ pivot` 的放左侧。设划分后基准下标为 `m`：

- `m == n - k`：当前元素即为答案；
- `m < n - k`：答案在右半段；
- `m > n - k`：答案在左半段。

代码里常令 `k = len(nums) - k`，目标下标即为 `k`，与下面实现一致。

### 方法二：小根堆（容量 k）

维护大小为 `k` 的**小根堆**：堆顶是这 `k` 个数里**最小**的，也就是全局第 `k` 大。遍历数组，堆未满则入堆；若当前数大于堆顶则替换堆顶。

### 方法三：排序

`sorted(nums)[-k]` 或 `sorted(nums, reverse=True)[k-1]`，作为 baseline。

## 代码实现

### 方法一：快速选择（推荐掌握 ⭐）

```python
import random


class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        k = len(nums) - k

        def part(lo: int, hi: int) -> int:
            p = random.randint(lo, hi)
            nums[p], nums[hi] = nums[hi], nums[p]
            i = lo
            for j in range(lo, hi):
                if nums[j] <= nums[hi]:
                    nums[i], nums[j] = nums[j], nums[i]
                    i += 1
            nums[i], nums[hi] = nums[hi], nums[i]
            return i

        lo, hi = 0, len(nums) - 1
        while True:
            m = part(lo, hi)
            if m == k:
                return nums[m]
            if m < k:
                lo = m + 1
            else:
                hi = m - 1
```

**要点**：`k` 重写为 `n - k` 后，求的是**升序第 `k` 小**（0 起始下标 `k`）。`part` 为 Lomuto 划分，`<= pivot` 放左侧，与「第 k 小」模板一致。

---

### 方法二：`heapq.nlargest`（简洁）

```python
import heapq


class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        return heapq.nlargest(k, nums)[-1]
```

---

### 方法三：手写小根堆（容量 k）

```python
import heapq


class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        h: list[int] = []
        for x in nums:
            if len(h) < k:
                heapq.heappush(h, x)
            elif x > h[0]:
                heapq.heapreplace(h, x)
        return h[0]
```

---

### 方法四：排序（Baseline）

```python
class Solution:
    def findKthLargest(self, nums: list[int], k: int) -> int:
        return sorted(nums)[-k]
```

## ⚠️ 常见陷阱

1. **堆的大小与类型**  
   第 `k` 大用**小根堆**保留 `k` 个；若用大根堆堆 `n` 个再弹 `k-1` 次，复杂度和常数都差。

2. **`k` 与 `n-k` 搞反**  
   快速选择里必须先明确：最终要固定的下标是 **`len(nums) - k`**（升序视角）。

3. **是否允许改原数组**  
   快速选择会打乱 `nums`；若不能修改，应先 `nums = nums.copy()` 或对副本操作。

4. **最坏时间**  
   固定基准或有序数组时快排划分可能退化；**随机 pivot**（或三数取中）是面试常考点。

## 复杂度对比表

| 方法 | 时间 | 空间 | 备注 |
|------|------|------|------|
| 排序 | O(n log n) | O(n) | 最短代码 |
| 小根堆 / `nlargest` | O(n log k) | O(k) | k 小时很常用 |
| 快速选择 | 平均 O(n)，最坏 O(n²) | O(1) | 题面若强调 O(n) 平均，答这个 |

## 总结

- **第 k 大** ↔ 升序下标 **`n - k`** ↔ 快速选择一次定位。  
- **工程常用**：`k` 不大用堆 O(n log k)；**面试深挖**：随机化快速选择平均 O(n)。  
- 与「最小 K 个数」共用 **partition** 骨架，差别在**目标下标**与**要一个值还是 k 个数**。
