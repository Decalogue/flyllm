# 139. 单词拆分

**标签**：`字典树` `记忆化搜索` `动态规划` `字符串` `哈希表` `中等`  
**分类**：线性 DP、字符串分割  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给你一个字符串 `s` 和一个字符串列表 `wordDict` 作为字典。请你判断是否可以利用字典中出现的单词**拼接出** `s`。

**要求**：

- 字典中的单词在拆分中**可以重复使用**；
- 字典中的单词**没有**重复（题目保证 `wordDict` 中字符串互不相同）。

若可以拆出整个 `s`，返回 `true`；否则返回 `false`。

### 示例 1

```
输入：s = "leetcode", wordDict = ["leet","code"]
输出：true
解释：可以用 "leet" 和 "code" 拼接成 "leetcode"。
```

### 示例 2

```
输入：s = "applepenapple", wordDict = ["apple","pen"]
输出：true
解释：可以用 "apple"、"pen"、"apple" 拼接（单词可重复使用）。
```

### 示例 3

```
输入：s = "catsandog", wordDict = ["cats","dog","sand","and","cat"]
输出：false
```

### 提示

- `1 <= s.length <= 300`
- `1 <= wordDict.length <= 1000`
- `1 <= wordDict[i].length <= 20`
- `s` 和 `wordDict[i]` 仅由小写英文字母组成
- `wordDict` 中的所有字符串**互不相同**

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：`dp[i]` 表示前缀 `s[0:i)`（长度为 `i`）能否被字典单词拼出；枚举**最后一段单词**的起点 `j`，若 `dp[j]` 为真且 `s[j:i]` 在字典中，则 `dp[i]=true`。

**关键词**：`线性 DP` `前缀` `最后一段` `字典哈希集合`

### 复杂度速记

- **朴素双层循环**：时间 **O(n²)** 次转移；每次子串 `s[j:i]` 与集合比较，若算上子串长度最坏 **O(n³)**（实现相关）。  
- **按最大词长截断**：只让 `j` 在 `[i-maxLen, i)` 内枚举，时间 **O(n · maxLen)**，**maxLen ≤ 20** 时常数很小。

### 记忆口诀

「前缀能拼 + 尾巴在词典 → 更长前缀也能拼。」

---

## 解题思路

### 状态定义

- `dp[i]`：`s` 的前 `i` 个字符组成的子串 `s[0:i]`（即 `s[:i]`）能否由 `wordDict` 中的词拼出。  
- 边界：`dp[0] = True`（空串视为可拼）。  
- 目标：`dp[n]`，`n = len(s)`。

### 转移

对 `i = 1 .. n`，枚举分割点 `j`（`0 <= j < i`）：

- 若 `dp[j] == True` 且 `s[j:i]` 出现在字典中，则 `dp[i] = True`，并可 `break`。

含义：前 `j` 个字符已可拼，剩下一段正好是字典里的一个词。

### 优化：限制枚举长度

记 `maxLen = max(len(w) for w in wordDict)`。最后一段词长不超过 `maxLen`，故只需枚举：

\[
j \in [\max(0,\, i - \max\text{Len}),\, i)
\]

这样内层循环至多 `maxLen` 次，总时间 **O(n · maxLen)**，本题 `maxLen ≤ 20` 非常划算。

### 其他思路（口述即可）

- **BFS**：把「当前已匹配前缀长度」看作结点，从 `0` 出发，能接词典词则连边，看能否到达 `n`。本质与 DP 相同，状态仍是前缀长度。  
- **记忆化 DFS**：`dfs(i)` 表示从 `i` 出发能否拼完，枚举下一个词；与 DP 一一对应。  
- **字典树**：词典极大、公共前缀多时可加速「从位置 `i` 能匹配哪些前缀」；Hot 100 数据规模下 **哈希集合 + 截断 j** 足够。

## 代码实现

### 写法一：DP + 哈希集合（推荐）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: list[str]) -> bool:
        ws = set(wordDict)
        n = len(s)
        dp = [False] * (n + 1)
        dp[0] = True
        for i in range(1, n + 1):
            for j in range(i):
                if dp[j] and s[j:i] in ws:
                    dp[i] = True
                    break
        return dp[n]
```

### 写法二：DP + 按最大词长截断（更优常数）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: list[str]) -> bool:
        ws = set(wordDict)
        max_len = max(len(w) for w in wordDict)
        n = len(s)
        dp = [False] * (n + 1)
        dp[0] = True
        for i in range(1, n + 1):
            start = max(0, i - max_len)
            for j in range(start, i):
                if dp[j] and s[j:i] in ws:
                    dp[i] = True
                    break
        return dp[n]
```

## ⚠️ 常见陷阱

1. **`dp` 长度用 `n+1`**  
   - `dp[i]` 对应前缀长度 `i`，下标 `n` 才是整串。

2. **误以为单词只能用一次**  
   - 题目允许**重复使用**同一单词（见示例 2）。

3. **只检查字典词是否是 `s` 的子串**  
   - 错误：需要**首尾相接拼满整个 `s`**，必须用 DP / 搜索 做**完整分割**判定。

4. **复杂度口误**  
   - 写「O(n²)」时建议说明：Python 中 `s[j:i]` 切片有长度代价；面试可主动提 **maxLen 截断** 到 **O(n·maxLen)**。

---

## 复杂度分析

### 写法一

- **时间**：`i` 与 `j` 双重循环 **O(n²)**；每次 `s[j:i] in ws` 若创建子串 **O(i-j)**，最坏整体可达 **O(n³)**。实际数据常仍可接受。  
- **空间**：`O(n)` 的 `dp`，字典 **`O(词典总字符)`** 或存 `set` 引用。

### 写法二

- **时间**：**O(n · maxLen)**，`maxLen` 为词典中最长单词长度；本题 **maxLen ≤ 20**。  
- **空间**：同上 **O(n)**。

## 总结

139 是标准的**前缀线性 DP**：`dp[i]` 由更短前缀 `dp[j]` 与「尾词在不在词典」转移；实现上用 **`set` 判词** + **`maxLen` 截断枚举 `j`** 即可兼顾简洁与效率。进阶追问可答 BFS、记忆化 DFS 或 Trie。
