## 题目描述

给你一个整数数组 `nums`，有一个大小为 `k` 的滑动窗口从数组最左侧移到最右侧。你只可以看到窗口内的 k 个数字，窗口每次向右移动一位。返回每个窗口中的最大值。

### 示例 1

```
输入：nums = [1,3,-1,-3,5,3,6,7], k = 3
输出：[3,3,5,5,6,7]
解释：
滑动窗口位置                最大值
[1  3  -1] -3  5  3  6  7    3
 1 [3  -1  -3] 5  3  6  7    3
 1  3 [-1  -3  5] 3  6  7    5
 1  3  -1 [-3  5  3] 6  7    5
 1  3  -1  -3 [5  3  6] 7    6
 1  3  -1  -3  5 [3  6  7]   7
```

### 示例 2

```
输入：nums = [1], k = 1
输出：[1]
```

### 提示

- 1 <= nums.length <= 10^5
- -10^4 <= nums[i] <= 10^4
- 1 <= k <= nums.length

## 📋 面试要点速查（知识卡片）

### 核心思路
**一句话总结**：用 **单调递减队列**（deque 存**下标**，队头到队尾对应值单调递减），队头即为当前窗口最大值下标；新元素入队时从队尾弹出所有比它小的下标，队头下标出窗时 popleft。

**关键词**：`单调队列` `deque` `存下标` `队头即最大值`

### 复杂度速记
- **时间复杂度**：O(n) - 每个下标最多入队、出队各一次
- **空间复杂度**：O(k) - 单调队列最多 k 个下标

### 记忆口诀
「单调递减队，队头是最大；入队前队尾弹小，出窗就队头弹。」

### 代码模板（可直接套用）

```python
from collections import deque

# 单调递减队列：dq 存下标，队头到队尾对应 nums 值单调递减
dq = deque()
res = []
for i in range(len(nums)):
    # 队头已出窗则弹出
    if dq and dq[0] == i - k:
        dq.popleft()
    # 队尾比当前小则弹出（它们不可能再成为后续窗口最大值）
    while dq and nums[dq[-1]] < nums[i]:
        dq.pop()
    dq.append(i)
    # 形成完整窗口后记录队头（当前窗口最大值）
    if i >= k - 1:
        res.append(nums[dq[0]])
return res
```

## 解题思路

### 核心思想

维护一个**单调递减的双端队列**（队头到队尾，对应 `nums` 的值单调递减）：
- **队头**：当前窗口内最大值的**下标**，即窗口最大值为 `nums[dq[0]]`。
- **入队**：新下标 i 入队前，从**队尾**弹出所有满足 `nums[队尾下标] < nums[i]` 的下标（这些值之后都不可能再当最大值）。
- **出窗**：若队头下标等于 `i - k`，说明该下标已离开窗口，从**队头** popleft。

**关键点：**
- **存下标不存值**：便于判断队头是否出窗（`dq[0] == i - k`）；若存值，有重复时无法正确判断谁出窗。
- **单调递减**：保证队头是当前窗口最大；新元素从队尾入队前弹掉比它小的，保持递减。

### 算法步骤

1. 使用 `collections.deque` 存**下标**，保证队头到队尾对应 `nums` 值单调递减。
2. 从左到右遍历 `i`：
   - 若 `dq` 非空且 `dq[0] == i - k`，队头出窗，`dq.popleft()`。
   - 若 `dq` 非空且 `nums[dq[-1]] < nums[i]`，从队尾 `dq.pop()`，直到队尾 ≥ `nums[i]` 或队空。
   - `dq.append(i)`。
   - 若已形成完整窗口（`i >= k - 1`），`res.append(nums[dq[0]])`。
3. 返回 `res`。

### 为什么选择单调队列？

- **优势**：每个下标最多入队、出队各一次，总 O(n)；窗口最大值 O(1) 取队头。
- **适用场景**：滑动窗口最值、区间最值。
- **替代方案**：暴力每窗口扫 k 个取 max，O(nk)；或大根堆 + 懒删除，O(n log n)。

## 代码实现

### 单调递减队列（推荐 ⭐）

```python
from collections import deque
from typing import List

class Solution:
    def maxSlidingWindow(self, nums: List[int], k: int) -> List[int]:
        """
        单调递减队列：dq 存下标，队头到队尾对应值单调递减，队头即当前窗口最大值下标。
        队头出窗则 popleft；新元素入队前从队尾弹出所有比它小的下标。
        时间复杂度: O(n)
        空间复杂度: O(k)
        """
        if not nums or k <= 0:
            return []
        dq = deque()
        res = []
        for i in range(len(nums)):
            if dq and dq[0] == i - k:
                dq.popleft()
            while dq and nums[dq[-1]] < nums[i]:
                dq.pop()
            dq.append(i)
            if i >= k - 1:
                res.append(nums[dq[0]])
        return res
```

**代码要点**：
- ✅ **存下标**：`dq` 里是下标，出窗判断 `dq[0] == i - k`，取值用 `nums[dq[0]]`。
- ✅ **单调递减**：从队尾弹出比当前 `nums[i]` 小的下标，再 `append(i)`。
- ✅ 只有 `i >= k - 1` 时才往 `res` 里追加（窗口已完整）。

### 执行过程示例

以 `nums = [1,3,-1,-3,5,3,6,7], k = 3` 为例：

- i=0：dq=[0]，未形成窗口。
- i=1：3>1，弹 0，dq=[1]；未形成窗口。
- i=2：-1<3，不弹，dq=[1,2]；i>=2，res=[nums[1]]=[3]。
- i=3：-3<-1，不弹，dq=[1,2,3]；队头 1 未出窗；res=[3,3]。
- i=4：5 入队前弹 3、2、1（都<5），dq=[4]；res=[3,3,5]。
- i=5：3<5 不弹，dq=[4,5]；res=[3,3,5,5]。
- i=6：6 入队前弹 5、4，dq=[6]；res=[3,3,5,5,6]。
- i=7：7 入队前弹 6，dq=[7]；res=[3,3,5,5,6,7]。

## ⚠️ 常见陷阱（面试易错点）

1. **存值不存下标**
   - ❌ deque 里存 `nums[i]`，出窗时用 `dq[0] == nums[i-k]`：若窗口内有重复值，会误弹或弹错。
   - ✅ 存**下标**，出窗判断 `dq[0] == i - k`，取值时用 `nums[dq[0]]`。

2. **出窗判断写错**
   - ❌ 写成 `dq[0] < i - k` 或 `dq[0] <= i - k`（应严格等于才弹）。
   - ✅ 队头下标等于 `i - k` 时说明该下标刚好离开窗口，`dq.popleft()`。

3. **边界**
   - `k == 1`：可单独返回 `list(nums)` 或交给通用逻辑（第一次就形成窗口）。
   - 空数组或 `k <= 0`：返回 `[]`。

## 复杂度分析

### 时间复杂度
- 每个下标最多入队一次、出队一次，总 O(n)。

### 空间复杂度
- 单调队列内最多 k 个下标，O(k)。

### 复杂度对比表

| 方法           | 时间复杂度 | 空间复杂度 | 备注     |
|----------------|------------|------------|----------|
| 单调队列（推荐）| O(n)       | O(k)       | 标准解法 |
| 暴力           | O(nk)      | O(1)       | 每窗口扫 k 个 |
| 大根堆+懒删除  | O(n log n) | O(k)       | 实现稍复杂 |

## 关键点

1. **为什么是单调「递减」队列？**
   - 要的是**窗口最大值**：队头保持为当前窗口内最大值的下标；新元素比队尾大时，队尾那些在后续窗口里不可能再当最大值，从队尾弹掉，保持从队头到队尾「值」递减。

2. **和单调栈的区别**
   - 单调栈没有「出窗」：元素入栈后只会在后面被弹出，不会从头部移除。
   - 单调队列有「窗口」：队头可能因下标超出窗口而 popleft，所以要用 deque，支持 O(1) 队头删除。

## 💬 面试问题（模拟面试场景）

### 基础问题

1. **Q: 为什么 deque 里存下标不存值？**
   - A: 出窗时要判断「哪个下标离开了窗口」，只有存下标才能精确判断 `dq[0] == i - k`；存值的话有重复数时无法知道该弹谁。

2. **Q: 为什么从队尾弹、从队头弹？**
   - A: 新元素从右边进窗口，所以从队尾入队；入队前从队尾弹出比它小的，保证单调递减。窗口左边界左移时，最大值的下标可能离开窗口，所以从队头 popleft。

3. **Q: 时间复杂度为什么是 O(n)？**
   - A: 每个下标最多入队一次、出队一次，总操作 2n，所以 O(n)。
