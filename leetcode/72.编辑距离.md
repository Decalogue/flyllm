# 72. 编辑距离

**标签**：`字符串` `动态规划` `困难`  
**分类**：双串前缀 DP、最小编辑步数  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 高频

---

## 题目描述

给你两个字符串 `word1` 和 `word2`，返回将 `word1` 转换成 `word2` 所使用的**最少操作数**。

你可以进行以下三种操作（每次计 **1** 步）：

1. **插入**一个字符；  
2. **删除**一个字符；  
3. **替换**一个字符。

### 示例 1

```
输入：word1 = "horse", word2 = "ros"
输出：3
解释：
horse -> rorse（将 'h' 替换为 'r'）
rorse -> rose（删除 'r'）
rose -> ros（删除 'e'）
```

### 示例 2

```
输入：word1 = "intention", word2 = "execution"
输出：5
```

### 示例 3

```
输入：word1 = "", word2 = ""
输出：0
```

### 提示

- `0 <= word1.length, word2.length <= 500`
- `word1` 和 `word2` 由小写英文字母组成

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：`dp[i][j]` 表示 `word1` 的前 `i` 个字符变成 `word2` 的前 `j` 个字符的**最少操作数**；末位相同则 `dp[i][j]=dp[i-1][j-1]`；否则在三类操作上取最优：**删 / 插 / 换** 各对应 `dp[i-1][j]`、`dp[i][j-1]`、`dp[i-1][j-1]` 加 1。

**关键词**：`双串前缀 DP` **`dp` 里是长度、字符下标要减 1`** `删插换对应上左左上` `边界一行一列`

### 复杂度速记

- **时间**：O(m·n)。  
- **空间**：O(m·n) 二维表；可滚为 **O(n)** 一行（需保留左上对角 `prev`）。

### 记忆口诀

「相等抄对角；不等三选一，各加一步。」

---

## 解题思路

### 状态定义

- `dp[i][j]`：`word1[0:i]`（长度为 `i` 的前缀）变为 `word2[0:j]` 的最少操作数。  
- 下标约定：`i ∈ [0, m]`，`j ∈ [0, n]`，`m = len(word1)`，`n = len(word2)`。

### 边界（空串）

- `dp[i][0] = i`：把长度为 `i` 的串变成空串，只能 **i 次删除**。  
- `dp[0][j] = j`：空串变成长度 `j`，只能 **j 次插入**。

### `dp[i][j]` 与 `i-1`、`j-1`：下标怎么对齐（最易错）

**先分清两套下标：**

| 记号 | 含义 | 取值 |
|------|------|------|
| **`dp[i][j]` 里的 `i`、`j`** | 两个前缀的**长度**（各有几个字符要处理） | `0 … m`、`0 … n` |
| **`word1[i-1]`、`word2[j-1]`** | 各自前缀里**最后一个字符**在字符串中的**下标** | 仅当 `i≥1`、`j≥1` 时有意义 |

因此：**`dp[i][j]` 讨论的是「`word1` 前 `i` 个」和「`word2` 前 `j` 个」**；这一对前缀的**末位字符**正好是 **`word1[i-1]`** 与 **`word2[j-1]`**（下标从 0 数，长度为 `i` 的末位下标是 `i-1`）。  
常见错法是把 `i` 当成「`word1` 当前字符下标」，于是误以为该和 `word1[i]` 比——**不对**，表里一行 `i` 对应的是**长度**，代码里才写成 **`i-1`** 取末字符。

**用表格想「子问题」**（通常**第 `i` 行**对应 `word1` 前 `i` 个字符，**第 `j` 列**对应 `word2` 前 `j` 个）：

- **`dp[i][j]`**：子问题是 **`word1[0:i] → word2[0:j]`**。  
- **`dp[i-1][j]`**：左边只取前 **`i-1`** 个字符，右边仍要 **`j` 个** → 表里 **往上** 一行（`i` 减 1，`j` 不变）。  
- **`dp[i][j-1]`**：左边仍 **`i` 个**，右边只要求 **`j-1` 个** → **往左** 一列（`i` 不变，`j` 减 1）。  
- **`dp[i-1][j-1]`**：两边各缩掉一个末位 → **往左上**（`i`、`j` 各减 1）。

**记一句**：**`dp` 里写的是长度 `i,j`；比字符相等要写 `word1[i-1]`、`word2[j-1]`。不等时三种走法，正好对应「上、左、左上」三个格子。**

### 转移

- 若 **`word1[i-1] == word2[j-1]`**：两前缀末位已相同，**不用对这两个末位再做操作**，`dp[i][j] = dp[i-1][j-1]`。  
- 否则必须对「`word1` 的末位 / `word2` 的末位 / 两者关系」动 **1** 步，且上一步子问题只能是 **`dp[i-1][j]`、`dp[i][j-1]`、`dp[i-1][j-1]`** 之一（对应删、插、换），取最小：

\[
dp[i][j] = 1 + \min\big(dp[i-1][j],\, dp[i][j-1],\, dp[i-1][j-1]\big)
\]

### 不等时三项与「上、左、左上」一一对应

前提：`word1[i-1] ≠ word2[j-1]`。下面每条都是：**先做 1 步操作，再解决更小矩形上的子问题**。

1. **`dp[i-1][j] + 1` —— 删**  
   - 删掉 `word1[i-1]` 后，左边剩**长度 `i-1`**，目标仍是 **长度 `j`** → **`dp[i-1][j]`**。  
   - 在表里：**从 `(i,j)` 往上一格**。

2. **`dp[i][j-1] + 1` —— 插**  
   - 理解为：先把 **`word1` 前 `i` 个** 变成 **`word2` 前 `j-1` 个**（子问题 **`dp[i][j-1]`**），再在末尾**插入** `word2[j-1]`，凑齐长度 `j`。  
   - 注意：子问题里 **左边长度仍是 `i`**（还没为「插入」缩短左边），变短的是**右边目标** → **`dp[i][j-1]`**。  
   - 在表里：**从 `(i,j)` 往左一格**。

3. **`dp[i-1][j-1] + 1` —— 换**  
   - 把 `word1[i-1]` 换成 `word2[j-1]`，两边末位同时消掉一层 → **长度各减 1** → **`dp[i-1][j-1]`**。  
   - 在表里：**从 `(i,j)` 往左上**。

**为何插入是 `dp[i][j-1]` 而不是 `dp[i-1][j]`？**  
插入一步**不减少**「已构造好的 `word1` 那一段」在 DP 里的长度 `i`（你是在这一侧**加长**）；减少的是「还差多少没对齐到 `word2`」——先对齐到 **`j-1`**，再插 1 个字符完成 **`j`**。若误想成「插完左边变短」就会错配成别的状态。

三者穷尽了「最后一步」的所有可能，故取 **`min`**。末位已相等时不需要这 1 步，**不要**再套这个公式。

### 与 LCS 的对比（口述）

[1143. 最长公共子序列](1143.最长公共子序列.md) 在不等时是 **`max` 两方向**，因为子序列可跳过字符；编辑距离在不等时要付 **1 步代价** 并在 **删 / 插 / 换** 间取 **min**，模型不同但都是「双串前缀」填表。

## 代码实现

### 写法一：二维 DP（推荐理解与面试默写）

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        dp = [[0] * (n + 1) for _ in range(m + 1)]
        for i in range(m + 1):
            dp[i][0] = i
        for j in range(n + 1):
            dp[0][j] = j
        for i in range(1, m + 1):
            for j in range(1, n + 1):
                if word1[i - 1] == word2[j - 1]:
                    dp[i][j] = dp[i - 1][j - 1]
                else:
                    dp[i][j] = 1 + min(
                        dp[i - 1][j],      # 删除 word1[i-1]
                        dp[i][j - 1],      # 插入（对齐 word2[j-1]）
                        dp[i - 1][j - 1],  # 替换
                    )
        return dp[m][n]
```

### 写法二：一维滚动（O(n) 空间）

内层 `j` **从小到大**；处理 `dp[j]` 前先保存「左上」`diag = dp[j]`（更新前的 `dp[j]` 即上一行的 `dp[i-1][j-1]`），或使用临时变量轮换。

```python
class Solution:
    def minDistance(self, word1: str, word2: str) -> int:
        m, n = len(word1), len(word2)
        dp = list(range(n + 1))
        for i in range(1, m + 1):
            diag = dp[0]
            dp[0] = i
            for j in range(1, n + 1):
                nd = dp[j]
                if word1[i - 1] == word2[j - 1]:
                    dp[j] = diag
                else:
                    dp[j] = 1 + min(dp[j], dp[j - 1], diag)
                diag = nd
        return dp[n]
```

## ⚠️ 常见陷阱

1. **把 `dp[i][j]` 的 `i` 当成 `word1` 的「当前字符下标」**  
   - **`i` 是前缀长度**；末位字符必须是 **`word1[i-1]`**。错用 `word1[i]` 会导致整表偏移。

2. **边界只初始化一行**  
   - 必须同时初始化 **`dp[i][0]`** 与 **`dp[0][j]`**，否则空串情况错。

3. **相等时多加了 1**  
   - 末位相同应 **直接继承** `dp[i-1][j-1]`，不要再 `+1`。

4. **三种操作的含义搞反**  
   - `dp[i-1][j]`：**删** `word1` 末位；`dp[i][j-1]`：**先**对齐到 `word2` 少一位 **再插**（见上文「为何插入是 `dp[i][j-1]`」）。

5. **与「最长公共子序列」转移混用**  
   - LCS 用 `max`；编辑距离在不等时用 **`min` + 1** 与三个前驱。

## 复杂度分析

- **时间复杂度**：O(m·n)。  
- **空间复杂度**：O(m·n)（二维）或 O(min(m, n))（可对较短串长度滚动，减少一行长度）。
