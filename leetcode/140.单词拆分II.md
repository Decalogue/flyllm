# 140. 单词拆分 II

**标签**：`字典树` `记忆化搜索` `动态规划` `回溯` `字符串` `困难`  
**分类**：回溯、记忆化、字符串分割  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 中频

---

## 题目描述

给定一个**不含重复单词**的字符串列表 `wordDict` 和一个字符串 `s`，返回 `s` 由 `wordDict` 中出现的单词**按任意顺序、可重复使用**拼接而成的**所有可能句子**。单词之间用**单个空格**连接。

- 不要求字典里每个单词都用上。  
- 若不存在合法拆分，返回**空列表** `[]`。  
- 答案中**同一句子**只出现一次；返回顺序**任意**均可。

本题是 [139. 单词拆分](139.单词拆分.md) 的加强版：139 只判**能否**拆，140 要列出**所有**拆法。

### 示例 1

```
输入：s = "catsanddog", wordDict = ["cat","cats","and","sand","dog"]
输出：["cats and dog","cat sand dog"]
```

### 示例 2

```
输入：s = "pineapplepenapple", wordDict = ["apple","pen","applepen","pine","pineapple"]
输出：[
  "pine apple pen apple",
  "pineapple pen apple",
  "pine applepen apple"
]
```

### 示例 3

```
输入：s = "catsandog", wordDict = ["cats","dog","sand","and","cat"]
输出：[]
```

### 提示

- `1 <= s.length <= 20`（数据较小，与「合法句子数可能指数级」有关）
- `1 <= wordDict.length <= 1000`
- `1 <= wordDict[i].length <= 10`
- `s` 和 `wordDict[i]` 仅由小写英文字母组成
- `wordDict` 中所有字符串**互不相同**

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：从下标 `i` 开始 DFS：枚举**下一个词**的结束位置 `j`，若 `s[i:j]` 在字典中，则把该词接到 `dfs(j)` 返回的**所有后缀句**前面；用 **记忆化**（`@lru_cache` 或 `dict`）避免重复子问题。

**关键词**：`记忆化 DFS` `前缀枚举` `指数级输出` `可选 DP 剪枝`

### 复杂度速记

- **时间**：最坏**指数级**（合法句子数可指数增长）；记忆化保证**每个后缀起点 `i` 只扩展一次**，单次扩展要枚举 `j` 并拼接结果。  
- **空间**：递归栈 O(n) + 记忆化存各起点的结果列表；输出本身也可能很大。

### 记忆口诀

「从 i 切词进字典，后缀递归拼句子；memo[i] 存到底，重复起点不重来。」

---

## 解题思路

### 与 139 的关系

- **139**：`dp[i]` 表示前缀 `s[0:i)` 是否可拆 → **布尔**。  
- **140**：`dfs(i)` 表示后缀 `s[i:]` 能拼出的**所有句子** → **列表**，结构相同但信息更细。

### 算法：记忆化 DFS

1. 将 `wordDict` 放入 **`set`**，O(1) 判断子串是否为词。  
2. 定义 `dfs(i)`：表示从位置 `i` 开始到串尾的所有合法句子（字符串列表）。  
3. **边界**：`i == n` 时，只有一种「空后缀」，返回 `[""]`（便于统一拼接：空串表示已拆完）。  
4. **转移**：对每个 `j ∈ (i, n]`，若 `s[i:j] in wordDict`，对每个 `tail in dfs(j)`，若 `tail` 为空则句子为 `word`，否则为 `word + " " + tail`。  
5. **记忆化**：`memo[i]` 缓存 `dfs(i)` 的返回值；同一 `i` 被多条路径访问时直接返回。

### 可选剪枝：后缀可拆性（139 式 DP）

在开始 DFS 前，用类似 139 的 DP 从右向左算 **`ok[i]`**：`s[i:]` **是否至少存在一种**合法拆分。若 `ok[i]` 为假，`dfs(i)` 直接返回 `[]`，可减少无效枚举。  
数据很小时非必须，面试可口述「先判可达再搜」。

### 实现注意

- 返回值用 **`tuple` + `lru_cache`** 时需注意：`lru_cache` 要求返回值可哈希；或改用 **`dict[int, list[str]]`** 手写记忆化更直观。  
- 子串用 `s[i:j]`，配合 **`maxLen`** 只枚举 `j` 到 `i+maxLen`，常数更小。

## 代码实现

### 写法一：记忆化 DFS（`functools.lru_cache`）

`lru_cache` 的返回值需为**可哈希**类型，故内部返回 **`tuple[str, ...]`**，最外层再 `list(...)`。

```python
from functools import lru_cache


class Solution:
    def wordBreak(self, s: str, wordDict: list[str]) -> list[str]:
        ws = set(wordDict)
        n = len(s)
        max_len = max(len(w) for w in wordDict)

        @lru_cache(maxsize=None)
        def dfs(i: int) -> tuple[str, ...]:
            if i == n:
                return ("",)
            out: list[str] = []
            upper = min(n, i + max_len)
            for j in range(i + 1, upper + 1):
                w = s[i:j]
                if w not in ws:
                    continue
                for tail in dfs(j):
                    out.append(w if not tail else f"{w} {tail}")
            return tuple(out)

        return list(dfs(0))
```

### 写法二：手写 `memo`（返回 `list[str]`，易读）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: list[str]) -> list[str]:
        ws = set(wordDict)
        n = len(s)
        max_len = max(len(w) for w in wordDict)
        memo: dict[int, list[str]] = {}

        def dfs(i: int) -> list[str]:
            if i in memo:
                return memo[i]
            if i == n:
                return [""]
            res: list[str] = []
            upper = min(n, i + max_len)
            for j in range(i + 1, upper + 1):
                w = s[i:j]
                if w not in ws:
                    continue
                for tail in dfs(j):
                    res.append(w if tail == "" else w + " " + tail)
            memo[i] = res
            return res

        return dfs(0)
```

**说明**：`i == n` 时返回 `[""]` 仅作拼接占位；题目保证 `s` 非空时，`dfs(0)` 得到的是完整句子列表，不会出现仅空串的答案。

### 写法三：DFS + 后缀 DP 剪枝（口述加分项）

```python
class Solution:
    def wordBreak(self, s: str, wordDict: list[str]) -> list[str]:
        ws = set(wordDict)
        n = len(s)
        max_len = max(len(w) for w in wordDict)

        # ok[i]：s[i:] 是否存在至少一种合法拆分
        ok = [False] * (n + 1)
        ok[n] = True
        for i in range(n - 1, -1, -1):
            lim = min(n, i + max_len)
            for j in range(i + 1, lim + 1):
                if s[i:j] in ws and ok[j]:
                    ok[i] = True
                    break

        memo: dict[int, list[str]] = {}

        def dfs(i: int) -> list[str]:
            if not ok[i]:
                return []
            if i in memo:
                return memo[i]
            if i == n:
                return [""]
            res: list[str] = []
            lim = min(n, i + max_len)
            for j in range(i + 1, lim + 1):
                w = s[i:j]
                if w not in ws:
                    continue
                for tail in dfs(j):
                    res.append(w if tail == "" else w + " " + tail)
            memo[i] = res
            return res

        return dfs(0)
```

## ⚠️ 常见陷阱

1. **忘记记忆化**  
   - 无 `memo` 时同一后缀起点会被重复搜索，超时。

2. **边界返回值**  
   - `i == n` 用 `[""]` 占位，拼接时 `word + " " + ""` 需写成只 `word`（避免尾部空格）。

3. **与 139 混淆**  
   - 140 要**所有方案**，不能只返回布尔；但可用 139 思路做**剪枝**。

4. **输出去重**  
   - 题目保证字典无重复词、从固定 `s` 生成，一般不会重复句子；若自定义数据有歧义路径产生同一句子，需 `set` 去重。

5. **`lru_cache` 与 `list`**  
   - 缓存要求可哈希，不能对返回 `list` 的函数直接用 `lru_cache`（需 `tuple` 或手写 `memo`）。

---

## 复杂度分析

- **时间**：记 `W` 为词典最长词长。每个起点 `i` 至多枚举 `O(W)` 个 `j`；记忆化共 `O(n)` 个状态。但**拼接与复制**所有句子时，总时间受**输出规模**支配，最坏指数级。  
- **空间**：`O(n)` 递归栈 + 记忆化中存储的所有中间字符串；同样受输出总量影响。

## 总结

140 在 139 的「前缀可拆」模型上，把布尔改成**带记忆化的 DFS**，从位置 `i` 枚举下一个词并合并后缀所有句子；配合 **`maxLen` 限制 `j`** 控制常数，必要时用**后缀可达 DP**剪枝。面试优先写**手写 `memo` + list**，清晰可讲；追求代码行数可用 **`lru_cache` + tuple**。
