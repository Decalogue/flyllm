# 213. 打家劫舍 II

**标签**：`数组` `动态规划` `中等`  
**分类**：环形 DP、拆区间、线性打家劫舍  
**难度**：⭐⭐ 中等  
**频率**：🔥🔥🔥 高频

---

## 题目描述

与 [198. 打家劫舍](198.打家劫舍.md) 相同规则：不能偷**相邻**两间房。区别是：房屋排成**环形**，即**第一间与最后一间也是邻居**（同时偷首尾会报警）。

给定非负整数数组 `nums`，求在不触发报警的前提下，一夜能偷到的**最大金额**。

### 示例 1

```
输入：nums = [2,3,2]
输出：3
解释：不能同时偷 2（首）和 2（尾），最优为只偷中间 3。
```

### 示例 2

```
输入：nums = [1,2,3,1]
输出：4
解释：偷第 1 间与第 3 间（下标 0 与 2），1 + 3 = 4。
```

### 示例 3

```
输入：nums = [1,2,3]
输出：3
```

### 提示

- `1 <= nums.length <= 100`
- `0 <= nums[i] <= 1000`

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：环形只多约束「**首尾不能都选**」；任意最优解要么**不偷最后一间**（在 `nums[0..n-2]` 上做线性劫舍），要么**不偷第一间**（在 `nums[1..n-1]` 上做线性劫舍），取二者较大值；**单间**单独返回。

**关键词**：`环形拆两段` `两次 198` `分类讨论首尾`

### 复杂度速记

- **时间**：O(n)，两次线性 DP，每次 O(区间长)。  
- **空间**：O(1)，滚动变量。

### 记忆口诀

「环变线：舍头一段、舍尾一段，两次线性取 max。」

---

## 解题思路

### 为何可以拆成两段

设下标 `0` 与 `n-1` 为环上的首尾。任一合法方案里：

- 若**不偷**最后一间 `n-1`，则问题等价于在**直线** `nums[0 .. n-2]` 上做 [198](198.打家劫舍.md)；  
- 若**偷**最后一间，则**一定不能偷**第一间 `0`，问题等价于在直线 `nums[1 .. n-1]` 上做 198。

这两种情况**穷尽了所有合法方案**（偷尾 ⇒ 不偷首；不偷尾 ⇒ 首可偷可不偷，已包含在第一种区间的最优里）。因此：

\[
\text{answer} = \max\big(\mathrm{rob\_linear}(nums[0:n-1]),\; \mathrm{rob\_linear}(nums[1:n])\big)
\]

### 边界

- **`len(nums) == 1`**：没有「相邻的另一头」，直接返回 `nums[0]`。若仍做 `max(linear[:-1], linear[1:])`，会得到 `max(0,0)=0`，**错误**，必须先判。

### 线性子问题

子数组上的最优即 198：`prev2, prev1` 滚动，`cur = max(prev1, prev2 + x)`。

---

## 代码实现

### 推荐写法（抽取 `linear` + 特判 `n==1`）

```python
class Solution:
    def rob(self, nums: list[int]) -> int:
        if len(nums) == 1:
            return nums[0]

        def linear(arr: list[int]) -> int:
            prev2, prev1 = 0, 0
            for x in arr:
                prev2, prev1 = prev1, max(prev1, prev2 + x)
            return prev1

        return max(linear(nums[:-1]), linear(nums[1:]))
```

### 与 198 相同的 `a, b` 滚动（内层）

```python
def linear(arr: list[int]) -> int:
    a = b = 0
    for x in arr:
        a, b = b, max(b, a + x)
    return b
```

---

## ⚠️ 常见陷阱

1. **忘记 `n == 1`**  
   - `nums[:-1]` 与 `nums[1:]` 都为空，`linear` 返回 0，答案错。

2. **误以为要偷满整条街**  
   - 可以只偷一间或隔多间，转移仍是 `max(不偷当前, 偷当前 + 前前最优)`。

3. **尝试在环上直接写 `dp[i]` 依赖「模 n」**  
   - 容易绕晕；**拆两段**最清晰、也是面试标准答法。

4. **与 198 混淆**  
   - 198 无首尾相邻；213 必须处理**首 OR 尾**二选一（体现在两段区间上）。

---

## 复杂度分析

- **时间复杂度**：O(n) — 两次线性扫描，总长约 `2n`。  
- **空间复杂度**：O(1) — 仅滚动变量；子数组用切片会复制，若要求严格 O(1) 可改为传入 `(nums, start, end)` 下标范围在原地扫描（面试一般接受切片）。

## 总结

213 在 198 的转移不变前提下，用**分类讨论**把**环形**压成**两条线段**：不含尾、不含首，各做一次线性劫舍再取 **max**，并牢记 **`n == 1` 特判**。这是环形「相邻不取」类问题的常用套路。
