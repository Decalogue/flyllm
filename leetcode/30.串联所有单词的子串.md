# 30. 串联所有单词的子串

**标签**：`字符串` `滑动窗口` `哈希表` `困难`  
**分类**：滑动窗口、哈希、字符串  
**难度**：⭐⭐⭐ 困难  
**频率**：🔥🔥 中高频

---

## 题目描述

给定一个字符串 `s` 和一个字符串数组 `words`。`words` 中所有字符串**长度相同**。

`s` 中的**串联子串**是指一个包含 `words` 中**所有**字符串、以**任意顺序**排列连接起来的子串。

例如，若 `words = ["ab","cd","ef"]`，则 `"abcdef"`、`"abefcd"`、`"cdabef"`、`"cdefab"`、`"efabcd"`、`"efcdab"` 都是串联子串；`"acdbef"` 不是，因为不是任意 words 排列的连接。

返回所有串联子串在 `s` 中的**开始索引**。可以以任意顺序返回答案。

### 示例 1

```
输入：s = "barfoothefoobarman", words = ["foo","bar"]
输出：[0,9]
解释：words.length == 2 且 words[i].length == 3，串联子串长度必须为 6。
"barfoo" 从 0 开始；"foobar" 从 9 开始。
```

### 示例 2

```
输入：s = "wordgoodgoodgoodbestword", words = ["word","good","best","word"]
输出：[]
解释：串联子串长度必须为 16，s 中不存在长度为 16 且等于 words 某排列连接的子串。
```

### 示例 3

```
输入：s = "barfoofoobarthefoobarman", words = ["bar","foo","the"]
输出：[6,9,12]
解释：串联子串长度必须为 9。"foobarthe"、"barthefoo"、"thefoobar" 分别从 6、9、12 开始。
```

### 提示

- `1 <= s.length <= 10^4`
- `1 <= words.length <= 5000`
- `1 <= words[i].length <= 30`
- `words[i]` 和 `s` 由小写英文字母组成

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**按单词长度分起点 + 滑动窗口**：串联子串长度固定为 `word_len * len(words)`。以 `0, 1, ..., word_len-1` 为**起点**各做一次滑动窗口，每次步进 `word_len`，这样窗口按「整词」滑动。用 `Counter(words)` 得到目标词频 `target_cnt`，窗口内用 `cnt` 统计；用 `overload` 表示「出现次数超过 target 的单词种类数」。当窗口长度够且 `overload == 0` 时，窗口内单词与 `words` 恰好一致。

**关键词**：`word_len 种起点` `步长 word_len` `overload 判恰好匹配`

### 复杂度速记

- **时间**：O(word_len × (n / word_len)) = O(n)——每种起点线性扫一遍，共 word_len 种。
- **空间**：O(words 词数)——存 target_cnt 与窗口 cnt。

### 记忆口诀

「按词长分起点，步长词长滑动；overload 记多出来的种类，为 0 且窗口满即答案。」

---

## 解题思路

### 核心思想

- 串联子串长度固定：`window_len = word_len * len(words)`，且由若干**完整单词**拼接而成。
- 若从位置 0 开始按词切分，单词边界是 0, word_len, 2*word_len, ...；若从位置 1 开始，边界是 1, 1+word_len, ...。因此只有 **word_len 种**「对齐方式」，每种对齐下窗口的左右边界都落在单词边界上。
- 对每种起点 `start in [0, word_len)`，用右端点 `right = start + word_len, start + 2*word_len, ...` 滑动。窗口内用哈希表 `cnt` 统计每个单词出现次数，与 `target_cnt = Counter(words)` 比较。用 **overload** 记录「当前出现次数**大于** target 的单词种类数」：若窗口长度为 window_len 且 overload 为 0，则没有多也没有少，恰好匹配。

### 算法步骤

1. `word_len = len(words[0])`，`window_len = word_len * len(words)`，`target_cnt = Counter(words)`。
2. 对 `start` 从 0 到 `word_len - 1`：
   - 清空 `cnt`，`overload = 0`。
   - 对 `right = start + word_len, start + 2*word_len, ..., len(s)`（即窗口最后一个单词的右端点）：
     - **进窗**：`in_word = s[right - word_len : right]`；若加一前 `cnt[in_word] == target_cnt[in_word]` 则 `overload += 1`，然后 `cnt[in_word] += 1`。
     - `left = right - window_len`；若 `left < 0` 则跳过。
     - 若 `overload == 0`，则 `left` 为一个答案。
     - **出窗**：`out_word = s[left : left + word_len]`，`cnt[out_word] -= 1`；若减一后 `cnt[out_word] == target_cnt[out_word]` 则 `overload -= 1`。
3. 返回所有收集到的 `left`。

## 代码实现

```python
class Solution:
    def findSubstring(self, s: str, words: List[str]) -> List[int]:
        word_len = len(words[0])  # 一个单词的长度
        window_len = word_len * len(words)  # 所有单词的总长度，即窗口大小

        # 目标：窗口中的单词出现次数必须与 target_cnt 完全一致
        target_cnt = Counter(words)

        ans = []
        # 枚举第一个窗口的左端点，做 word_len 次起点不同的滑动窗口
        for start in range(word_len):
            cnt = defaultdict(int)
            overload = 0  # 统计过多的单词个数（包括不在 words 中的单词）
            # 枚举窗口最后一个单词的右开端点
            for right in range(start + word_len, len(s) + 1, word_len):
                # 1. in_word 进入窗口
                in_word = s[right - word_len: right]
                # 下面 cnt[in_word] += 1 后，in_word 的出现次数过多
                if cnt[in_word] == target_cnt[in_word]:
                    overload += 1
                cnt[in_word] += 1

                left = right - window_len  # 窗口第一个单词的左端点
                if left < 0:  # 窗口大小不足 window_len
                    continue

                # 2. 更新答案
                # 如果没有超出 target_cnt 的单词，那么也不会有少于 target_cnt 的单词
                if overload == 0:
                    ans.append(left)

                # 3. 窗口最左边的单词 out_word 离开窗口，为下一轮循环做准备
                out_word = s[left: left + word_len]
                cnt[out_word] -= 1
                if cnt[out_word] == target_cnt[out_word]:
                    overload -= 1

        return ans
```

## 复杂度分析

| 项   | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n)   | n = len(s)；共 word_len 种起点，每种起点下 right 的取值个数 O(n/word_len)，总 O(n)。 |
| 空间 | O(m)   | m = len(words)；target_cnt 与 cnt 的键数不超过 words 中不同单词数。 |

---

## 易错点

- **必须枚举 word_len 种起点**：若只从 0 开始滑动，会漏掉如 `s = "abab"`, `words = ["ab","ba"]` 这种从位置 1 开始的答案。
- **overload 的含义**：是「出现次数**多于** target 的单词**种类**数」，不是个数。这样当窗口内恰好等于 target 时 overload 为 0；有任意单词多或少都会使 overload ≠ 0。
- **先判答案再出窗**：当前窗口 [left, right) 已满且 overload==0 时记录 left，再在本轮末尾把 out_word 移出，这样下一轮 right 右移时窗口仍是按「整词」滑动。
