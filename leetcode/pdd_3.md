# 拼多多三面：算法题预测（多模态算法 / 高级 CV 两方向）

> 说明：以下为根据 **JD 关键词** 与 **一二面已考题型**（斐波那契拼接串定位字符、LC4 两有序数组中位数）做的**概率性预测**，用于备考对齐，非真题。三面算法岗常见仍是 **1～2 道手写**，但可能穿插 **工程/Serving 追问**（不计入本节「算法题」列表）。

---

## 与一二面的衔接（命题习惯上的推断）

| 轮次 | 已覆盖能力 | 三面更可能补的缺口 |
|------|------------|-------------------|
| 一面 | 字符串 + 递推结构 + 不构造全量 | **图 / DP / 贪心 / 树** 或 **检索与有序结构** 的另一变种 |
| 二面 | 双有序数组 + 二分划分（Hard 经典） | 再考「同构双数组划分」概率偏低；更可能换 **TopK、区间、堆、单调栈、最短路径** 等 |

## 方向一：多模态算法工程师（CV / 多模态大模型）

JD 侧重：商品理解、属性、AIGC、**大规模数据 / 高并发 / 高吞吐 Serving**、CNN/ViT、PyTorch。

### 高概率编码题族

1. **TopK / 多路归并 / 堆**  
   - 与「排序、融合多路有序流」气质接近二面但不重复 LC4；常考：合并 K 个有序链表、前 K 个高频元素、数据流中位数（双堆）。  
   - **Serving 追问映射**：在线特征/日志聚合、限内存找热点。

2. **二分答案 / 二分 on 值域**  
   - 最小化最大值、分割数组的最大值、有序矩阵第 K 小（对值二分 + count）。  
   - **业务映射**：延迟/资源约束下的可调参搜索（口头联系即可）。

3. **区间与扫描线**  
   - 合并区间、会议室 II、插入区间。  
   - **业务映射**：批次推理窗口、资源调度时间段。

4. **字符串 / 子串统计（滑窗、哈希）**  
   - 与 OCR、文本侧多模态相关；如：异位词、最小覆盖子串、最长无重复子串。  
   - 一面已考「递推串」，三面更可能是 **滑窗/计数** 而非同款 Fib 串。

5. **图：最短路 / 拓扑**  
   - 网格 BFS、网络延迟时间、课程表（拓扑）。  
   - **业务映射**：依赖 DAG、流水线阶段（简化为图模型）。

### 中概率（若三面偏「难 + 综合」）

- **DP**：打家劫舍类、最长递增子序列、编辑距离（文本/纠错口头场景）。  
- **贪心 + 证明**：跳跃游戏 II、加油站 / 最低加油次数（871 同族）、任务调度。

### 与 JD 的对应（答题时可主动挂钩）

- **大规模 / 吞吐**：代码题写完后准备 **时间/空间复杂度、批处理 vs 流式、近似 TopK（蓄水池/Count-Min 思路口头）**。  
- **多模态**：题目本身未必是多模态，但 **Follow-up** 可能问「如何用 embedding 检索」「ANN 用什么结构」——属系统设计/检索，非纯 LC。

## 方向二：高级 CV 算法工程师

JD 侧重：商品图 **OCR**、**同款/检索**、检测跟踪、分割、**CLIP/ViT/BLIP**、无监督/自监督、电商图像检索。

### 高概率编码题族

1. **二维网格 / 矩阵**  
   - 岛屿数量、矩阵最长递增路径、旋转图像、螺旋矩阵。  
   - **CV 映射**：连通域、邻域遍历思维与代码习惯。

2. **字符串与序列（OCR / 文本后处理）**  
   - 最长公共子序列/子串、编辑距离、正则/通配符匹配（难）、括号/有效字符串。  
   - **CV 映射**：识别结果纠错、对齐、序列标注后处理（口头）。

3. **二分 / 排序 / 双指针（检索与对齐）**  
   - 在排序数组中找边界、两数之和变体、接雨水（双指针+单调）。  
   - **检索映射**：有序表上查找、简单版本「相似度排序后的截断」（抽象为数组题）。

4. **栈与单调栈**  
   - 每日温度、柱状图最大矩形、接雨水。  
   - **工程映射**：一维信号/直方图类统计（与检测后处理 histogram 弱相关）。

5. **贪心 / 区间**  
   - 无重叠区间、用最少数量的箭引爆气球、合并区间。  
   - **业务映射**：标注框/时间窗去重（抽象后仍是区间题）。

### 中概率

- **回溯 / 组合**：子集、排列、N 皇后（若考察代码完整性）。  
- **树**：路径和、LCA（若前两轮未考树）。  
- **位运算 / 状态压缩**：棋盘、小集合 DP（相对少见，可作备选）。

### 与 JD 的对应（答题时可主动挂钩）

- **同款 / 检索**：纯代码轮更常考 **TopK、堆、二分**；深度检索架构放在 **项目面/系统设计**。  
- **检测/分割**：卷积细节多在 **八股+项目**；算法轮仍可能是 **网格 BFS/DFS、DP 路径**。

## 备考顺序建议（三面前 3～5 天）

1. **每方向各选 2 题「高概率族」** 做到能默写边界、能口述不变式。  
2. **刻意练 1 题「贪心」**：把「为什么不会更差」讲清楚（三面常卡在这里）。  
3. **复习 LC4 的二分划分话术**：二面已考，三面可能追问「和 TopK/第 K 小的关系」。  
4. 准备 **30 秒版本**：每道题对应的 **电商/多模态/CV** 业务一句话映射（加分项，非必需）。

## 题外：可能不算「算法题」但三面常出现

- ViT 相对 CNN 的复杂度、如何降计算（patch、窗口注意力、蒸馏）。  
- CLIP/对比学习：InfoNCE、负样本、温度系数（概念 + 公式级）。  
- 向量检索：HNSW/IVF 选型、召回与精度权衡（与编码题分开展示）。  

若三面以 **项目深挖** 为主，手写题可能降为 **Medium** 或 **1 题**；仍以当天面试官风格为准。
