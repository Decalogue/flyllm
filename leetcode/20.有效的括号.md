# 20. 有效的括号

**标签**：`栈` `字符串` `简单`  
**分类**：栈应用、括号匹配  
**难度**：⭐ 简单  
**频率**：🔥🔥🔥 高频

---

## 题目描述

给定一个只包括 `'('`、`')'`、`'{'`、`'}'`、`'['`、`']'` 的字符串 `s`，判断字符串是否为**有效括号串**。

**有效**需满足：

1. 左括号必须用**相同类型**的右括号闭合。  
2. 左括号必须以**正确顺序**闭合。  
3. 每个右括号都有一个对应的**同类型**左括号。

### 示例 1

```
输入：s = "()"
输出：true
```

### 示例 2

```
输入：s = "()[]{}"
输出：true
```

### 示例 3

```
输入：s = "(]"
输出：false
```

### 示例 4

```
输入：s = "([)]"
输出：false
```

### 示例 5

```
输入：s = "{[]}"
输出：true
```

### 提示

- `1 <= s.length <= 10^4`
- `s` 仅由括号 `"()[]{}"` 组成

---

## 📋 面试要点速查（知识卡片）

### 核心思路

**一句话总结**：**左括号入栈**；遇右括号则看栈顶是否为**匹配的左括号**，是则弹出，否则非法；结束时栈必须**为空**。

**关键词**：`栈` `配对表` `顺序`

### 复杂度速记

- **时间**：O(n) 一遍扫描。  
- **空间**：O(n) 栈最坏存满左括号。

### 记忆口诀

「左进右对顶，对不上就挂；扫完栈要空。」

---

## 解题思路

### 核心思想

括号匹配具有**后进先出**结构：最近遇到的左括号，应当最先被对应的右括号关闭。用栈保存**尚未闭合的左括号**；每来一个右括号，必须与**当前栈顶**（最后一个未闭合的左括号）类型一致。

### 算法步骤

1. 初始化空栈 `st`，建立右括号到左括号的映射 `')'→'('` 等。  
2. 遍历每个字符 `c`：  
   - 若是左括号，入栈。  
   - 若是右括号：若栈空或栈顶不是匹配的左括号，返回 `False`；否则弹出栈顶。  
3. 遍历结束后，若栈为空则 `True`，否则还有未闭合左括号，为 `False`。

### 为什么不用计数？

仅靠「`(` 与 `)` 数量相等」**不够**，例如 `"([)]"` 数量对但顺序错，必须用栈刻画**嵌套关系**。

## 代码实现

### 写法一：哈希映射右→左（推荐）

```python
class Solution:
    def isValid(self, s: str) -> bool:
        st: list[str] = []
        pair = {")": "(", "]": "[", "}": "{"}
        for c in s:
            if c in "([{":
                st.append(c)
            else:
                if not st or st[-1] != pair[c]:
                    return False
                st.pop()
        return len(st) == 0
```

### 写法二：遇到右括号时压入「期望左括号」

思路等价，部分题解用「栈里存期望遇到的符号」；左括号写法一更直观。

## ⚠️ 常见陷阱

1. **只判断数量相等**  
   - ❌ `"([)]"` 会误判为合法。  
   - ✅ 必须用栈保证**嵌套顺序**。

2. **右括号来时栈已空**  
   - 如 `")("` 或单独 `"]"`，应先判 `not st` 再比栈顶。

3. **忘记最后检查栈空**  
   - 如 `"((("` 遍历完从未报错，但栈非空，应返回 `False`。

4. **与 32 题混淆**  
   - 详见下文「与 32 的对比」，勿把「整串合法」与「最长有效子串长度」混用同一套栈语义。

## 与 32. 最长有效括号的对比

两题都用栈处理括号，但**问题目标**和**栈里存什么**不同，面试时先分清题意再选模型。

| 对比项 | **20. 有效的括号** | **32. 最长有效括号** |
|--------|-------------------|---------------------|
| **输出** | `bool`：整个 `s` 是否合法 | `int`：最长**连续**合法括号**子串**的长度 |
| **括号** | `'(' ')' '[' ']' '{' '}'` | 仅 `'('` 与 `')'` |
| **栈中元素** | **左括号字符**（与右括号类型配对） | **下标**（用于算「到当前位置的有效段有多长」） |
| **栈初值** | 空栈 | **`[-1]`** 作为「有效段起点前一位」的虚拟分界 |
| **遇右括号** | **先判再弹**：栈空或栈顶类型不对 → 立刻非法，**不 pop**；只有配对成功才 `pop` 栈顶左括号 | **先弹再判**：对 `')'` **无条件 `pop` 一次**，再根据弹后栈是否为空决定算长度还是压入新分界 |
| **遍历结束** | 栈必须为空才合法 | 与「栈是否为空」无直接 bool 关系，看维护的**最大长度** |
| **典型反例** | `"([)]"` 整串不合法 | `")()())"` 整串不合法，但最长有效子串长度为 4 |

### 弹出（pop）的时机差异（易混）

- **20**：`pop` 发生在**已经确认**「栈非空且栈顶与当前右括号匹配」**之后**；若不匹配，`pop` **不会发生**，直接返回 `False`。语义是：**消掉一对已闭合的括号**。  
- **32**：每遇到 `')'` **先执行一次** `pop()`（消掉匹配的 `(` 下标，或消掉底部分界 `-1`），**再根据弹完后的栈**决定：若仍非空，用当前栈顶下标更新长度；若为空，说明这个 `')'` 是「落单的」，把当前位置 `i` 压栈当下一段的**分界**。语义是：**用 pop 推进状态，再用新栈顶量距离**。

因此不能把 20 的「先 peek 再决定是否 pop」直接改成 32 的「见 `)` 就 pop」，反之亦然。

**一句话**：**20** 关心「**整条串**能不能配对完」；**32** 关心「**某一段连续子串**能配对多长」，且允许前后有无效字符，因此 32 的栈要记下标并维护**无效分界**，不能用 20 的「只存左括号字符」硬套。

完整题解见：[32. 最长有效括号](32.最长有效括号.md)。

## 复杂度分析

- **时间复杂度**：O(n)，每个字符入栈/出栈至多一次。  
- **空间复杂度**：O(n)，最坏字符串全为左括号。

---

## 关键点

1. **栈顶即「当前最内层」未闭合左括号**，与右括号一一对应。  
2. **映射表**避免长 `if-elif`，扩展新括号类型也方便。
